## 1. 문제 인식 (Problem) — 2~3p

### 1-1. 중소 제조업의 제조 데이터 문제 — "설계와 생산이 단절되어 있다"

국내 제조업 537,580개 사업체 중 10~299명 규모의 중소기업은 77,389개(14.4%)이다. 이 중 **95.6%가 PLM(제품 수명주기 관리) 시스템 없이** 엑셀, 공유폴더, 메일로 도면과 BOM(부품표)을 관리하고 있다. 생산 현장도 마찬가지다. 소기업 스마트공장 도입률 28.5%, 그중 MES 비중 14.4%로 **MES 실질 도입률은 약 4%**에 불과하다.

정부 스마트공장 실태조사에서 ERP(76.3%)는 보급되었으나, **PLM은 조사 항목에 포함조차 되지 않고, MES는 소기업 대부분이 미도입**이다. 설계 데이터와 생산 데이터가 각각 엑셀·수기에 머물러 있어 서로 연결되지 않는다.

{이미지 첨부 - Before/After 흐름도: 기존(도면→폴더, BOM→엑셀, 생산실적→수기, 공급사→머릿속) vs Fabbit(AI 자동 추출→구조화→설계+생산 통합→온톨로지 연결)}

### 1-2. 현장에서 발생하는 구체적 손실

**설계 데이터 문제:**

| 현장 문제               | 정량 근거                                                             | 출처           |
| ----------------------- | --------------------------------------------------------------------- | -------------- |
| **BOM 수동 입력**       | 엔지니어 1인당 연간 **52~260시간** 엑셀 BOM 작업                      | OpenBOM        |
| **도면 검색·버전 혼선** | 업무시간의 **25~30%** 정보 검색에 소비                                | AIIM, McKinsey |
| **BOM 오류 → 재발주**   | BOM 오류가 전체 ECO의 **최대 30%** 차지, 건당 행정비 **$1,500~3,000** | Aberdeen       |

**생산 현장 문제:**

| 현장 문제                 | 정량 근거                                                             | 출처                        |
| ------------------------- | --------------------------------------------------------------------- | --------------------------- |
| **생산 현황 파악 불가**   | 소기업 MES 실질 도입률 **약 4%** — 생산수량·불량률을 실시간 파악 불가 | 2024 스마트제조혁신실태조사 |
| **불량 원인 역추적 불가** | 설계 데이터와 생산 데이터가 분리 → 불량→부품·도면 역추적 불가         | 현장 인터뷰                 |
| **설계→생산 단절**        | 설계 변경이 현장 작업지시서에 반영 안 됨 → 구버전으로 생산            | 현장 인터뷰                 |

> **핵심 인사이트**: 문제는 개별 기능의 부재가 아니라 **"설계와 생산 데이터가 갇혀 있고, 서로 단절되어 있다"**는 것이다. 데이터가 연결되지 않으면 검색도, 변경 추적도, 품질 역추적도, AI 활용도 불가능하다.

### 1-3. 문제의 시급성 — 왜 지금인가

1. **제조 현장 고령화**: 숙련공 퇴직 시 머릿속 지식(부품 이력, 공급사 노하우)이 사라진다. 시스템에 없으면 복구 불가.
2. **공급망 불안정**: 미중 갈등, 소재 가격 변동으로 부품 대체·공급사 전환이 빈번해졌으나, 변경 영향을 추적할 수 없어 대응이 늦다.
3. **AI 전제조건 미충족**: 기업 AI 사용률이 급증(2023년 8.7% → 2025년 20.2%, OECD)하지만, **데이터가 구조화되지 않으면 AI는 쓸 수 없다.**
4. **통합 플랫폼 공백**: AI 추출 스타트업(투자 10억+)은 추출→Excel에서 끝나고, 기존 PLM/MES는 별도 제품. 설계+생산을 AI로 통합하는 플랫폼 영역이 비어 있다.

### 1-4. 고객 검증

콜드콜 36건+ 및 산업 컨퍼런스 현장 영업을 실행하며 확인한 결과:

- **타겟 보정**: 1~9명 영세기업은 "관리할 capa 자체가 없다" → **10~49명, 매출 10억+**으로 재설정
- **용어 보정**: "PLM"을 아는 업체 0건 → **"도면·BOM 관리"**로 풀어 설명해야 인식
- **핵심 니즈 확인**: "도면 찾는 데 시간 다 쓴다", "BOM 엑셀 수동 입력이 너무 비효율적"

## 2. 실현 가능성 (Solution) — 3~4p

### 2-1. Fabbit의 해결 방안 — 3단계 전략

**Fabbit**은 중소 제조업의 설계·생산 데이터를 AI로 통합하고, AI가 그 데이터에서 위험을 먼저 감지하여 행동까지 자동화하는 제조 데이터 플랫폼이다.

| 단계                          | 역할                                                               | 고객 가치                                |
| ----------------------------- | ------------------------------------------------------------------ | ---------------------------------------- |
| **1단계: AI 추출 (진입)**     | 도면·BOM을 AI가 자동 데이터화                                      | 도입 장벽 제거. 수동 입력 30분→3분       |
| **2단계: 설계+생산 통합**     | PLM 코어 기능을 기반으로 협약기간 내 경량 MES까지 확장             | 설계부터 생산까지 한 곳에서 관리         |
| **3단계: 데이터 연결 (핵심)** | 온톨로지로 설계·생산·공급사 데이터 연결 → AI 사전 감지·행동 자동화 | 변경 영향 추적, 품질 역추적, 리스크 감지 |

{이미지 첨부 - Fabbit 3단계 전략 흐름도: [1] AI 추출(진입) → [2] 설계+생산 통합(PLM+경량 MES) → [3] 온톨로지 연결 → Agentic AI(감지→대안→행동)}

### 2-2. 핵심 기술 및 차별성

#### AI 자동 추출 (1단계 — 현재 구현 완료)

- **BOM 자동 분석**: 도면(DWG/PDF) 업로드 시 LLM 기반 AI가 부품표를 자동 인식. 품명/재질/수량/규격을 추출하여 시스템 내 구조화 BOM으로 등록
- **도면 자동 인식 (고도화 예정)**: 표제란(Title Block) 인식, 도면 간 참조 관계 추출
- 기존 방식: 도면을 보며 Excel에 수동 입력 (건당 30분~1시간) → **Fabbit: 3분 이내**

#### 구조화 BOM 관리 (데이터 축적)

- 계층형 BOM 트리(Assembly-Sub Assembly-Part), Where-used(역추적), 리비전 추적
- 도면-BOM 연계 검색: "이 부품이 어디에 쓰이는지", "최신 버전이 맞는지" 즉시 확인

#### 설계+생산 통합 (2단계 — 설계부터 생산까지 한 곳에서)

소기업이 PLM과 MES를 별도로 도입하는 것은 비현실적이다. Fabbit은 현재 설계 관리(도면·BOM·변경관리)를 중심으로 구현되어 있으며, 협약기간 내 경량 생산 관리(작업지시·생산실적·품질추적)까지 확장하여 설계 변경이 생산에 반영되고 생산 불량이 설계로 역추적되는 연결 구조를 구축할 계획이다.

| 영역            | 기능                                      | 기존 방식          | Fabbit                  |
| --------------- | ----------------------------------------- | ------------------ | ----------------------- |
| 설계 (PLM)      | 도면·BOM 관리, 변경관리(ECR/ECO)          | 폴더+엑셀          | 통합 제공               |
| 생산 (경량 MES) | 작업지시, 생산실적, 품질추적              | 수기장부+엑셀      | 협약기간 내 통합 예정   |
| **연결**        | 설계 변경→작업지시 반영, 불량→부품 역추적 | 불가 (시스템 분리) | **연결 구조 구축 예정** |

#### Agentic AI — 데이터에서 AI가 먼저 발견하고 행동한다 (3단계 — 핵심 가치)

축적된 데이터를 그래프 DB(Apache AGE)로 연결. AI가 데이터를 지속 분석하여 **위험을 사전 감지하고 대안 탐색·행동 초안까지 자동화**한다.

| AI 단계                   | 방식                         | 예시                                      |
| ------------------------- | ---------------------------- | ----------------------------------------- |
| **Reactive** (질의응답)   | 사용자가 질문 → AI 답변      | "이 부품 어디에 쓰여?"                    |
| **Proactive** (사전 감지) | AI가 이상 감지 → 선제 알림   | "단일 공급사 납기 지연 3회 → 리스크 알림" |
| **Agentic** (자율 행동)   | 감지 → 대안 탐색 → 행동 초안 | "대체 공급사 3곳 추출 → 발주 초안 제시"   |

> 팔란티어는 이 개념을 **Systems of Action**이라 부른다. Fabbit은 AI 추출로 진입장벽을 제거하여, 팔란티어 Foundry의 효과를 **중소 제조업에서 실현**한다.

### 2-3. 경쟁사 대비 차별성

| 비교 축                | 비링커 (AI 도구) | 기존 PLM  | 마이링크 LinkBiz | **Fabbit**                   |
| ---------------------- | ---------------- | --------- | ---------------- | ---------------------------- |
| AI 자동 추출           | 있음 (→Excel)    | **없음**  | **없음**         | 있음 (→시스템 내 등록)       |
| 설계+생산 통합         | 없음             | PLM만     | PLM+PMS+MES      | **PLM 중심 + 경량 MES 확장** |
| 데이터 연결 (온톨로지) | 없음             | 없음      | 없음             | **있음**                     |
| AI 사전 감지·행동      | 없음             | 없음      | 일정 최적화만    | **있음 (Agentic AI)**        |
| 진입 장벽              | 구독 필요        | 수동 입력 | 모듈 선택 필요   | **Freemium 무료 진입**       |

{이미지 첨부 - 경쟁 포지셔닝 2x2 매트릭스: X축 "AI 데이터 추출(없음↔있음)", Y축 "설계+생산 통합(분리↔통합)". Fabbit=우상단}

> **비링커는 추출에서 끝나고, 기존 PLM/MES는 별도 제품이다. 마이링크는 통합형이지만 AI 추출이 없다. Fabbit은 AI 추출로 진입해 설계 관리와 생산 연결까지 확장하고, 쌓인 데이터 기반 AI 활용으로 발전시키는 전략을 갖고 있다.**

## 4. 팀 구성 (Team) — 1p

### 4-1. 대표자 역량

PLM 솔루션 기업에서 BOM 관리 모듈을 직접 개발하며 제조업 도메인 지식을 확보했다. 이후 SI 기업에서 프론트엔드·백엔드·인프라를 아우르는 풀스택 역량을 쌓았으며, 8명 규모 프로젝트의 기획·실행·일정 관리를 주도한 경험이 있다.

**실행 증거**: PLM 도메인 이해와 풀스택 개발 역량을 겸비하여, **도면·BOM AI 추출, PLM 코어 기능, 프론트엔드, 인프라를 단독 구축**한 상태이다. MVP 설계부터 구현·운영까지 1인 실행이 가능하다.

**창업 동기**: PLM 업계에서 직접 경험한 문제의식을 바탕으로, AI 기술의 성숙과 제조 데이터 플랫폼 시장의 공백을 확인하고 창업을 결정. 전 직장 대표도 방향성을 인정하며 자문 네트워크로 참여.
