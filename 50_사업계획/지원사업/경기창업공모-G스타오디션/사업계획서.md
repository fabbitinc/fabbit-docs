## 착안 배경

> 평가: 기획성 — 개발 동기, 목적, 필요성

### 중소 제조업의 제조 데이터 문제 — "데이터가 갇혀 있다"

국내 제조업 537,580개 사업체 중 10~299명 규모의 중소기업은 77,389개(14.4%)이며, 이 중 **95.6%가 PLM(제품 수명주기 관리) 없이** 엑셀, 공유폴더, 메일로 도면과 BOM(부품표)을 관리하고 있다. 도면은 폴더에, BOM은 엑셀에, 공급사 정보는 담당자 머릿속에 흩어져 있어 **검색·변경추적·의사결정이 불가능**하다.

| 현장 문제           | 정량 근거                                                               |
| ------------------- | ----------------------------------------------------------------------- |
| 도면/BOM 수동 입력  | 엔지니어 1인당 연간 52~260시간 엑셀 BOM 작업에 소요                     |
| 도면 검색·버전 혼선 | 업무시간의 25~30%를 정보 검색에 소비 (AIIM, McKinsey)                   |
| BOM 오류 → 재발주   | BOM 오류가 전체 설계변경(ECO)의 최대 30% 차지, 건당 행정비 $1,500~3,000 |
| 변경 영향 파악 불가 | 부품 하나를 바꿔도 어떤 제품·공급사·공정에 영향이 가는지 추적할 수 없음 |

정부 스마트공장 실태조사에서 ERP(76.3%), MES(14.4%)는 도입률을 집계하지만, **PLM은 조사 항목 자체에 포함되지 않는다.** 제조 데이터 관리가 시스템이 아닌 사람에게 의존하고 있으며, 이 데이터가 구조화되지 않는 한 AI를 활용한 의사결정 지원도 불가능하다.

### 개발 동기

PLM 솔루션 기업에서 BOM 관리 모듈을 직접 개발하며, 중소 제조업 현장의 문제를 목격했다. 고객은 "빠른 도입과 쉬운 사용"을 원했지만, 고객사별 커스텀 개발이 반복되면서 제품은 복잡해지고 도입 장벽은 높아졌다. 근본 원인은 **데이터가 구조화되지 않은 상태에서 관리 기능만 얹으려 한 것**이었다.

한편, AI 도면 분석 스타트업(비링커)이 투자 10억 이상을 유치하며 **AI로 제조 데이터를 추출하는 시장의 존재를 입증**했다. 그러나 비링커는 도면에서 데이터를 추출(→Excel)하는 데 그치며, 추출된 데이터를 연결하여 활용하는 단계로는 나아가지 않았다. 비링커가 위탁생산(매뉴팩처링)으로 방향을 전환하면서, **"AI로 데이터를 쌓고, 쌓인 데이터에서 AI가 스스로 위험을 감지하고 행동하는" 플랫폼 영역에 진입 기회**가 열려 있다.

### 목적

Fabbit은 중소 제조업의 **갇힌 데이터를 해방하고, AI가 그 데이터에서 위험을 먼저 감지하여 행동까지 자동화하는 플랫폼**이다.

1. **진입 (AI 추출로 데이터화)**: 도면·BOM을 AI가 자동 추출하여 도입 장벽을 없앤다. 엑셀에서 시스템으로 전환하는 첫 관문을 AI가 대신 넘긴다.
2. **핵심 가치 (AI가 먼저 발견하고 행동한다)**: 축적된 부품·공급사·도면·담당자 데이터를 온톨로지로 연결하고, **AI가 변경 영향·리스크·원가 이상을 사전 감지하여 대안 탐색·행동 초안까지 자동화**한다 (Agentic AI). 팔란티어가 이를 Systems of Action이라 부른다.

{이미지 첨부 - Fabbit 2단계 전략 흐름도: [1단계] 도면·BOM → AI 자동 추출 → 구조화 데이터 축적 / [2단계] 온톨로지 연결 → Agentic AI(사전 감지 → 대안 탐색 → 행동 자동화)}

## 보유 기술

> 평가: 기술성/성장성 — 경쟁력 있는 기술 여부 및 확보 방안, 기술 파급효과

### 1단계 기술 — AI 자동 추출로 데이터화 (진입 장벽 해소)

중소 제조업이 시스템을 도입하지 못하는 가장 큰 이유는 **기존 데이터를 옮기는 비용**이다. Fabbit은 AI가 이 과정을 대신한다.

- **BOM 자동 분석**: 도면(DWG/PDF) 업로드 시 LLM 기반 AI가 부품표를 자동 인식하여 품명/재질/수량/규격을 추출, 시스템 내 구조화 BOM으로 자동 등록 (현재 구현 완료)
- **도면 자동 인식 (고도화 예정)**: 표제란(Title Block) 인식, 도면 간 참조 관계 추출로 도면-BOM 자동 연계
- 기존 방식: 엔지니어가 도면을 보며 Excel에 수동 입력 (건당 30분~1시간)
- Fabbit: 업로드 → AI 자동 추출 → 구조화 등록 (3분 이내 목표)

### 2단계 기술 — 구조화 BOM 관리 (데이터 축적)

AI가 추출한 데이터를 Excel로 내보내는 데 그치지 않고, **시스템 안에서 계층형 BOM 트리(Assembly-Sub Assembly-Part)로 관리**한다. Where-used(역추적), 리비전 추적, 도면-BOM 연계 검색이 가능하여 "이 부품이 어디에 쓰이는지", "최신 버전이 맞는지"를 즉시 확인할 수 있다.

> 1단계에서 데이터를 쌓고, 2단계에서 관리하면서 데이터가 축적된다. 이 축적이 3단계의 전제 조건이다.

### 3단계 기술 — AI가 먼저 발견하고 행동하는 제조 데이터 플랫폼 (핵심 가치)

축적된 부품·공급사·담당자·도면 데이터를 그래프 DB(Apache AGE)로 연결한다. 기존 지식그래프+AI는 "사용자가 질문하면 답하는" 수동적 방식이지만, Fabbit은 **AI가 데이터를 지속 분석하여 위험을 사전 감지하고 대안 탐색·행동 초안까지 자동화**한다 (Agentic AI). 팔란티어는 이 개념을 **Systems of Action**이라 부르며, 분석에서 끝나지 않고 실제 비즈니스 행동으로 연결하는 것이 핵심이다.

- **사전 감지**: 부품 변경 시 영향받는 제품·공급사를 자동 탐색하여 담당자에게 선제 알림. 단일 소싱 리스크, 납기 지연 패턴을 AI가 사전 포착.
- **자율 행동**: 리스크 감지 → 대체 공급사 후보 추출 → 담당자에게 발주 초안 제시까지 자동화.
- 인프라 준비 완료: `age_client`(그래프 DB), `ontology`(품목 분류), `llm_client`(자연어 쿼리)

**온톨로지 기반 플랫폼의 검증된 효과:**

| 사례                 | 핵심 성과                                                   | 출처                 |
| -------------------- | ----------------------------------------------------------- | -------------------- |
| 팔란티어 Foundry     | ROI **315%**, 투자 회수 **6개월 미만**, 공급망 **30% 절감** | Forrester TEI (2023) |
| Airbus × Foundry     | A350 인도 속도 **33% 가속화**, 연 **$17억+** 비용 절감      | Airbus Skywise       |
| 지멘스 IKG           | as-maintained BOM 비용 **80% 절감**, 견적 속도 **90%↑**     | ISWC 2018            |
| RFS (독일 중소 제조) | 90일 만에 재고 **12% 감축**, 약 **150억원** 현금 확보       | Palantir Case Study  |

> 위 사례는 대기업 중심이다. Fabbit은 AI 추출로 진입장벽을 제거하여 이 효과를 **중소 제조업에서 실현**하는 것을 목표로 한다. 디지털 스레드 시장 CAGR 21.5% (2024~2030, MarketsandMarkets).

### 기술 파급효과

- **제조업 데이터 인프라 구축**: 엑셀/폴더에 갇힌 제조 데이터를 구조화·연결하여, 스마트공장/MES/ERP 연계의 기반 데이터 인프라 제공
- **AI가 실제로 동작하는 토대**: OECD 기준 기업 AI 사용률이 급증(2023년 8.7% → 2025년 20.2%)하지만, 데이터가 없으면 AI는 쓸 수 없다. Fabbit이 데이터 축적 → AI 활용의 경로를 제공. Gartner는 2025년까지 비즈니스 데이터 분석 혁신의 **70%가 지식그래프 기술**에 의해 추진될 것으로 전망
- **PLM 진입장벽 해소**: PLM 도입률 4.4%인 중소 제조업에 AI 자동 추출로 도입 비용을 제거하여, 디지털 전환 가속

### 경쟁 차별성

| 비교 축                        | 비링커 (AI 도구) | 기존 PLM (코너스톤 등) | **Fabbit**                                      |
| ------------------------------ | ---------------- | ---------------------- | ----------------------------------------------- |
| AI 자동 추출                   | 있음 (→Excel)    | **없음**               | 있음 (→시스템 내 자동 등록)                     |
| 구조화 BOM/변경관리            | 없음             | 있음                   | 있음                                            |
| 데이터 연결 (온톨로지)         | 없음             | 없음                   | **있음 (부품-공급사-도면-담당자)**              |
| AI 자율 감지·행동 (Agentic AI) | 없음             | 없음                   | **있음 — 위험 감지·대안 탐색·행동 초안 자동화** |
| 과금                           | 플랜별 3.3~5.5만 | 인당 2~4만             | **회사 단위 무료~59.9만**                       |

{이미지 첨부 - 경쟁 포지셔닝 2x2 매트릭스: X축 "AI 데이터 추출(없음↔있음)", Y축 "데이터 연결·AI 활용(없음↔있음)". 좌하=엑셀/폴더, 우하=비링커(추출만), 좌상=기존PLM(관리만), 우상=Fabbit(추출+연결+AI)}

> 비링커는 추출에서 끝나고, 기존 PLM은 수동 입력에서 시작한다. Fabbit은 AI 추출로 데이터를 축적하고, 이후 설계 관리와 데이터 연결 기반 AI 활용으로 확장하는 전략을 갖고 있다.

### 시장 진출 전략

1. **국내 (Year 1~2)**: 경기도/수도권 + 경남(기계)/대구(금속) 중소 제조업 직접 영업. 무료 Starter로 진입 → 유료 전환. 바우처 연계로 도입 장벽 해소.
2. **해외 (Year 2~3)**: 클라우드 SaaS 구조로 다국어 대응 용이. 동아시아(일본, 동남아) 우선 — 북미보다 늦게 제조업이 부상한 시장으로 PLM 도입률이 더 낮아 기회가 큼.

### 글로벌 사업화 의지 및 역량

- **제품 구조**: 클라우드 SaaS로 별도 설치 없이 글로벌 서비스 가능. Cloudflare R2 기반으로 글로벌 CDN/저장소 즉시 확장 가능.
- **다국어**: 프론트엔드 i18n 구조 설계 완료. 개발 고도화 대비 낮은 비용으로 현지화 가능.
- **시장 기회**: 글로벌 PLM 시장 아시아태평양이 가장 빠른 성장률. 비링커도 해외 매출 30%+, 미국 법인 설립 추진 중 → 한국 AI PLM 스타트업의 해외 시장 수요 입증.
- **비전**: AI가 제조 데이터에서 위험을 먼저 발견하고 행동까지 자동화하는 플랫폼 (Agentic AI). 팔란티어 Foundry(ROI 315%, Airbus 인도 33% 가속화)의 중소 제조업 버전을 목표.

## 조직 역량

> 평가: 조직역량 — 창업자 및 팀의 전문성

### 대표자 전문성

컴퓨터공학 전공 후 **PLM 솔루션 기업**에서 BOM 관리 모듈을 개발하며 제조업 도메인 지식을 확보했다. 이후 SI 기업에서 프론트엔드/백엔드/인프라를 아우르는 풀스택 역량을 쌓았으며, 8명 규모 프로젝트의 기획/실행/일정 관리를 주도한 경험이 있다.

PLM 도메인 이해와 풀스택 개발 역량을 겸비하여, **MVP 설계부터 구현/운영까지 1인 실행이 가능**하다. 현재 도면·BOM AI 추출, PLM 코어 기능, 프론트엔드, 인프라를 단독 구축한 상태이다.

### 팀 구성

대표가 제품 설계/개발 전체를 담당하고, 팀원이 영업/고객 접점을 담당하는 **제품-시장 2인 체제**이다.

| ---- | ------ | ------------------- | ------------------------------------------------------------------------------------------------ |
| 대표 | 문성하 | 제품 설계/개발 전체 | 컴퓨터공학 전공, PLM 솔루션 기업(BOM 모듈 개발) 1년, SI(풀스택/프로젝트 관리) 1.5년              |
| 팀원 | 남형우 | 영업/기술지원       | 고려대 생명공학과, DB 모니터링 제품 기술지원 경력. 콜드콜 36건+, 산업 컨퍼런스 현장 영업 실행 중 |
