# 제품 로드맵

Fabbit의 기능 목록, 현재 구현 상태, 단계별 우선순위를 정리한다.
사업계획서 Solution 항목의 개발 로드맵 근거로 사용한다.

> [!info] 제품 방향 변경 (2026-03-07)
> "클라우드 PLM" → **"제조 데이터 플랫폼"**으로 포지셔닝 변경.
> 설계 관리(PLM) + 경량 생산 관리(MES)를 하나의 플랫폼에서 제공.
> 상세 배경은 [[10_제품/아이템-정의]], [[10_제품/plm/제조-도메인-시스템]] 참고.

## 2. 기능 로드맵

### Tier 1: MVP (비링커 동등 + 핵심 차별)

비링커와 최소 동등해야 시장 진입 가능. 차별 기능이 없으면 존재 이유 없음.

| 기능 | 해결 문제 | 현재 상태 | 비링커 대비 | 비고 |
|------|----------|----------|-----------|------|
| 도면 AI 인식 정확도 고도화 | P1 | 부분 구현 (mapping/synthesis) | 동등 필요 | 비링커 수준(3초 내 추출) 도달 최우선 |
| 도면 뷰어 (DWG/PDF inline) | P2 | 미구현 | 동등 | 비링커는 CAD 없이 열람 가능 |
| 도면 버전관리 | P2 | 미구현 (파일 저장만) | 동등 | 업로드 이력 → 버전 체계로 승격 |
| 구조화 BOM 관리 (계층, where-used) | P3, P5 | 부분 구현 (`/parts/{id}/bom/tree`, reverse 포함) | **차별** | 리비전/비교 기능 추가 필요 |
| BOM 리비전 추적 | P3, P5 | 미구현 | **차별** | 도면 리비전과 BOM 리비전 연동 |
| BOM↔도면 연계 검색 | P2 | 부분 구현 (Part 상세에 drawing 관계 포함) | **차별** | 역추적 UX/검색 고도화 필요 |

> [!info] MVP 판단 기준
> - **동등 기능** (뷰어, 버전관리, AI 인식): 없으면 비링커 대비 열위 → 고객이 비링커 선택
> - **차별 기능** (구조화 BOM, 리비전, 연계 검색): 없으면 "비링커에 뭐가 다르냐"에 답 못함

### Tier 2: 베타 (파일럿 후 검증 기반)

MVP에서 고객 확보 후, 실제 사용 데이터로 수요 검증 후 구현.
경량 MES 기능을 이 단계에서 추가하여 "설계+생산 통합"을 실현한다.

#### 설계 관리 (PLM 고도화)

| 기능 | 해결 문제 | 비링커 대비 | 진입 조건 |
|------|----------|-----------|----------|
| ECR/ECO 워크플로우 고도화 | P4 | **차별** | 기본 CR 워크플로우는 구현됨. 파일럿 3사+ 기준으로 정책/권한/승인규칙 고도화 |
| 변경 영향 분석 (where-used 기반) | P4 | **차별** | ECR/ECO와 함께 |
| BOM 비교 (리비전 간 diff) | P3 | **차별** | BOM 리비전 데이터 축적 후 |
| 역할 기반 접근 제어 (RBAC) 고도화 | — | 동등 | 조직/프로젝트 기본 역할은 구현됨. 세분화 권한 정책은 운영 기준으로 확장 |

#### 생산 관리 (경량 MES)

상세 기획은 [[10_제품/plm/경량-mes-기획]]을 기준으로 한다.
이 로드맵에서는 전체 우선순위만 요약한다.

| 기능 | 해결 문제 | 데이터 모델 | 진입 조건 |
|------|----------|-----------|----------|
| **작업지시** | BOM 확정 → 현장 작업 배분 | `work_order` (BOM 참조, 수량, 납기, 우선순위, 상태) | MES MVP 1순위. Tier 1 BOM 관리 완료 후 |
| **생산실적** | 양품/불량 수량, 작업시간 기록 | `production_log` (작업지시 참조, 양품수, 불량수, 작업자) | MES MVP 2순위. 작업지시와 함께 |
| **불량기록 / 기본 품질** | 불량 수량·유형 기록, 최소 역추적 | `defect_log` 또는 `quality_inspection` 축소형 | MES MVP 3순위. 생산실적과 함께 |
| **품질관리 워크플로우** | 검사 기준, 판정, 재작업/폐기 처리 | `quality_inspection` | 베타. PLM 부품 데이터와 연결 |
| **자재·로트 추적** | 투입 자재와 완제품 연결 | `material_lot`, `material_usage` | 정식. 운영 데이터 축적 후 |
| **설비/가동 관리** | 설비 상태·비가동 사유 추적 | `equipment_log` | 확장. PLC/SCADA 수요 발생 시 |

> [!info] 경량 MES 판단 기준
> - 작업지시 + 생산실적은 **MES MVP 코어**
> - MVP 품질은 **불량기록**까지, 베타부터 검사·판정 워크플로우로 확장
> - 품질 데이터는 **PLM과 시너지 가장 큼** (불량 → 부품/도면 역추적 = 설계↔생산 피드백 루프)
> - 소기업 MES 실질 도입률 약 4%. "도면·BOM 관리 + 생산·품질 현황을 한 곳에서"가 포지셔닝
> - 경쟁사: 마이링크 LinkBiz(모듈형 통합)가 가장 유사. AI 추출 + 온톨로지가 Fabbit 차별점

#### AI 고도화

| 기능 | 해결 문제 | 비링커 대비 | 진입 조건 |
|------|----------|-----------|----------|
| **자연어 데이터 조회** (LLM 기반) | P2, P3 | **차별** | 설계+생산 데이터 축적 후 |

> [!info] 자연어 데이터 조회
> 시스템에 축적된 설계·생산 데이터(BOM, 부품, 공급사, 생산실적, 품질)를 자연어로 조회하는 기능.
> - 예시: "이 부품의 사용처 알려줘", "최근 불량률 높은 부품은?", "공급사 중 가장 싼 곳은?"
> - 방식: LLM → text-to-query (DB 조회) → 결과 응답. 생성형이 아닌 **검색형**으로 환각 차단.
> - 선행 조건: 설계 데이터(Tier 1) + 생산 데이터(Tier 2 경량 MES)가 축적되어야 쿼리 가치 극대화.
> - 인프라: `llm_client`, `age_client`(그래프 DB), `part`, `supplier`, `ontology` 모듈 활용.

> [!info] 포지셔닝 요약
> - **비링커 대비**: AI 동등 + 설계 관리(BOM·리비전·변경관리) + 생산 관리(경량 MES)에서 우위
> - **기존 PLM 대비**: PLM 기능은 동등하게 따라가되, AI 자동화 + 생산 연계에서 우위
> - **마이링크 대비**: PLM+MES 통합 컨셉은 유사하나, AI 자동 추출 + 온톨로지 기반 Agentic AI가 차별점
> - **Fabbit = AI 추출(진입) + 설계·생산 통합(가치) + 온톨로지(핵심 차별)**
