# 제품 로드맵

Fabbit의 기능 목록, 현재 구현 상태, 단계별 우선순위를 정리한다.
사업계획서 Solution 항목의 개발 로드맵 근거로 사용한다.

## 1. 현재 구현 상태

코드베이스(`server/`, `web/`, `drawing-converter/`) 기준.

### 서버 모듈

| 모듈 | 상태 | 설명 |
|------|------|------|
| `auth`, `activation` | ✅ 구현 | 인증, 회원가입 |
| `project` | ✅ 구현 | 프로젝트 단위 관리 |

## 2. 기능 로드맵

### Tier 1: MVP (비링커 동등 + 핵심 차별)

비링커와 최소 동등해야 시장 진입 가능. 차별 기능이 없으면 존재 이유 없음.

| 기능 | 해결 문제 | 현재 상태 | 비링커 대비 | 비고 |
|------|----------|----------|-----------|------|
| 도면 AI 인식 정확도 고도화 | P1 | 부분 구현 (mapping/synthesis) | 동등 필요 | 비링커 수준(3초 내 추출) 도달 최우선 |
| 도면 뷰어 (DWG/PDF inline) | P2 | 미구현 | 동등 | 비링커는 CAD 없이 열람 가능 |
| 도면 버전관리 | P2 | 미구현 (파일 저장만) | 동등 | 업로드 이력 → 버전 체계로 승격 |
| 구조화 BOM 관리 (계층, where-used) | P3, P5 | 미구현 | **차별** | part 모듈 확장 필요 |
| BOM 리비전 추적 | P3, P5 | 미구현 | **차별** | 도면 리비전과 BOM 리비전 연동 |
| BOM↔도면 연계 검색 | P2 | 미구현 | **차별** | "이 부품이 쓰인 도면" 역추적 |

> [!info] MVP 판단 기준
> - **동등 기능** (뷰어, 버전관리, AI 인식): 없으면 비링커 대비 열위 → 고객이 비링커 선택
> - **차별 기능** (구조화 BOM, 리비전, 연계 검색): 없으면 "비링커에 뭐가 다르냐"에 답 못함

### Tier 2: 베타 (파일럿 후 검증 기반)

MVP에서 고객 확보 후, 실제 사용 데이터로 수요 검증 후 구현.

| 기능 | 해결 문제 | 비링커 대비 | 진입 조건 |
|------|----------|-----------|----------|
| ECR/ECO 워크플로우 | P4 | **차별** | 파일럿 3사+, "변경관리 필요" 피드백 |
| 변경 영향 분석 (where-used 기반) | P4 | **차별** | ECR/ECO와 함께 |
| **PLM 자연어 대화** (LLM 기반 데이터 조회) | P2, P3 | **차별** | 구조화 BOM + 부품 + 공급사 데이터 축적 후 |
| 역할 기반 접근 제어 (RBAC) | — | 동등 | 다수 사용자 기업 도입 시 |
| BOM 비교 (리비전 간 diff) | P3 | **차별** | BOM 리비전 데이터 축적 후 |

> [!info] PLM 자연어 대화
> 시스템에 축적된 구조화 데이터(BOM, 부품, 공급사)를 자연어로 조회하는 기능.
> - 예시: "이 부품의 사용처 알려줘", "공급사 중 가장 싼 곳은?", "최근 변경된 BOM 목록"
> - 방식: LLM → text-to-query (DB 조회) → 결과 응답. 생성형이 아닌 **검색형**으로 환각 차단.
> - 선행 조건: 구조화 BOM(Tier 1)이 먼저 구현되어야 쿼리할 데이터가 존재함.
> - 차별 근거: 비링커는 데이터를 Excel로 내보내므로 시스템 내 쿼리 불가, 기존 PLM은 LLM 인프라 없음. Fabbit만 양쪽(데이터+LLM) 보유.
> - 인프라: `llm_client`, `age_client`(그래프 DB), `part`, `supplier`, `ontology` 모듈 활용.

## 3. 단계별 전환 조건

| 전환 | 트리거 | 핵심 기능 추가 | 예상 인력 |
|------|--------|-------------|----------|
| MVP → 베타 | 파일럿 3사+, BOM 정확도 90%+ | ECR/ECO, PLM 자연어 대화, RBAC, BOM diff | 확인 필요 |
| 베타 → 정식 | 유료전환 10사+, 변경관리 수요 검증 | ERP 연동, 협력사 포털, Excel import | 확인 필요 |
| 정식 → 확장 | ARR 5억+ | MBOM, 공정, 다국어 | 확인 필요 |

## 4. 경쟁사 기능 대비표

전체 기능 기준으로 Fabbit이 각 경쟁사 대비 어디에 위치하는지 표시.

| 기능 | 비링커 | 코너스톤 PLM | 싱글톤 A•one | JK-PLM | **Fabbit (목표)** |
|------|-------|-----------|-----------|--------|-----------------|
| 도면 AI 인식 | ✅ | ❌ | ❌ | ❌ | ✅ (MVP) |
| BOM 자동생성 | ✅ (→Excel) | ❌ | ❌ | ❌ | ✅ (→시스템, MVP) |
| 도면 뷰어 | ✅ | ✅ | ✅ | ✅ | ✅ (MVP) |
| 도면 버전관리 | ✅ | ✅ | ✅ | ✅ | ✅ (MVP) |
| 구조화 BOM 관리 | ❌ | ✅ (Adv) | ✅ | ✅ | ✅ (MVP) |
| BOM 리비전 추적 | ❌ | ✅ | ✅ | ✅ | ✅ (MVP) |
| BOM↔도면 연계 검색 | ❌ | ✅ | ✅ | ✅ | ✅ (MVP) |
| ECR/ECO | ❌ | 옵션 | ✅ | ✅ | ✅ (베타) |
| PLM 자연어 대화 | ❌ | ❌ | ❌ | ❌ | ✅ (베타) |
| RBAC | ✅ | ✅ | ✅ | ✅ | ✅ (베타) |
| ERP 연동 | ❌ | ❌ | ✅ | ✅ | ✅ (정식) |
| 협력사 포털 | ❌ | ❌ | ❌ | ❌ | ✅ (정식) |
| MBOM | ❌ | ❌ | ❌ | ✅ | ✅ (확장) |

> [!info] 포지셔닝 요약
> - **비링커 대비**: AI는 동등, PLM 기능(BOM 관리·리비전·변경관리)에서 우위
> - **기존 PLM 대비**: PLM 기능은 동등하게 따라가되, AI 자동화에서 우위
> - **Fabbit = AI(비링커 수준) + PLM(기존 PLM 수준)의 결합**
