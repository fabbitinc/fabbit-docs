# 경량 MES 기획

이 문서는 Fabbit의 경량 MES 범위, 주요 기능, MVP 우선순위, 단계별 확장
계획을 정리한다. 목표는 소기업 제조 현장에서 가장 먼저 필요한 실행 관리
기능을 좁은 범위로 도입하고, 이후 품질·추적성·연동으로 확장하는 기준을
만드는 것이다.

## 1. 기획 원칙

Fabbit의 경량 MES는 "대규모 공장 통합 제어 시스템"이 아니라, PLM에 축적된
설계 데이터를 생산 실행으로 연결하는 운영 레이어다.

- **타겟 고객**: 10~299명 중소 제조업, PLM/MES 미사용, Excel·수기 기반 운영
- **핵심 목표**: Released BOM을 기준으로 작업지시를 만들고, 생산실적과
  불량 데이터를 다시 설계 데이터와 연결
- **기본 원칙**: 현장 입력 부담이 적어야 하며, 기존 ERP/전문 MES를 대체하기보다
  비어 있는 운영 구간을 메운다
- **도입 기준**: "오늘 무엇을 얼마나 만들었는지"와 "어디서 불량이 났는지"를
  당일 안에 파악할 수 있어야 한다

> [!warning] 초기 범위에서 제외
>
> 아래 항목은 초기 범위에 넣지 않는다.
> - PLC/SCADA 직접 연동
> - APS 수준의 정교한 생산계획 최적화
> - 창고관리시스템(WMS) 전체 대체
> - 설비 예방보전, OEE 고도화
> - 대기업형 다공장 표준 공정 운영

## 2. 주요 기능 체계

경량 MES의 주요 기능은 "실행, 기록, 피드백, 추적" 순서로 확장한다.

| 기능 영역 | 핵심 질문 | 주요 사용자 액션 | 기대 효과 | 권장 단계 |
|-----------|-----------|------------------|-----------|-----------|
| **생산 기준본 관리** | 어떤 BOM/리비전으로 생산해야 하나 | Released BOM과 도면 버전을 작업 기준으로 고정 | 구버전 생산 방지 | 선행 조건 |
| **작업지시** | 오늘 무엇을 몇 개 만들어야 하나 | 작업지시 생성, 배정, 상태 변경, 우선순위 조정 | 현장 업무 시작점 표준화 | MVP |
| **생산실적** | 실제로 얼마나 생산했나 | 양품/불량 수량, 작업시간, 작업자 입력 | 일일 생산 가시성 확보 | MVP |
| **불량기록** | 어떤 불량이 어디서 발생했나 | 불량 수량, 유형, 메모 기록 | 품질 문제의 최소 추적성 확보 | MVP |
| **품질관리 워크플로우** | 검사 결과를 어떻게 판정·조치하나 | 검사 체크, 합격/불합격 판정, 재작업/폐기 분류 | 품질 데이터의 운영 활용도 상승 | 베타 |
| **자재·로트 추적** | 어떤 자재가 어떤 생산에 투입됐나 | 자재 사용 등록, 로트 연결, 역추적 조회 | 문제 발생 시 회수 범위 축소 | 정식 |
| **공정/라우팅** | 어떤 순서로 작업이 진행돼야 하나 | 공정 단계 정의, 공정별 진행 관리 | 복수 공정 제품의 흐름 제어 | 확장 |
| **설비/가동 관리** | 설비 상태가 생산에 어떤 영향을 주나 | 설비 상태 기록, 고장 이력, 비가동 사유 입력 | 생산 차질 원인 분석 | 확장 |
| **현황판/알림** | 지금 어디가 밀리고 있나 | 대시보드 확인, 지연/불량 알림 수신 | 관리자 의사결정 속도 향상 | MVP |

## 3. MVP 우선순위

MVP는 "엑셀 생산관리 탈출"에 필요한 최소 범위만 포함한다. 기능을 많이 넣는
것보다, 현장에서 매일 쓰는 흐름을 먼저 닫는 것이 중요하다.

### 선행 조건

MES 화면을 만들기 전에 아래 조건이 먼저 충족되어야 한다.

1. Released BOM과 도면 리비전을 생산 기준본으로 고정할 수 있어야 한다.
2. 작업지시가 참조할 부품/수량/납기 정보가 구조화되어 있어야 한다.
3. 조직/팀/멤버 권한이 최소 수준으로 정리되어 있어야 한다.

### MVP 포함 기능

1. **작업지시**
   작업의 시작점이다. 무엇을 언제까지 얼마나 생산할지 정하지 못하면,
   이후 실적과 품질 데이터도 쌓이지 않는다.
2. **생산실적**
   양품/불량/작업시간을 기록해야 일일 생산 현황이 보인다. 경영자와
   현장 관리자 모두 가장 먼저 체감하는 가치다.
3. **불량기록**
   초기에는 정밀한 검사 시스템보다 "불량 수량과 유형을 남기는 것"이 더
   중요하다. 최소 기록만 있어도 설계 데이터와 품질 데이터를 연결할 수 있다.
4. **현황판/알림**
   생산 중, 완료, 지연, 불량 발생을 한 화면에서 보게 해야 관리자 사용성이
   생긴다.

### MVP에서 제외하는 기능

- 상세 품질검사 체크리스트와 승인 워크플로우
- 자재 입고/투입/재고 차감 전체 흐름
- 설비 상태 자동 수집
- 복잡한 공정 라우팅
- ERP 양방향 동기화

> [!info] 품질 기능의 범위 구분
>
> MVP의 품질은 **불량기록**까지다.
> 베타부터 **검사 기준, 판정, 조치 워크플로우**를 포함한 품질관리로 확장한다.

## 4. 단계별 세부 기획

이 장은 실제 추진 순서를 정의한다. 각 단계는 앞단계에서 축적된 데이터가
다음 단계의 전제조건이 된다.

### Phase 0. 생산 기준본 준비

PLM 데이터를 생산에 넘길 수 있는 최소 기준을 먼저 만든다.

- Released BOM 스냅샷 정의
- 도면/부품 리비전과 작업지시 참조 규칙 정리
- 생산 단위(라인, 셀, 팀 또는 담당자) 정의
- 작업지시 상태값 초안 정의
  - 예: `draft`, `released`, `in_progress`, `done`, `cancelled`

**완료 기준**

- "어떤 버전으로 생산했는지"를 작업지시 단위로 설명할 수 있다
- 생산 기준본이 변경되더라도 기존 작업 이력은 유지된다

### Phase 1. MES MVP

생산 실행과 실적 수집의 기본 흐름을 닫는다.

#### 1-1. 작업지시

- 작업지시 생성, 조회, 수정, 취소
- 수량, 납기, 우선순위, 담당 라인/팀 지정
- 작업지시 상세에서 참조 BOM/도면 확인

#### 1-2. 생산실적

- 작업 시작/종료 또는 일괄 실적 입력
- 양품 수량, 불량 수량, 작업시간 기록
- 작업지시 상태 자동 갱신

#### 1-3. 불량기록

- 불량 유형 코드 선택
- 간단 메모, 발생 공정, 발생 시점 기록
- 부품/도면 역추적을 위한 연결 키 저장

#### 1-4. 현황판

- 오늘 작업지시 현황
- 진행 중/완료/지연 건수
- 일일 생산량, 불량량 요약

**MVP 성공 지표**

- 작업지시 생성 후 현장 전달까지 10분 이내
- 생산 종료 당일 실적 입력률 90% 이상
- 불량 발생 건의 80% 이상이 유형 코드로 기록
- 관리자가 엑셀 없이 일일 생산량을 확인 가능

### Phase 2. 베타

MVP에서 쌓인 실행 데이터를 품질과 변경관리로 연결한다.

- 검사 기준 기반 품질검사 기록
- 합격/불합격, 재작업, 폐기 처리 분기
- 설계 변경(ECO) 발생 시 진행 중 작업지시 영향 표시
- 현장 입력 UX 고도화
  - 모바일 우선 입력
  - QR/바코드 기반 작업 호출
- 관리자 알림
  - 지연 작업
  - 반복 불량
  - 설계 변경 영향 작업

**베타 목표**

- 생산 데이터가 단순 기록을 넘어 품질 판단과 변경 반영에 쓰이기 시작
- "불량 → 부품/도면" 역추적 데모가 가능

### Phase 3. 정식

추적성과 외부 연동을 강화한다. 이 단계부터는 운영 신뢰성이 제품 경쟁력에
직접 연결된다.

- 자재/로트 단위 투입 기록
- 완제품 기준 역추적 조회
- ERP 연동
  - 작업오더 수신 또는 실적 송신 중 한 방향부터 시작
- 생산 마감, 자재 소모, 실적 확정 처리 분리
- 고객사 운영 규칙에 맞춘 권한/감사 로그 강화

**정식 목표**

- 특정 불량 건이 어떤 자재/로트/작업지시와 연결되는지 설명 가능
- 고객 ERP와 중복 입력을 줄일 수 있음

### Phase 4. 확장

복수 공정, 설비, OT 연동이 필요한 고객으로 확장한다.

- 공정 라우팅
- 설비/가동/비가동 사유 관리
- OEE 등 설비 기반 지표
- PLC/SCADA 연동
- 다공장/다라인 운영

**확장 목표**

- Fabbit이 경량 MES를 넘어 MOM 일부 역할까지 수행
- 단, 이 단계는 소기업 초기 타겟을 넘어서므로 선행 고객 확보 후 검토

## 5. 기능별 권장 순서

복잡도를 고려한 추천 순서는 아래와 같다.

| 순서 | 기능 | 이유 |
|------|------|------|
| 1 | 생산 기준본 관리 | 작업지시의 기준이 없으면 이후 데이터가 모두 흔들림 |
| 2 | 작업지시 | 현장 실행의 시작점 |
| 3 | 생산실적 | 가시성 확보의 핵심 |
| 4 | 불량기록 | 품질 피드백 루프의 최소 단위 |
| 5 | 현황판/알림 | 관리자 사용성 확보 |
| 6 | 품질관리 워크플로우 | 단순 기록을 운영 프로세스로 승격 |
| 7 | 자재·로트 추적 | 회수/역추적 가치가 큼, 하지만 운영 복잡도 증가 |
| 8 | ERP 연동 | 고객별 편차가 커서 파일럿 이후가 적절 |
| 9 | 공정/설비 관리 | 난이도와 구현 범위가 가장 큼 |

## 6. Fabbit 기준 포지셔닝

Fabbit의 경량 MES는 전문 MES의 축소판이 아니라, PLM과 붙어 있을 때
가치가 커지는 실행 관리 계층이다.

- **비링커 대비**: 추출 이후 생산 실행까지 연결
- **기존 PLM 대비**: 설계 관리에서 끝나지 않고 현장 피드백 수집
- **전문 MES 대비**: PLC/SCADA 없이도 도입 가능한 가벼운 범위

> [!info] 한 문장 정의
>
> Fabbit의 경량 MES는 "Released BOM을 작업지시로 내려 보내고,
> 생산실적과 불량 데이터를 다시 설계 데이터로 연결하는 실행 관리 레이어"다.
