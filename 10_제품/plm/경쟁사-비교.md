# 경쟁사/대체재 비교

이 문서는 Fabbit의 경쟁 환경을 정리한다.
사업계획서 Solution 항목의 차별성 근거로 사용한다.

## 1. 엔터프라이즈 PLM

대기업 중심 솔루션. 중소 제조업에는 비용·복잡도 장벽이 높다.

| 항목 | PTC Windchill | Siemens Teamcenter | Dassault ENOVIA |
|------|-------------|-------------------|----------------|
| 가격 (사용자/월) | ~$1,000 (시작가) | $150~300 | 커스텀 (~$15 클라우드 시작) |
| 연간 비용 (사용자) | 커스텀 | ~$7,000 | 커스텀 |
| 최소 도입 비용 | 커스텀 견적 | $10K~50K (구현비) | 커스텀 견적 |
| 도입 기간 | 6~12개월+ | 3~6개월 | 6~12개월+ |
| 배포 방식 | 온프레미스/클라우드 | 온프레미스/MindSphere | 온프레미스/3DEXPERIENCE 클라우드 |
| 주요 고객 | 대기업, 항공/방산 | 대기업, 자동차 | 대기업, 항공/자동차 |
| 한국어 지원 | 있음 | 있음 | 있음 |
| BOM 관리 | 있음 | 있음 | 있음 |
| 도면 AI 인식 | 없음 | 없음 | 없음 |

> [!info] 중소기업 관점의 핵심 장벽
> - 도입 비용: 최소 수천만 원~수억 원
> - 도입 기간: 최소 3개월, 통상 6~12개월
> - 복잡도: 기존 관리방식보다 복잡해서 도입 실패 사례 다수
> - 전담 인력: IT 운영/관리 인력 필요

## 3. 현행 방식 (엑셀/파일서버/종이)

중소 제조업의 **실질적 경쟁 상대**. 대부분의 타겟 고객이 현재 이 방식으로 운영 중.

### 엑셀 BOM 관리의 문제점

- **버전 충돌**: 부서별로 각자 복사본 사용 → 단일 진실 원천(Single Source of Truth) 부재
- **수동 입력 오류**: 연간 780시간 엔지니어링 시간 낭비, $39K 인건비 (중견기업 기준)
- **수량/사양 오류**: 제품 원가의 3~5%가 잘못된 수량 발주로 낭비
- **변경 반영 지연**: 설계 변경(ECO) 프로세스가 수일~수주 소요
- **확장 불가**: 품목 수 증가 시 관리 한계

### 파일서버/공유폴더 도면관리의 문제점

- **검색 시간**: 엔지니어가 업무시간의 최대 30%를 올바른 부품/도면 정보 검색에 소비
- **버전 혼선**: 최신 도면 여부 확인 불가 → 구버전으로 생산/발주
- **접근 제어 없음**: 누가 언제 수정했는지 이력 추적 불가
- **협업 불가**: 외부(협력사) 공유 시 이메일/USB 기반

### 종이 도면의 문제점

- 변경사항 즉시 반영 불가
- 물리적 분실/훼손 리스크
- 검색·복제에 시간 소요

## 4. Fabbit 포지셔닝

| 비교 축 | 엔터프라이즈 PLM | 클라우드 PLM | 엑셀/파일서버 | **Fabbit** |
|---------|--------------|-----------|------------|-----------|
| 도입 비용 | 수천만~수억 | $360~2,100/인/년 | 0원 | 미정 |
| 도입 기간 | 3~12개월 | 즉시~수주 | 없음 | 즉시 (목표) |
| 학습 곡선 | 높음 (도입 실패 다수) | 중간 | 없음 | 낮음 (목표) |
| BOM 관리 | 있음 | 있음 | 수동 | AI 자동생성 |
| 도면 AI 인식 | **없음** | **없음** | **없음** | **있음 (핵심 차별점)** |
| 한국어 지원 | 있음 | 대부분 없음 | — | 있음 |
| 타겟 고객 | 대기업 | 중견~중소 | 전체 | 중소 제조업 |

> [!info] 핵심 차별점 요약
> 1. **도면 AI 인식 → BOM 자동생성**: 경쟁사 어디에도 없는 기능
> 2. **즉시 도입 + 낮은 학습곡선**: 엑셀에서 바로 전환 가능
> 3. **한국어 네이티브**: 클라우드 PLM 중 한국어 지원 부재 시장 공략
> 4. **중소 제조업 특화 가격**: 엔터프라이즈 PLM 대비 1/10 이하 목표
