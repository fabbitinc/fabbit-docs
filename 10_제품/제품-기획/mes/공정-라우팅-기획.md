# 공정-라우팅 기획

이 문서는 확장 단계에서 검토할 공정·라우팅 기능을 정의한다. 목표는 복수 공정을
거치는 제품의 작업 순서를 시스템에서 제어하는 것이다.

## 1. 문제와 목표

- 단순 조립형 제품을 넘어서면 작업 순서를 통제해야 한다.
- 공정별 상태를 분리해야 병목 구간이 보인다.
- 단일 작업지시만으로는 복수 공정 제품을 설명하기 어렵다.

## 2. 주요 사용자

- 생산기술
- 생산관리자
- 현장 리더

## 3. 핵심 시나리오

1. 품목별 표준 공정을 정의한다.
2. 작업지시가 공정 단계로 나뉜다.
3. 각 공정의 시작/완료 상태를 추적한다.
4. 병목 공정과 지연 원인을 확인한다.

## 4. 화면 기획

### 공정 정의 화면

- 공정 순서
- 표준 소요시간
- 담당 조직
- 선후행 관계

### 공정 진행 화면

- 단계별 상태
- 현재 공정
- 지연 단계 표시

## 5. 기능 기획

- 품목별 표준 공정 정의
- 작업지시의 공정 분할
- 공정별 상태 관리
- 병목 공정 조회

## 6. 정책과 예외

- 초기 고객이 단일 공정 위주면 도입하지 않는다
- 공정 정의는 품목 또는 품목군 단위로 관리 검토
- 수작업 공정 변경 이력 필요

## 7. 데이터 연결

- 작업지시
- 생산실적
- 향후 설비, 작업자, 표준시간 데이터

## 8. 확장 범위

- 공정 정의
- 공정별 상태 관리
- 병목 분석

## 9. 후속 확장

- 공정별 검사 연결
- 표준시간 대비 편차 분석
- 공정 자동 배정
