# 품질관리 기획

이 문서는 MES 베타 단계의 품질관리 기능을 정의한다. 불량기록이 "사실 기록"에
초점을 둔다면, 품질관리는 검사, 판정, 조치까지 포함한 운영 프로세스다.

## 1. 문제와 목표

- 불량이 기록돼도 합격/불합격, 재작업, 폐기 기준이 따로 놀면 운영에 쓰기 어렵다.
- 품질 담당자는 검사 결과를 표준화해 남기고 싶다.
- 설계와 연결된 품질 피드백이 필요하다.

## 2. 주요 사용자

- 품질 담당자
- 생산관리자
- 현장 리더

## 3. 핵심 시나리오

1. 품질 담당자가 작업 또는 제품 기준으로 검사를 수행한다.
2. 검사 결과를 합격, 불합격, 재작업, 폐기 중 하나로 판정한다.
3. 필요 시 불량기록, 작업지시, 품목 데이터와 연결한다.
4. 반복 불량은 설계 변경 검토로 이어진다.

## 4. 화면 기획

### 검사 목록 화면

- 대기, 검사 중, 판정 완료, 재작업 대기 상태
- 작업번호, 품목, 검사일, 판정 결과

### 검사 상세 화면

- 검사 기준
- 측정/판정 결과
- 조치 결과
- 연결 불량기록

### 조치 화면

- 재작업 요청
- 폐기 처리
- 설계 검토 필요 표시

## 5. 기능 기획

- 검사 기준 등록
- 검사 결과 기록
- 판정 상태 관리
- 재작업/폐기 분기
- 품질 이슈 누적 조회

## 6. 정책과 예외

- 검사 완료 후 일반 사용자는 수정 불가
- 품질 책임자만 판정 변경 가능
- 재작업 건은 원작업과 연결해 이력 유지

## 7. 데이터 연결

- 불량기록
- 작업지시
- 생산실적
- 품목 / 도면 / 변경관리

## 8. MVP 제외 범위

- 통계적 공정관리(SPC)
- 측정장비 자동 연동
- 복잡한 승인 체계

## 9. 후속 확장

- 검사표 템플릿
- 사진/파일 첨부
- 고객 클레임 연결
- 변경관리 자동 제안
