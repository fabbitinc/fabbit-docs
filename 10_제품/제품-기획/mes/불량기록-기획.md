# 불량기록 기획

이 문서는 MES MVP에서 가장 먼저 필요한 품질 기능인 불량기록을 정의한다.
목표는 정교한 검사 시스템보다 먼저, 불량 발생 사실과 유형을 구조화해 남기는
것이다.

## 1. 문제와 목표

- 불량이 발생해도 원인과 유형이 남지 않아 재발 방지가 어렵다.
- 설계와 생산이 분리되어 있어 어떤 부품/도면과 연관되는지 추적하기 어렵다.
- MVP 단계에서는 최소한의 품질 피드백 루프가 필요하다.

## 2. 주요 사용자

- 작업자
- 현장 리더
- 품질 담당자

## 3. 핵심 시나리오

1. 생산실적 입력 중 불량 수량이 기록된다.
2. 불량 유형과 간단 메모를 남긴다.
3. 작업지시, 품목, 시점과 연결한다.
4. 이후 품질관리 문서에서 검사/조치와 연결한다.

## 4. 화면 기획

### 불량 입력 화면

- 기본 정보
  - 작업번호, 품목명, 생산일시
- 입력 항목
  - 불량 수량
  - 불량 유형 코드
  - 발생 공정
  - 메모

### 불량 목록 화면

- 기간, 품목, 유형, 담당 기준 필터
- 자주 발생하는 불량 유형 정렬
- 작업지시/품목 상세로 이동

### 품목 또는 작업 상세 내 불량 섹션

- 누적 불량 건수
- 최근 불량 유형
- 연결된 품질조치 여부

## 5. 기능 기획

- 불량 유형 선택형 입력
- 작업지시/생산실적과 연결
- 메모 기록
- 기간/유형별 조회
- 반복 불량 탐지용 기본 집계

## 6. 정책과 예외

- MVP에서는 사진 첨부 없이 시작 가능
- 자유 텍스트만 허용하지 않고 유형 코드를 기본값으로 둔다
- 품질 담당자가 유형 코드를 정정할 수 있다

## 7. 데이터 연결

- 작업지시
- 생산실적
- 품목 / 도면 / BOM
- 향후 품질관리 조치 기록

## 8. MVP 범위

- 불량 수량
- 불량 유형 코드
- 메모
- 기본 조회

## 9. 후속 확장

- 사진 첨부
- 원인 분류
- 재작업/폐기 분기
- ECO 영향 분석 연결
