# 불량기록 기획

이 문서는 MES MVP에서 가장 먼저 필요한 품질 기능인 불량기록을 정의한다.
목표는 정교한 검사 시스템보다 먼저, 불량 발생 사실과 유형을 구조화해 남기는
것이다.

## 1. 문제와 목표

- 불량이 발생해도 원인과 유형이 남지 않아 재발 방지가 어렵다.
- 설계와 생산이 분리되어 있어 어떤 부품/도면과 연관되는지 추적하기 어렵다.
- MVP 단계에서는 최소한의 품질 피드백 루프가 필요하다.

## 2. 주요 사용자

- 작업자
- 현장 리더
- 품질 담당자

## 3. 메뉴명과 진입 경로

MVP 단계에서는 사용자가 "품질" 아래에서 불량을 찾는 흐름이 자연스럽다.

| 구분 | 권장 명칭 | 비고 |
|------|-----------|------|
| 상위 메뉴 | `생산` | 공통 상위 메뉴 |
| 하위 메뉴 | `품질` | 메뉴는 넓게 |
| 화면 제목 | `불량 기록` | MVP 핵심 화면 제목 |
| 기본 CTA | `불량 기록 추가` | 입력 의도 명확화 |

진입 경로 예시는 아래와 같다.

`생산 > 품질 > 불량 기록`

## 4. 핵심 시나리오

1. 생산실적 입력 중 불량 수량이 기록된다.
2. 불량 유형과 간단 메모를 남긴다.
3. 작업지시, 품목, 시점과 연결한다.
4. 이후 품질관리 문서에서 검사/조치와 연결한다.

## 5. 화면 기획

### 불량 입력 화면

- 기본 정보
  - 작업번호, 품목명, 생산일시
- 입력 항목
  - 불량 수량
  - 불량 유형 코드
  - 발생 공정
  - 메모

### 불량 목록 화면

- 기간, 품목, 유형, 담당 기준 필터
- 자주 발생하는 불량 유형 정렬
- 작업지시/품목 상세로 이동

### 품목 또는 작업 상세 내 불량 섹션

- 누적 불량 건수
- 최근 불량 유형
- 연결된 품질조치 여부

## 6. 화면 예시

### 불량 입력 화면 예시

```text
[생산] / 품질

┌──────────────────────────────────────────────────────────────┐
│ 불량 기록 추가                              저장              │
├──────────────────────────────────────────────────────────────┤
│ 작업번호: WO-240301   품목명: 본체 조립   생산일시: 03-07     │
├──────────────────────────────────────────────────────────────┤
│ 불량 수량 [ 2 ]   불량 유형 [ 체결 불량 v ]                   │
│ 발생 공정 [ 조립 v ]                                          │
│ 메모      [ 볼트 체결 불량 확인 ]                             │
└──────────────────────────────────────────────────────────────┘
```

### 불량 목록 화면 예시

```text
불량 기록

┌──────────────────────────────────────────────────────────────┐
│ 기간 [오늘]  유형 [전체]  품목 [전체]  담당 [전체]           │
├──────────────────────────────────────────────────────────────┤
│ 일시       품목명        유형         수량   연결 작업        │
│ 09:50      본체 조립     체결 불량      2    WO-240301        │
│ 10:30      커버 가공     치수 불량      1    WO-240302        │
└──────────────────────────────────────────────────────────────┘
```

## 7. 플로우차트

```text
생산실적 입력 중 불량 수량 발생
            ↓
       불량 기록 진입
            ↓
   유형 / 수량 / 공정 / 메모 입력
            ↓
         저장 및 연결
            ↓
품질 검토 필요?
 ┌──┴───────┐
 ↓          ↓
예          아니오
 ↓          ↓
품질관리로 전달   기록만 유지
```

## 8. 기능 기획

- 불량 유형 선택형 입력
- 작업지시/생산실적과 연결
- 메모 기록
- 기간/유형별 조회
- 반복 불량 탐지용 기본 집계

## 9. 정책과 예외

- MVP에서는 사진 첨부 없이 시작 가능
- 자유 텍스트만 허용하지 않고 유형 코드를 기본값으로 둔다
- 품질 담당자가 유형 코드를 정정할 수 있다

## 10. 데이터 연결

- 작업지시
- 생산실적
- 품목 / 도면 / BOM
- 향후 품질관리 조치 기록

## 11. MVP 범위

- 불량 수량
- 불량 유형 코드
- 메모
- 기본 조회

## 12. 후속 확장

- 사진 첨부
- 원인 분류
- 재작업/폐기 분기
- ECO 영향 분석 연결
