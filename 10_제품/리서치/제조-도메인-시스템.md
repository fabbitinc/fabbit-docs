# 제조 도메인 시스템

제조업 IT 스택의 핵심 시스템(PLC, SCADA, MES, PLM, ERP, SCM)의
역할, 기능, 데이터 흐름, 계층 구조를 정리한다.
Fabbit 제품 확장 방향의 도메인 기초 자료로 활용한다.

## 1. ISA-95 계층 구조

ISA-95는 제조업 IT/OT 시스템 간 통합을 정의하는 국제 표준이다.
각 시스템이 어느 계층에 위치하는지 파악하면 역할과 데이터 흐름이
명확해진다.

```
  [ISA-95 생산 운영 계층]                    [Engineering Domain]

  Level 4  ─  ERP / SCM       경영 계획      PLM / PDM + CAD/CAM
  Level 3  ─  MOM (MES)       운영 관리       ↕ (ISA-95에 속하지 않음)
  Level 2  ─  SCADA           감시 제어       설계 데이터·BOM·변경이력을
  Level 1  ─  PLC             직접 제어       관리하는 별도 도메인.
  Level 0  ─  센서/액추에이터  물리 장치       ERP·MOM과 연동한다.
```

> [!info] PLM은 ISA-95 밖에 위치한다
>
> ISA-95는 생산 운영 계층(Level 0~4)을 정의하며, PLM/PDM+CAD/CAM은
> Engineering Domain으로 별도 분류된다. PLM은 ERP(품목·BOM·ECO)와
> MOM(도면·작업표준·spec)에 데이터를 공급하고, MOM으로부터
> as-built·불량/품질·공정 피드백을 받는 양방향 연동 관계이다.

> [!info] OT vs IT
>
> - Level 0~2는 OT(Operational Technology) 영역으로, 현장 설비와
>   직접 연결된다.
> - Level 3~4는 IT 영역으로, 데이터 기반의 관리·계획 시스템이다.
> - MOM(MES)은 OT와 IT의 경계에서 양쪽을 연결하는 허브 역할을 한다.
> - PLM은 Engineering Domain으로 ISA-95 계층 밖에 위치하며,
>   ERP·MOM과 데이터를 주고받는다.

## 2. 시스템별 역할과 기능

### PLC (Programmable Logic Controller)

산업 자동화 현장에서 기계나 공정을 제어하는 산업용 컴퓨터다.
기존 릴레이 제어반을 반도체 소자로 대체하여, 프로그램 변경만으로
동작 제어(논리, 타이머, 카운터 등)가 가능하다.

| 항목      | 내용                                         |
| --------- | -------------------------------------------- |
| 계층      | Level 1 (직접 제어)                          |
| 핵심 역할 | 센서 입력 → 논리 연산 → 액추에이터 출력      |
| 주요 기능 | 시퀀스 제어, 타이머/카운터, PID 제어, 인터록 |
| 데이터    | 센서값, 모터 상태, 밸브 개폐, 알람           |
| 주요 벤더 | Siemens, Mitsubishi, Rockwell, LS Electric   |

### SCADA (Supervisory Control And Data Acquisition)

PLC에서 올라오는 데이터를 수집·시각화하고, 원격으로 설비를
감시·제어하는 시스템이다.

| 항목         | 내용                                                     |
| ------------ | -------------------------------------------------------- |
| 계층         | Level 2 (감시 제어)                                      |
| 핵심 역할    | 다수 PLC 데이터 통합 모니터링, 원격 제어                 |
| 주요 기능    | 실시간 대시보드, 알람 관리, 트렌드 분석, 데이터 로깅     |
| 데이터       | PLC 집계 데이터, 알람 이력, 운전 파라미터                |
| PLC와의 관계 | PLC가 개별 기계를 제어하면, SCADA는 공장 전체를 감시한다 |

### MES (Manufacturing Execution System) / MOM

PLC/SCADA(현장)와 ERP(경영) 사이에서 실시간 생산 현황을
관리·조율하는 시스템이다. ISA-95 원문에서는 MOM(Manufacturing
Operations Management)이 상위 개념이며, MES는 MOM의 핵심 모듈이다.
MOM = MES + QMS + EAM + 자재/재고 + Historian.

| 항목      | 내용                                                  |
| --------- | ----------------------------------------------------- |
| 계층      | Level 3 (운영 관리)                                   |
| 핵심 역할 | 생산 계획 → 현장 실행 → 실적 수집                     |
| 주요 기능 | 작업지시, 생산실적 추적, 품질관리, 설비관리, 자재추적 |
| 데이터    | 작업지시서, 생산수량, 불량률, 가동률, 로트 이력       |

MES 핵심 기능을 세분화하면 다음과 같다.

| 기능     | 설명                                      | 데이터 모델                                      |
| -------- | ----------------------------------------- | ------------------------------------------------ |
| 작업지시 | ERP 생산계획을 현장 작업 순서로 변환·배분 | `work_order` (BOM 참조, 수량, 납기, 우선순위)    |
| 생산실적 | 양품/불량 수량, 작업시간, 작업자 기록     | `production_log` (작업지시 참조, 양품수, 불량수) |
| 품질관리 | 검사 기준 대비 합격/불합격, 불량유형 분류 | `quality_inspection` (검사 기준, 불량유형 코드)  |
| 설비관리 | 기계별 가동/비가동/고장 상태 추적         | `equipment_log` (가동상태, 고장이력)             |
| 자재추적 | 원자재 입고 → 투입 → 완제품 로트 추적     | `material_lot`, `material_usage`                 |

### PLM (Product Lifecycle Management)

제품의 기획부터 설계, 생산, 단종까지 전체 수명 주기의
데이터를 관리하는 시스템이다.

| 항목      | 내용                                                           |
| --------- | -------------------------------------------------------------- |
| 계층      | **Engineering Domain** (ISA-95 밖, ERP·MOM과 연동)             |
| 핵심 역할 | 설계 데이터·BOM·변경이력 체계적 관리                           |
| 주요 기능 | 도면 관리, BOM 관리, 버전/리비전, 변경관리(ECR/ECO), 부품 분류 |
| 데이터    | CAD 도면, BOM 트리, 변경요청, 부품 사양, 공급사 정보           |

> [!info] Fabbit의 위치
> Fabbit은 PLM 시스템으로, 도면 AI 인식과 구조화 BOM 관리를
> 핵심 차별점으로 한다. 상세 기능은
> [[10_제품/전략/제품-로드맵]]을 참고한다.

### ERP (Enterprise Resource Planning)

기업의 구매, 생산, 재고, 회계, 인사 등 경영 자원을 통합
관리하는 시스템이다.

| 항목      | 내용                                                     |
| --------- | -------------------------------------------------------- |
| 계층      | Level 4 (경영 계획)                                      |
| 핵심 역할 | 기업 전체 자원의 계획·실행·통제                          |
| 주요 기능 | 구매/조달, 생산계획(MRP), 재고관리, 회계/원가, 인사/급여 |
| 데이터    | 발주서, 생산계획, 재고 수불, 매출/매입, 원가             |
| 주요 벤더 | SAP, Oracle, 더존, 영림원                                |

### SCM (Supply Chain Management)

원자재 조달부터 생산, 유통, 최종 고객 납품까지 공급망 전체를
최적화하는 시스템이다.

| 항목         | 내용                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| 계층         | Level 4 (경영 계획)                                                         |
| 핵심 역할    | 공급망 가시성 확보, 조달·물류 최적화                                        |
| 주요 기능    | 수요 예측, 조달 관리, 물류 추적, 협력사 관리                                |
| 데이터       | 수요 예측, 발주/납품 이력, 운송 추적, 협력사 평가                           |
| ERP와의 관계 | ERP의 구매/물류 모듈을 확장한 형태. 소기업에서는 ERP에 포함되는 경우가 많다 |

## 3. 시스템 간 데이터 흐름

```
  [Engineering Domain]                          [ISA-95 생산 운영 계층]

  ┌───────────────────┐
  │      CAD/CAM      │
  │     설계 도구       │
  └────────┬──────────┘
           │ 도면/3D
           ▼
  ┌───────────────────┐   품목,BOM    Level 4   ┌─────────────────┐  ┌───────┐
  │     PLM / PDM     │──ECO,릴리스──────────►  │       ERP       │◄►│  SCM  │
  │                   │              │         │ 구매/생산계획      │  └───────┘
  │  도면/BOM/변경관리    │   도면,spec   │        │ 재고/회계         │
  │                   │───작업표준────┐│         └────────┬────────┘
  └───────────────────┘  recipe     ││   생산오더       │ ▲
           ▲                        ▼│   자재계획       ▼ │ 생산실적
           │              Level 3 ┌──┴──────────────────┴───────┐
           │                      │        MOM (MES)            │
           │                      │ 작업지시/생산실적/품질            │
           │ as-built             │ 설비보전/자재·재고               │
           │ 불량/품질결과           └──────────────┬───────────────┘
           │ 공정 피드백                         │ ▲
           │ 변경요청              Level 2       ▼ │
           │                      ┌─────────────────────────────┐
           └──────────────────────┤       SCADA / HMI           │
                                  └──────────────┬──────────────┘
                                                 │ ▲
                                  Level 1        ▼ │
                                  ┌─────────────────────────────┐
                                  │     PLC / RTU / Controller  │
                                  └──────────────┬──────────────┘
                                                 │ ▲
                                  Level 0        ▼ │
                                  ┌─────────────────────────────┐
                                  │    센서 / 액추에이터            │
                                  └─────────────────────────────┘
```

주요 연동 포인트를 정리하면 다음과 같다.

| 연동            | 방향   | 전달 데이터                                      |
| --------------- | ------ | ------------------------------------------------ |
| PLM → ERP       | 단방향 | 품목, 확정 BOM, ECO 릴리스                        |
| PLM → MOM       | 단방향 | 도면, 작업표준, spec, recipe                      |
| MOM → PLM       | 단방향 | as-built, 불량/품질 결과, 공정 피드백, 변경요청  |
| ERP → MOM       | 양방향 | 생산계획/작업지시(하향), 생산실적/자재소모(상향)  |
| MOM → SCADA/PLC | 양방향 | 레시피/파라미터(하향), 설비 데이터/실적(상향)     |
| ERP ↔ SCM       | 양방향 | 발주/납품, 수요 예측, 협력사 정보                 |

## 4. 도입 우선순위 (소기업 관점)

소기업은 예산과 인력이 제한적이므로, 당장 운영에 필수적인
시스템부터 도입한다.

```
1순위: PLC / 설비     ← 이것 없이는 생산 자체가 불가능
2순위: ERP            ← 자재·재고·회계 관리, 경영 필수
3순위: MES            ← 생산 현황 파악, 품질·불량 추적
4순위: PLM            ← 설계 데이터·이력 체계화
5순위: SCM            ← 공급망 복잡도가 높아진 이후
```

> [!warning] PLM 도입 현실
> 소기업은 PLM 없이도 Excel·파일서버로 도면/BOM을 관리한다.
> PLM의 가치는 제품 종류가 많아지고, 설계 변경이 잦아지고,
> 이력 추적이 꼬이기 시작할 때 체감된다.
> "PLM을 도입하세요"보다 "도면 버전 관리 엉망이죠?"가
> 더 효과적인 진입 메시지다.

## 5. Fabbit 확장 방향

현재 Fabbit은 PLM 영역에 집중하고 있다. 소기업 타겟에서
PLM 단독의 우선순위가 낮은 문제를 해결하려면, MES 경량 기능을
결합하여 진입 가치를 높이는 전략이 유효하다.

### PLM + MES 경량 통합 시나리오

```
PLM (현재)                    MES (추가)
─────────────────────────    ─────────────────────────
도면 관리                     작업지시 (BOM → 작업지시서)
BOM 관리                      생산실적 (양품/불량 수량)
변경관리 (ECR/ECO)             품질추적 (불량 → 부품 역추적)
부품/공급사                    자재추적 (로트 이력)
```

### MES 기능 추가 우선순위

상세 단계 기획은 [[10_제품/기획/mes/index]] 참고.

| 순위  | 기능                  | 이유                                                   |
| ----- | --------------------- | ------------------------------------------------------ |
| 1순위 | 작업지시              | 현장 실행의 시작점. 기준 작업 없이는 실적도 쌓이지 않음 |
| 2순위 | 생산실적              | 일일 생산 가시성 확보의 핵심                           |
| 3순위 | 불량기록 / 기본 품질  | 최소 품질 피드백 루프 형성, PLM 데이터와 연결 가능      |
| 4순위 | 품질관리 워크플로우   | 검사 기준, 판정, 재작업/폐기 처리까지 확장             |
| 5순위 | 자재추적              | 로트 역추적 가치가 크지만 운영 복잡도 상승              |
| 6순위 | 설비관리              | PLC/SCADA 연동 필요하여 난이도 높음                    |

### 포지셔닝 메시지

소기업 대상 포지셔닝은 시스템 이름이 아닌 문제 해결로 접근한다.

- "도면·BOM 관리 + 생산·품질 현황을 한 곳에서"
- "설계부터 생산까지, 엑셀 없이 한 번에"
