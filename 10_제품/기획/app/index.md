# 앱 온보딩·구독 기획

이 문서는 Fabbit 앱 내부의 온보딩과 구독 흐름을 다루는 상위 진입 문서다.
공개 사이트 이후 사용자가 실제로 앱에 진입해 워크스페이스를 만들고, 플랜을
선택하고, 구독을 변경하는 흐름을 이 문서군에서 관리한다.

## 1. 문서군의 역할

이 문서군은 공개 리드 수집이 끝난 뒤 앱 안에서 발생하는 제품 흐름을 정리한다.
즉, 방문자를 리드로 바꾸는 `site` 문서군 다음 단계에서 어떤 계정과 구독
경험을 제공할지를 정의한다.

- 회원가입과 이메일 인증 흐름
- 워크스페이스 생성과 기본 설정 흐름
- 가입 시 플랜 선택 경험
- Starter에서 유료 플랜으로의 업그레이드 흐름
- 구독 상태, 좌석, 사용량, 설정 같은 앱 내부 구독 관리 흐름

## 2. 문서군이 맡지 않는 범위

이 문서군은 앱 내부 경험을 다루지만, 정책 source of truth나 공개 사이트
메시지까지 흡수하지는 않는다. 아래 항목은 참조 대상이지 이 문서군의 직접
소유 범위가 아니다.

- 공개 랜딩 페이지와 공개 요금제 페이지 메시지
- 파일럿/데모 신청 폼과 공개 CTA 설계
- 제품 정의와 고객 세그먼트 정책 원문
- 서버 API 구현 상세와 청구 정산 백엔드 규칙

공개 경험은 [[10_제품/기획/site/index]]에서 관리한다.

## 4. 경계 원칙

`site`와 `app` 문서군은 서로 연결되지만, 사용자 시점의 경계가 분명해야 한다.
아래 원칙을 기준으로 문서 위치를 판단한다.

- 방문자가 공개 URL에서 보는 내용이면 `site`에 둔다.
- 계정 생성 이후 워크스페이스 안에서 처리되는 흐름이면 `app`에 둔다.
- 공개 가능한 가격 설명은 `site`에 두고, 실제 플랜 변경 처리 경험은 `app`에 둔다.
- 리드 분류와 운영 후속 조치는 제품 기획이 아니라 운영 정책 쪽으로 분리한다.
