# 가입·워크스페이스 생성·플랜 선택·업그레이드 기획

이 문서는 Fabbit 앱 내부에서 가입 이후 발생하는 온보딩과 구독 흐름의 첫
골격 문서다. 목적은 공개 사이트 문서군과 경계를 분리한 상태에서, 앱 안의
계정 생성과 구독 변경 흐름이 어디서부터 어디까지 이어지는지 상위 구조를 먼저
고정하는 것이다.

## 1. 문서 역할

이 문서는 세부 화면 명세를 모두 쓰기 전에, 앱 내부의 핵심 흐름을 하나의
문서에서 연결해 보는 기준선 역할을 맡는다. 이후 상세 문서를 쪼개더라도 이
문서의 범위와 용어를 벗어나지 않게 한다.

- 이메일 인증 기반 회원가입
- 워크스페이스 이름과 slug 설정
- 가입 시 기본 플랜 선택
- Starter 이후 업그레이드 진입점
- 좌석과 구독 관리의 후속 분리 기준

## 2. 범위

현재 골격 문서는 앱 내부의 주요 단계만 정의한다. 상세 필드, 에러 처리, 세부
정책은 후속 문서에서 보강한다.

1. 가입과 이메일 인증
2. 워크스페이스 생성과 기본 정보 입력
3. 무료/유료 공통 플랜 선택
4. Starter 선택 시 결제 스킵 후 사용 시작
5. 유료 플랜 선택 시 결제 필요 상태와 현재 미구현 처리
6. 업그레이드 또는 구독 변경 진입

## 3. 범위 밖

이 문서는 앱 내부 제품 흐름만 다루므로, 공개 사이트와 운영 정책 문맥은 직접
다루지 않는다.

- 랜딩 페이지 메시지와 공개 CTA
- 공개 사이트의 가입 CTA와 파일럿 신청 폼 설계
- 가격 정책의 source of truth
- 서버 API와 결제/정산 구현 상세

## 4. 상위 사용자 흐름

현재 기준의 상위 흐름은 아래처럼 잡는다. 이 구조를 기준으로 후속 화면 기획과
정책 연결을 정리한다.

```text
공개 사이트의 가입 CTA 또는 직접 진입
                 ↓
         회원가입 / 이메일 인증
                 ↓
           워크스페이스 생성
                 ↓
               플랜 선택
        ┌────────┴────────┐
        ↓                 ↓
   Starter 선택       유료 플랜 선택
        ↓                 ↓
     결제 스킵           결제 필요
        ↓                 ↓
   즉시 사용 시작     현재 결제 기능 미구현
                          ↓
                 후속 구현 또는 대체 처리 필요
                          ↓
                     앱 내부 사용
                          ↓
              업그레이드 / 좌석 변경 / 구독 관리
```

이 흐름에서 중요한 해석 기준은 아래와 같다.

- 가입 후 무료와 유료 모두 워크스페이스 생성과 플랜 선택을 거친다.
- Starter만 결제를 건너뛰고 바로 사용을 시작한다.
- 유료 플랜은 결제가 필요하지만, 현재 결제 기능은 아직 구현되지 않았다.
- 공개 사이트의 파일럿 신청은 별도 보조 문의 경로이며, 위 앱 온보딩 흐름에
  직접 포함하지 않는다.

## 5. 후속 작성 항목

이 문서는 구조만 세웠으므로, 다음 라운드에서 아래 항목을 순서대로 보강해야
한다.

| 항목 | 후속 문서 방향 |
|------|----------------|
| 가입 단계 상세 | 인증, 초대 수락, 진입 분기 정리 |
| 워크스페이스 생성 | 조직 정보, slug, 역할 입력 흐름 정리 |
| 플랜 선택 | 무료/유료 공통 선택 흐름과 planType, ownerSeatType 의미 정리 |
| 결제 단계 | 유료 플랜 결제 필요 조건과 현재 미구현 상태의 처리 방식 정리 |
| 업그레이드 | Starter에서 Team·Organization으로 가는 진입점과 단계 정리 |
| 좌석/구독 관리 | 설정 화면, 좌석 배정, 사용량 연결 흐름 정리 |

## 6. 참조 원칙

세부 기획을 작성할 때는 정책과 공개 문서를 각각 분리해 참조한다. 가격과
도입 정책은 정책 문서를 참조하고, 공개 메시지와 신청 경험은 `site` 문서군을
참조한다.
