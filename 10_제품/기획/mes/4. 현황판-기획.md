# 현황판 기획

이 문서는 관리자 관점의 MES 현황판을 정의한다. 목표는 여러 문서를 열지 않고도
오늘의 생산 상태, 지연, 불량을 한 화면에서 파악하는 것이다.

## 1. 문제와 목표

- 대표와 생산관리자는 "지금 어디가 밀리고 있는지"를 즉시 보고 싶다.
- 현장 데이터가 흩어져 있으면 빠른 대응이 어렵다.
- 초기 MES는 분석보다 운영 가시성이 먼저 필요하다.

## 2. 주요 사용자

- 대표
- 공장 책임자
- 생산관리자

## 3. 메뉴명과 진입 경로

현황판은 기능명이 아니라 "지금 상태를 본다"는 목적이 드러나야 한다.

| 구분 | 권장 명칭 | 비고 |
|------|-----------|------|
| 상위 메뉴 | `생산` | 공통 상위 메뉴 |
| 하위 메뉴 | `생산현황` | 대시보드보다 업무 언어에 가까움 |
| 화면 제목 | `생산현황` | 목록과 KPI 화면 공통 제목 |
| 기본 CTA | `작업 상세 보기` | Drill-down 중심 |

진입 경로 예시는 아래와 같다.

`생산 > 생산현황`

## 4. 핵심 시나리오

1. 사용자가 아침에 현황판을 열어 오늘 작업 현황을 본다.
2. 진행 중, 지연, 완료, 불량 발생 작업을 확인한다.
3. 상세 작업지시나 불량 목록으로 바로 이동한다.

## 5. 화면 기획

### 상단 KPI 카드

- 오늘 예정 작업 수
- 진행 중 작업 수
- 완료 작업 수
- 지연 작업 수
- 불량 발생 작업 수

### 중단 주요 위젯

- 오늘 작업지시 리스트
- 납기 임박 / 지연 작업
- 품목별 생산량 요약
- 최근 불량 유형 요약

### 하단 Drill-down 영역

- 작업 상세 이동
- 기간 변경
- 팀/라인 필터

## 6. 화면 예시

```text
[생산] / 생산현황

┌──────────────────────────────────────────────────────────────┐
│ 오늘 생산현황                                               │
├──────────────────────────────────────────────────────────────┤
│ 예정 12   진행 중 5   완료 6   지연 2   불량 발생 3         │
├──────────────────────────────────────────────────────────────┤
│ 오늘 작업지시               │ 최근 불량                      │
│ - WO-240301 진행 중         │ - 체결 불량 2건               │
│ - WO-240302 예정            │ - 치수 불량 1건               │
│ - WO-240303 지연            │ - 스크래치 1건                │
├──────────────────────────────────────────────────────────────┤
│ 품목별 생산량 요약 / 팀 필터 / 상세 보기                    │
└──────────────────────────────────────────────────────────────┘
```

## 7. 플로우차트

```text
생산현황 진입
    ↓
기간 / 팀 필터 선택
    ↓
KPI 및 작업 목록 확인
    ↓
이상 징후 발견?
 ┌──┴───────┐
 ↓          ↓
예          아니오
 ↓          ↓
작업 상세/불량 목록 이동   모니터링 유지
```

## 8. 기능 기획

- 오늘 기준 기본 현황 집계
- 상태별 필터
- 지연 작업 강조
- 품목/팀 단위 Drill-down
- 관련 상세 화면 이동

## 9. 정책과 예외

- MVP는 실시간 스트리밍보다 새로고침/주기적 갱신으로 시작 가능
- 데이터 미입력 작업은 별도 경고로 표시
- 관리자 권한 기준으로 조직 범위 제한

## 10. 데이터 연결

- 작업지시
- 생산실적
- 불량기록
- 향후 설비/자재 데이터

## 11. MVP 범위

- KPI 카드
- 오늘 작업 리스트
- 지연 작업 강조
- 불량 요약

## 12. 후속 확장

- 실시간 보드
- 주간/월간 추세
- 설비, 자재, 품질 통합 현황
- 알림 센터 연동
