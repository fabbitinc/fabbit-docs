# 작업지시 기획

이 문서는 MES MVP의 시작점인 작업지시 기능을 정의한다. 핵심 목적은
Released BOM과 도면 기준본을 현장 작업 단위로 내려 보내는 것이다.

## 1. 문제와 목표

- 생산관리자는 오늘 무엇을 몇 개 만들어야 하는지 한눈에 보고 싶다.
- 현장은 엑셀, 메신저, 구두 전달 없이 기준본이 고정된 작업을 받아야 한다.
- 설계가 바뀌었을 때 어떤 작업이 영향을 받는지 확인 가능해야 한다.

## 2. 주요 사용자

- 생산관리자
- 현장 리더
- 작업자

## 3. 메뉴명과 진입 경로

작업지시는 내부 용어로는 정확하지만, 메뉴에서는 조금 더 짧고 익숙한 단어가
낫다.

| 구분 | 권장 명칭 | 비고 |
|------|-----------|------|
| 상위 메뉴 | `생산` | `MES` 대신 사용자 언어 사용 |
| 하위 메뉴 | `작업` | 메뉴에서는 짧게 |
| 화면 제목 | `작업지시` | 상세 화면과 등록 화면에서 사용 |
| 기본 CTA | `새 작업 등록` | `작업지시 생성`보다 부드러움 |

진입 경로 예시는 아래와 같다.

`생산 > 작업 > 작업지시 목록 > 새 작업 등록`

## 4. 핵심 시나리오

1. 생산관리자가 Released BOM을 기준으로 작업지시를 생성한다.
2. 수량, 납기, 우선순위, 담당 라인/팀을 지정한다.
3. 현장 리더가 작업지시를 확인하고 진행 상태를 변경한다.
4. 작업 완료 후 생산실적 문서와 연결된다.

## 5. 화면 기획

### 목록 화면

- 상태별 탭
  - 전체, 예정, 진행 중, 완료, 지연, 취소
- 주요 컬럼
  - 작업번호, 품목명, 수량, 납기, 우선순위, 담당, 상태
- 빠른 액션
  - 생성, 상태 변경, 담당 변경, 우선순위 조정

### 상세 화면

- 기본 정보 영역
  - 작업번호, 상태, 납기, 수량, 담당
- 기준본 영역
  - 참조 BOM, 참조 도면, 리비전
- 진행 정보 영역
  - 시작 시각, 종료 시각, 누적 실적
- 연결 정보 영역
  - 생산실적, 불량기록, 변경 영향

### 생성/수정 화면

- 입력 항목
  - 품목, BOM 기준본, 수량, 납기, 우선순위, 담당 조직
- 검증 항목
  - 기준본 미선택 금지
  - 납기 과거 날짜 제한
  - 진행 중 작업의 기준본 직접 변경 제한

## 6. 화면 예시

### 목록 화면 예시

```text
[생산] / 작업

┌──────────────────────────────────────────────────────────────┐
│ 작업지시                                   + 새 작업 등록   │
├──────────────────────────────────────────────────────────────┤
│ 검색 [작업번호/품목명]  상태 [전체]  담당 [전체]  납기 [주간]│
├──────────────────────────────────────────────────────────────┤
│ 작업번호   품목명        수량   납기      담당     상태      │
│ WO-240301  본체 조립     100    03-10     1팀      진행 중   │
│ WO-240302  커버 가공      50    03-11     2팀      예정      │
│ WO-240303  최종 검사      30    03-09     품질팀   지연      │
├──────────────────────────────────────────────────────────────┤
│ 선택 작업 상세: 기준본 / 진행 현황 / 실적 / 불량 / 이력     │
└──────────────────────────────────────────────────────────────┘
```

### 상세 화면 예시

```text
작업지시 > WO-240301

┌────────────────────────┬─────────────────────────────────────┐
│ 기본 정보              │ 기준본 정보                         │
│ - 상태: 진행 중        │ - BOM: BOM-AX-003                  │
│ - 수량: 100            │ - 도면: DRW-AX-021 Rev.C           │
│ - 납기: 03-10          │ - 릴리스 일시: 03-07 09:00         │
├────────────────────────┼─────────────────────────────────────┤
│ 진행 정보              │ 연결 정보                           │
│ - 시작: 09:20          │ - 생산실적 2건                     │
│ - 종료: -              │ - 불량기록 1건                     │
│ - 누적 양품: 40        │ - ECO 영향: 없음                   │
└────────────────────────┴─────────────────────────────────────┘
```

## 7. 플로우차트

### 기본 작업지시 사이클

```text
Released BOM / 도면 리비전 확정
              ↓
      작업지시 초안 생성
              ↓
      담당/수량/납기 지정
              ↓
         released 배포
          ┌────┴────┐
          ↓         ↓
       취소        착수
          ↓         ↓
     cancelled   in_progress
                    ↓
               생산실적 입력
                    ↓
                   done
```

### 변경 영향 예외 흐름

```text
작업지시 released / in_progress
              ↓
        ECO 영향 발생 확인
          ┌────┴─────┐
          ↓          ↓
       미착수       진행 중
          ↓          ↓
 새 기준본으로 재발행   유지 / 중단 / 재지시 판단
```

## 8. 기능 기획

- 작업지시 생성, 수정, 취소
- 상태 전이
  - `draft`, `released`, `in_progress`, `done`, `cancelled`
- 우선순위와 납기 기준 정렬
- 작업지시 복제
- 작업지시별 참조 기준본 고정

## 9. 정책과 예외

- 작업이 `in_progress`가 되면 기준본 직접 교체 대신 새 작업지시 생성 검토
- `done` 상태 작업은 기본 수정 불가, 관리자만 예외 수정
- 취소된 작업도 이력은 남긴다

## 10. 데이터 연결

- BOM / Part
- 도면 리비전
- 조직 / 팀 / 멤버
- 생산실적
- 불량기록

## 11. MVP 범위

- 단일 작업지시 생성
- 상태 관리
- 담당 배정
- 기준본 연결
- 생산실적 연결

## 12. 후속 확장

- 작업지시 일괄 생성
- 반복 작업 템플릿
- ECO 영향 자동 표시
- QR/바코드 호출
