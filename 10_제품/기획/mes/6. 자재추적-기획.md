# 자재추적 기획

이 문서는 정식 단계에서 도입할 자재·로트 추적 기능을 정의한다. 목표는
문제 발생 시 어떤 자재가 어떤 작업과 완제품에 연결됐는지 역추적하는 것이다.

## 1. 문제와 목표

- 자재 이슈가 생기면 영향 범위를 빠르게 알기 어렵다.
- 생산실적만으로는 어떤 자재가 투입됐는지 설명이 안 된다.
- 회수, 재작업, 공급사 대응의 근거 데이터가 필요하다.

## 2. 주요 사용자

- 생산관리자
- 품질 담당자
- 구매 담당자

## 3. 메뉴명과 진입 경로

자재추적은 시스템 용어보다 "이력"과 "추적"의 맥락이 잘 보이게 해야 한다.

| 구분 | 권장 명칭 | 비고 |
|------|-----------|------|
| 상위 메뉴 | `생산` | 공통 상위 메뉴 |
| 하위 메뉴 | `자재이력` | 사용자 친화적 메뉴명 |
| 화면 제목 | `자재추적` | 기능 제목은 더 구체적으로 |
| 기본 CTA | `자재 연결` | 작업 단위 투입 등록 의미 |

진입 경로 예시는 아래와 같다.

`생산 > 자재이력 > 자재추적`

## 4. 핵심 시나리오

1. 특정 작업지시에 자재 또는 로트를 연결한다.
2. 생산 완료 후 완제품과 사용 자재 관계를 남긴다.
3. 자재 이슈 발생 시 영향 작업과 완제품을 조회한다.

## 5. 화면 기획

### 자재 투입 등록 화면

- 작업번호
- 자재 품목
- 로트번호
- 사용 수량

### 역추적 조회 화면

- 자재 로트 기준 조회
- 관련 작업지시, 생산일, 완제품 목록
- 공급사 정보 표시

## 6. 화면 예시

```text
[생산] / 자재이력

┌──────────────────────────────────────────────────────────────┐
│ 자재추적                                   + 자재 연결       │
├──────────────────────────────────────────────────────────────┤
│ 로트번호 [검색]  자재 품목 [전체]  기간 [최근 30일]          │
├──────────────────────────────────────────────────────────────┤
│ 로트번호   자재명      연결 작업      생산일      완제품 수    │
│ LOT-001    볼트 M4     WO-240301      03-07       40          │
│ LOT-002    커버판      WO-240302      03-07       50          │
└──────────────────────────────────────────────────────────────┘
```

## 7. 플로우차트

```text
작업지시 선택
    ↓
사용 자재 / 로트 연결
    ↓
생산 완료
    ↓
완제품과 사용 자재 관계 저장
    ↓
이슈 발생 시 로트 기준 역추적 조회
```

## 8. 기능 기획

- 자재 로트 연결
- 작업 단위 사용량 기록
- 자재 기준 역추적
- 완제품 기준 투입 자재 조회

## 9. 정책과 예외

- MVP 이전에는 도입하지 않는다
- 고객 ERP와 중복 입력 범위를 최소화해야 한다
- 로트 없는 운영 고객을 위한 예외 흐름 필요

## 10. 데이터 연결

- 작업지시
- 생산실적
- 품목 / 공급사
- 향후 ERP 재고 데이터

## 11. 정식 범위

- 자재 로트 등록
- 작업 투입 연결
- 기본 역추적 조회

## 12. 후속 확장

- 자동 재고 차감
- 회수 범위 계산
- 공급사 클레임 처리
