# MES 기획

이 문서는 Fabbit MES 도메인의 전체 기획 문서다. 경량 MES의 목적, 범위,
우선순위, 단계별 확장 계획과 기능별 상세 문서 구성을 함께 관리한다.

## 1. MES의 역할

Fabbit의 MES는 전문 MES의 축소판이 아니라, PLM 기준본을 현장 실행으로
내려 보내고 그 결과를 다시 설계 데이터에 연결하는 실행 관리 레이어다.

- **입력**: Released BOM, 도면 리비전, 부품/공급사 정보
- **실행**: 작업지시, 생산실적, 불량기록
- **피드백**: 품질, 자재, 추적성, 설계 영향

## 2. 타겟 사용자

| 사용자 | 주요 관심사 |
|--------|-------------|
| 대표 / 공장 책임자 | 오늘 생산량, 지연, 불량 현황 |
| 생산관리자 | 작업 배정, 납기, 진행 상태 |
| 현장 리더 / 작업자 | 오늘 해야 할 작업과 실적 입력 |
| 품질 담당자 | 불량 유형, 검사 결과, 재작업 여부 |

## 3. 범위 원칙

MES 기획은 아래 원칙을 따른다.

- 초기에는 **현장 입력 부담이 낮아야** 한다.
- **작업지시 → 생산실적 → 불량기록**을 먼저 닫는다.
- 품질, 자재, 설비는 데이터 축적 후 확장한다.
- ERP와 PLC/SCADA는 초기에 직접 대체하지 않는다.

> [!warning] 초기 제외 범위
>
> - 고급 생산계획 최적화
> - 설비 자동 수집
> - 다공장 운영
> - WMS 전체 대체
> - APS/MOM 전체 범위

## 4. 전체 우선순위

| 순위 | 기능 | 단계 | 이유 |
|------|------|------|------|
| 1 | 작업지시 | MVP | 현장 실행의 시작점 |
| 2 | 생산실적 | MVP | 생산 가시성 확보 |
| 3 | 불량기록 | MVP | 최소 품질 피드백 루프 |
| 4 | 현황판 | MVP | 관리자 의사결정 지원 |
| 5 | 품질관리 | 베타 | 검사/판정/조치 운영화 |
| 6 | 자재추적 | 정식 | 로트 역추적과 회수 범위 파악 |
| 7 | 공정-라우팅 | 확장 | 복수 공정 흐름 제어 |
| 8 | 설비관리 | 확장 | 가동/비가동과 생산 영향 분석 |

## 5. 단계별 계획

### Phase 0. 기준본 준비

- Released BOM 고정
- 작업지시 상태값 정의
- 생산 단위(라인/팀/담당자) 정의

### Phase 1. MES MVP

- 작업지시
- 생산실적
- 불량기록
- 현황판

### Phase 2. 베타

- 품질관리
- 설계 변경 영향 표시
- 모바일/현장 UX 개선

### Phase 3. 정식

- 자재·로트 추적
- ERP 연동
- 감사 로그 강화

### Phase 4. 확장

- 공정-라우팅
- 설비관리
- OT 연동

## 8. 사용자 노출 메뉴명

MES라는 용어는 내부 기획과 도메인 문서에서는 유효하지만, 사용자 메뉴명으로는
낯설 수 있다. 사용자 노출 명칭은 아래처럼 업무 언어로 푼다.

| 층위 | 후보 | 판단 |
|------|------|------|
| 상위 메뉴 | `생산`, `작업`, `제조운영`, `MES` | **`생산` 권장** |
| 작업 화면 | `작업`, `작업관리`, `작업지시` | 메뉴는 `작업`, 화면 제목은 `작업지시` 권장 |
| 실적 화면 | `실적`, `생산실적`, `실적관리` | 메뉴는 `실적` 권장 |
| 품질 화면 | `품질`, `불량/품질`, `검사` | MVP는 `품질` 권장 |
| 현황판 | `생산현황`, `현황`, `대시보드` | 메뉴는 `생산현황` 권장 |

권장 IA는 아래와 같다.

```text
생산
├─ 작업
├─ 실적
├─ 품질
└─ 생산현황
```

> [!info] 메뉴명 원칙
>
> - 상위 메뉴에서는 `MES`를 직접 쓰지 않는다.
> - 사용자는 "시스템 종류"보다 "내가 하는 업무"를 먼저 이해한다.
> - 상세 화면 제목에서는 `작업지시`, `생산실적`, `불량 기록`처럼 더 구체적인
>   명칭을 써도 된다.

## 9. 화면 기획 공통 규칙

MES 기능 문서는 화면 중심으로 정리한다.

- 목록 화면: 검색, 필터, 상태, 일괄 액션
- 상세 화면: 핵심 정보, 이력, 연결 데이터
- 생성/수정 화면: 필수 입력값, 검증, 저장 규칙
- 알림/예외: 지연, 충돌, 수정 제한, 잠금 조건

> [!info] 화면과 기능의 관계
>
> 기능 문서에는 "화면이 무엇을 보여주는지"와 "그 화면에서 어떤 동작이
> 가능한지"를 함께 적는다. 이 문서군은 와이어프레임 전 단계의 기준 문서다.

## 10. 문서 산출물 기준

각 기능 문서에는 아래 산출물을 포함하는 것을 기본값으로 한다.

1. 메뉴명 / 진입 경로
2. 화면 예시 1개 이상
3. 상태 또는 사용자 흐름 플로우차트 1개 이상
4. 기능, 정책, 예외 규칙
