# 설비관리 기획

이 문서는 확장 단계의 설비관리 기능을 정의한다. 목표는 생산 차질의 원인이
작업, 품질, 설비 중 어디에 있는지 구분할 수 있게 하는 것이다.

## 1. 문제와 목표

- 생산 지연의 원인이 설비인지 작업 운영인지 구분하기 어렵다.
- 비가동 사유가 누적되지 않으면 개선 우선순위를 세우기 어렵다.
- 초기에는 수동 기록으로 시작하더라도 설비 기준 데이터가 필요하다.

## 2. 주요 사용자

- 생산관리자
- 설비 담당자
- 공장 책임자

## 3. 메뉴명과 진입 경로

설비관리도 전문 용어이긴 하지만, 메뉴에서는 그대로 사용해도 이해 가능성이
높다.

| 구분 | 권장 명칭 | 비고 |
|------|-----------|------|
| 상위 메뉴 | `생산` | 공통 상위 메뉴 |
| 하위 메뉴 | `설비` | 메뉴는 짧게 |
| 화면 제목 | `설비관리` | 기능 제목은 구체적으로 |
| 기본 CTA | `설비 등록` | 관리 동작 명확화 |

진입 경로 예시는 아래와 같다.

`생산 > 설비 > 설비관리`

## 4. 핵심 시나리오

1. 설비 담당자가 설비 상태를 기록한다.
2. 고장 또는 비가동 사유를 남긴다.
3. 특정 작업 지연과 설비 이슈를 연결해 본다.

## 5. 화면 기획

### 설비 목록 화면

- 설비명
- 상태
- 현재 사용 작업
- 최근 고장 일시

### 설비 상세 화면

- 설비 기본 정보
- 상태 이력
- 고장/비가동 이력
- 연결 작업지시

## 6. 화면 예시

```text
[생산] / 설비

┌──────────────────────────────────────────────────────────────┐
│ 설비관리                                   + 설비 등록       │
├──────────────────────────────────────────────────────────────┤
│ 설비명      상태        현재 작업        최근 고장           │
│ 프레스-01   가동 중     WO-240301        -                  │
│ 검사기-02   비가동      -                03-07 10:20        │
├──────────────────────────────────────────────────────────────┤
│ 상세: 상태 이력 / 비가동 사유 / 연결 작업지시               │
└──────────────────────────────────────────────────────────────┘
```

## 7. 플로우차트

```text
설비 등록
   ↓
상태 기록
   ↓
비가동 / 고장 발생?
 ┌──┴───────┐
 ↓          ↓
예          아니오
 ↓          ↓
사유 기록      상태 유지
 ↓
연결 작업 영향 확인
```

## 8. 기능 기획

- 설비 마스터 등록
- 상태값 기록
- 비가동 사유 기록
- 작업지시와 설비 연결

## 9. 정책과 예외

- 초기에는 PLC 연동 없이 수동 기록으로 시작
- 설비 없는 현장은 팀/라인 기준으로 대체 가능
- 가동률 계산은 확장 단계에서 검토

## 10. 데이터 연결

- 작업지시
- 공정-라우팅
- 향후 PLC/SCADA 연동 데이터

## 11. 확장 범위

- 설비 목록/상태 관리
- 비가동 사유 이력
- 작업 연계

## 12. 후속 확장

- 자동 데이터 수집
- OEE
- 예방보전
- 알람 통합
