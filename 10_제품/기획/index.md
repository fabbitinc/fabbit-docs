# 제품 기획

이 문서는 Fabbit 제품 기획 문서군의 상위 진입 문서다. 현재 구조에서 제품
기획은 `site`, `app`, `plm`, `mes` 네 문서군으로 나뉘며, 각 문서군은
정책 SSOT를 참조해 실제 화면과 사용자 흐름을 설계한다.

## 1. 문서 역할

이 문서는 제품 기획 문서군의 범위와 연결 방식을 고정한다. 전략과 정책이
`무엇을 왜 하는가`를 정한다면, 제품 기획은 `어떤 화면과 흐름으로 구현할
것인가`를 정리한다.

이 문서가 맡는 역할은 아래와 같다.

- 현재 제품 기획 문서군의 상위 구조 안내
- 문서군별 책임 범위와 경계 정의
- 전략/정책 문서와 상세 기획 문서의 연결 순서 정리
- 후속 기능 기획이 참조해야 할 상위 진입점 제공

이 문서가 직접 다루지 않는 범위는 아래와 같다.

- 제품 정의와 시장 포지셔닝의 최종 판단
- 고객 세그먼트와 도입 정책의 운영 기준
- 플랜, 좌석, 과금, 업그레이드 정책의 최종 기준
- 서버와 웹 구현 상세

## 2. 참조 우선순위

제품 기획 문서는 단독으로 기준을 만들지 않는다. 아래 순서로 상위 문서를
참조한 뒤 상세 기획을 작성한다.

1. `아이템-정의.md`에서 제품 정의와 상위 가치 제안을 확인한다.
2. `고객-세그먼트-도입-정책.md`에서 타깃 고객과 기본 도입 경로를 확인한다.
3. `플랜-과금-정책.md`에서 플랜, 좌석, 가입, 구독 변경 기준을 확인한다.
4. 그 다음에 `site`, `app`, `plm`, `mes` 상세 기획 문서로 화면과 흐름을 설계한다.

즉, 전략과 정책은 기준이고, 제품 기획은 그 기준을 실제 경험으로 번역하는
파생 문서군이다.

## 3. 현재 문서군 구조

현재 제품 기획 문서군은 아래 네 갈래로 운영한다.

| 문서군 | 역할 | 대표 진입 문서 |
|--------|------|----------------|
| `site` | 공개 랜딩, 공개 요금제, 공개 신청 경험 | [[10_제품/기획/site/index]] |
| `app` | 회원가입, 워크스페이스 생성, 플랜 선택, 업그레이드, 구독 관리 | [[10_제품/기획/app/index]] |
| `plm` | 도면, BOM, 변경관리, 부품 등 설계 기준본 관리 | `10_제품/기획/plm/*` |
| `mes` | 작업지시, 생산실적, 품질, 생산현황 등 실행 관리 | [[10_제품/기획/mes/index]] |

현재 구조에서 가장 중요한 경계는 아래와 같다.

- `site`는 외부 방문자가 보는 공개 경험까지만 맡는다.
- `app`은 계정 생성 이후의 온보딩과 구독 흐름을 맡는다.
- `plm`은 설계 데이터 기준본을 다루는 코어 제품 문서군이다.
- `mes`는 설계 기준본이 현장 실행으로 이어지는 문서군이다.

## 4. 문서군 연결 방식

각 문서군은 독립되어 있지 않고 아래 순서로 이어진다.

```text
전략 / 정책
    ↓
site
    ↓
app
    ↓
plm / mes
```

이 흐름의 의미는 아래와 같다.

- `site`는 적합한 방문자를 파일럿, 데모, 상담으로 전환한다.
- `app`은 실제 계정 생성과 구독 시작 흐름을 다룬다.
- `plm`과 `mes`는 앱 안에서 제공할 핵심 제품 경험을 다룬다.

단, 공개 경험과 앱 내부 구독 흐름은 서로 연결되더라도 같은 문서에서 섞지
않는다.

## 5. 문서군별 작성 기준

상세 기획 문서를 작성할 때는 아래 기준을 공통으로 적용한다.

1. 문서가 맡는 범위와 맡지 않는 범위를 먼저 적는다.
2. 상위 정책 문서가 있으면 frontmatter와 본문에서 참조를 명시한다.
3. 공개 경험과 앱 내부 흐름을 한 문서에 같이 쓰지 않는다.
4. 가격 숫자, 가입 정책, 업그레이드 의미는 정책 문서를 다시 풀어쓰지 않고
   참조형으로 연결한다.
5. 구현 상태, 운영 규칙, CRM 처리 기준은 제품 기획 문서에 과도하게 섞지 않는다.

## 6. 현재 우선 진입점

현재 구조에서 먼저 읽어야 할 상위 문서는 아래와 같다.

- 공개 페이지 기획을 볼 때: [[10_제품/기획/site/index]]
- 앱 온보딩과 구독 흐름을 볼 때: [[10_제품/기획/app/index]]
- 실행 관리 기획을 볼 때: [[10_제품/기획/mes/index]]

`plm` 문서군은 현재 별도 상위 인덱스가 없으므로, 후속 작업에서 `plm`
상위 진입 문서를 추가해 구조를 더 선명하게 만드는 편이 맞다.

## 7. Next steps

다음 정리 작업은 아래 순서로 이어진다.

1. `plm` 문서군의 상위 진입 문서를 만든다.
2. `site` 하위 문서에서 공개 경험 밖의 가입, 운영, 구현 명세를 계속 걷어낸다.
3. `app` 문서군을 가입, 플랜 선택, 업그레이드, 좌석/구독 관리 세부 문서로
   확장한다.
