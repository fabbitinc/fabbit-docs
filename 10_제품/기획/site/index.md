# 사이트 기획

이 문서는 Fabbit 대외 사이트 기획의 기준 문서다. 제품 기획 문서군 중에서도
획득과 전환을 담당하는 웹 페이지를 정의하며, 랜딩 페이지, 독립 요금제
페이지, 가입과 파일럿 문의 같은 외부 진입 화면을 관리한다.

## 1. 사이트의 역할

이 문서군은 외부 방문자가 보는 공개 경험만 다룬다. 목적은 제품의 문제 해결
범위를 전달하고, 적합한 방문자를 `가입`으로 전환하며, 파일럿 문의는 보조
경로로 제공하는 것이다.

- 제품을 처음 접한 방문자에게 Fabbit의 문제 해결 범위를 짧은 시간 안에 전달
- 초기 타겟 고객이 `가입`으로 바로 진입할 수 있게 안내
- 파일럿이 필요한 팀에는 보조 문의 경로를 제공
- 사업계획서와 영업 자료의 핵심 메시지를 웹 경험으로 일치시키기

## 2. 사이트가 맡는 범위

`site` 문서군은 공개 페이지에서 방문자가 실제로 보게 되는 정보와 흐름만
정의한다. 정책 원문이나 앱 내부 처리 로직은 참조할 수 있지만, 이 문서군에서
직접 소유하지는 않는다.

- 랜딩 페이지의 메시지, 섹션 구조, CTA 배치
- 독립 요금제 페이지의 비교 기준, 전체 과금 공개 범위, 요금 계산기, CTA
- 공개 가입 진입점과 파일럿 문의 폼의 단계, 입력 항목, 완료 화면
- 공개 페이지 간 연결 구조와 진입/이탈 경로
- 외부 방문자 기준의 최소 추적 이벤트와 공개 경험 용어

## 3. 사이트가 맡지 않는 범위

`site` 문서군은 공개 경험 밖으로 들어가는 순간부터 주도권을 넘긴다. 특히
가입, 플랜 선택, 업그레이드, 구독 변경 같은 앱 내부 흐름은 별도 문서군으로
관리한다.

- 플랜/과금 정책의 source of truth
- 워크스페이스 회원가입과 생성 처리 순서
- 앱 내부 플랜 선택 화면과 상태 전이
- Starter에서 Team·Organization으로의 업그레이드 처리
- 좌석 배정, 구독 변경, 청구 설정 같은 앱 내부 운영 흐름
- 리드 분류 운영 규칙, CRM 처리, 백오피스 운영 정책
- 서버 API, 구현 상세, 결제/정산 로직 설명

가입, 플랜 선택, 업그레이드, 구독 관리는 [[10_제품/기획/app/index]]에서
다룬다.

## 4. 기획 원칙

이 문서군은 공개 페이지 기획답게 외부 노출 관점의 일관성을 우선한다. 아래
원칙은 랜딩, 요금제, 신청 흐름 문서에 공통으로 적용한다.

- 첫 화면에서는 `PLM`, `MES`, `온톨로지`보다 고객 언어를 우선한다.
- 현재 구현된 기능과 계획 중인 기능을 명확히 분리한다.
- 메인 CTA는 `가입`으로 두고, `파일럿 신청`은 보조 문의 경로로 둔다.
- `데모 요청`은 공개 CTA에서 제거한다.
- 요금 페이지는 독립 URL로 운영하고 전체 요금/과금 정보를 공개한다.
- 제품 스크린샷, 정량 근거, 도입 대상 선별 정보를 함께 배치한다.

## 5. 문서군 연결 관계

공개 사이트와 앱 내부 문서군은 서로 연결되지만 역할은 다르다. 아래 표를
기준으로 어떤 문서가 어디까지 책임지는지 구분한다.

| 주제 | 소속 문서군 | 역할 |
|------|-------------|------|
| 랜딩 메시지와 CTA | `site` | 공개 첫 화면의 설명과 전환 설계 |
| 요금제 페이지 공개 정보 | `site` | 전체 과금 공개, 계산기, 비활성 플랜 노출 |
| 공개 가입 및 파일럿 문의 | `site` | 가입 진입과 파일럿 문의 경험 |
| 회원가입과 워크스페이스 생성 | `app` | 앱 진입 이후의 온보딩 흐름 |
| 플랜 선택과 업그레이드 | `app` | 앱 내부 구독 흐름 |
| 좌석 관리와 구독 운영 | `app` | 워크스페이스 운영 흐름 |
