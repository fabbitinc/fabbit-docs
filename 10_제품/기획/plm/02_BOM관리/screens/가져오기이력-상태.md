# 작업 상태

합성 작업 상태는 BOM 데이터 신뢰성과 직접 연결되므로, 상태 전이와 폴링 규칙을
화면에서 일관되게 해석해야 한다.

- `PENDING`, `PROCESSING` 작업이 있으면 10초 간격 폴링을 유지한다.
- `COMPLETED`는 최종 반영 건수(`relationshipsCreated`)를 강조한다.
- `FAILED`는 오류 메시지 목록을 숨기지 않고 즉시 확인 가능해야 한다.
- 재시도 버튼은 MVP에 두지 않는다. 실패 복구는 새 합성 작업 생성으로 처리한다.
- `empty` 상태는 아직 BOM 반영 이력이 없다는 뜻이지 오류가 아니다.
