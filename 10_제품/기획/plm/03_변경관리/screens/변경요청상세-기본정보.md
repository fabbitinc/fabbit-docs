# 변경요청 상세 · 기본 정보

변경요청 상세는 `Issue.state`와 `crState`를 함께 보여주는 화면이다.
사용자는 현재 작업이 진행 중인지, 반영이 끝났는지, 반영 없이 종료됐는지
한 번에 구분할 수 있어야 한다.

## 동작 규칙

- 상단에 `번호`, `제목`, `기본 상태`, `변경요청 상태`를 함께 노출한다.
- 부품, 라벨, 첨부파일, 담당자는 첫 화면에서 바로 보이게 배치한다.
- `MERGED`이면 `mergedAt`, `mergedBy`를 별도 필드로 노출한다.
- `CLOSED`이면 `반영 없이 종료됨` 문구를 함께 표시한다.

## UI 상태

- `loading`: 상태 배지와 기본 정보 블록을 스켈레톤 처리한다.
- `error`: 목록 복귀 CTA와 재조회 버튼을 함께 노출한다.
- `read only`: 값은 모두 보이고 편집 액션만 숨긴다.
