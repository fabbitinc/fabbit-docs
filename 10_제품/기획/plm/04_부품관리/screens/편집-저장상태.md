# 편집 화면 · 저장과 상태

편집 화면은 `저장`만 담당한다. `Release`와 `상태 변경`은 상세 화면에서
별도 액션으로 처리하고, 편집 화면에서는 현재 Draft를 안전하게 갱신하는 데
집중한다.

## CTA 규칙

- 기본 CTA는 `저장`, 보조 CTA는 `취소` 두 개만 둔다.
- 저장 버튼은 필수값 누락, 잠금 상태, 권한 없음 상태에서 비활성화한다.
- 화면 하단 보조 문구로 `Release와 상태 변경은 상세 화면에서 진행`을 노출한다.

## 저장 처리

1. 사용자가 `저장`을 누른다.
2. 프론트는 필수값, 중복, 잠금 여부를 선검증한다.
3. 서버 저장이 성공하면 같은 화면에 머무르며 성공 토스트를 노출한다.
4. 저장 후 상세 화면 자동 이동은 하지 않는다.

## 오류 처리

- `VALIDATION_ERROR`: 필드 단위 오류를 해당 입력 아래에 연결한다.
- `DUPLICATE_PART_NO`: 품번 필드에 중복 오류를 표시한다.
- `LOCK_CONFLICT`: 잠금 배너를 갱신하고 저장 버튼을 비활성화한다.
- `REVISION_NOT_EDITABLE`: 화면 상단 오류 배너와 함께 상세 화면 복귀 CTA를
  노출한다.
- `PERMISSION_DENIED`: 전체 폼을 읽기 전용으로 전환한다.

## 이탈 방지

- 사용자가 변경사항이 있는 상태에서 `취소`, 뒤로 가기, 화면 이탈을 시도하면
  확인 모달을 띄운다.
- 저장 직후에는 이탈 경고를 해제한다.

## 로딩 / 빈 상태 / 잠금 상태

- `loading`: 저장 버튼 자리에 비활성 버튼 스켈레톤을 노출한다.
- `saving`: 저장 버튼 레이블을 `저장 중...`으로 바꾸고 중복 제출을 막는다.
- `locked`: 잠금 보유자, 잠금 시각, 자동 해제 기준을 표시한다.
- `permission denied`: 저장 버튼을 숨기고 `읽기 전용` 안내만 남긴다.
