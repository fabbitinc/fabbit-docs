# 업로드 다이얼로그 · 요약

부품 업로드 다이얼로그는 `매핑 선택 -> 파일 검증 -> 업로드 실행 -> 배치 진행률`
흐름을 한 화면에서 처리한다. 목록 화면의 보조 액션이지만, 검증과 비동기 처리
상태가 복합적으로 섞이므로 별도 화면 정책으로 관리한다.

## 핵심 규칙

- 업로드는 `xlsx`, `xls`, `csv` 파일만 허용하고, 여러 파일을 한 번에 추가할
  수 있다.
- 사용자는 업로드 전에 매핑을 반드시 선택해야 한다.
- 파일 추가 직후 헤더를 읽어 필수 컬럼 누락 여부를 판정한다.
- 필수 컬럼이 누락된 파일은 실패 상태로 남기고 업로드 실행 대상에서 제외한다.
- 매핑에 없는 추가 컬럼은 경고 배지로만 보여주고 업로드는 허용한다.
- ROOT BOM 매핑이면 파일별 `부품/공급사/도면/프로젝트` 루트 컨텍스트 입력이
  추가될 수 있다.
- 루트 컨텍스트 입력은 자동완성 검색과 직접 입력을 함께 허용한다.
- `모두 적용`은 같은 라벨의 값을 같은 배치의 다른 완료 파일에 일괄 복사한다.
- `기존 값 덮어쓰기` 스위치는 업로드 요청의 `overwrite` 값과 1:1로 연결된다.
- 업로드 실행 후에는 배치 진행률 패널로 전환하고, 완료 전에는 닫기를 막는다.
- 매핑이 없거나 매핑 조회에 실패하면 `속성 분석으로 이동` CTA를 노출한다.

## 리뷰 포인트

- 파일별 성공/실패/경고 상태가 한눈에 구분되어야 한다.
- 배치 진행률 화면은 파일명, 처리 건수, 실패 건수를 빠르게 읽을 수 있어야 한다.
- ROOT BOM 보강 입력은 파일 카드 안에서 바로 수정 가능해야 한다.
- 업로드 중에는 닫기, 파일 제거, 드래그 앤 드롭이 비활성화되어야 한다.
