# 목록 화면 · 요약

목록 화면은 기본 CRUD 진입용 조회 화면으로 본다. 화면별 세부 정책을 여러
문서로 쪼개지 않고, 핵심 규칙만 이 문서에 요약한다.

## 핵심 규칙

- 품번·품명 통합 검색과 `카테고리`, `상태`, `도면 유무`, `하위 부품 유무`
  필터를 함께 제공한다.
- `유형` 필터는 기획 범위에 유지하되 화면에는 아직 없다. 🔴 (미개발)
- 필터 조합은 AND 연산을 사용하고, 검색어/필터 변경 후 `검색` 버튼 또는
  Enter 입력 시 적용한다.
- 적용된 필터는 칩으로 노출하고, 칩 단위 해제가 가능해야 한다.
- 테이블은 페이지 크기 `15 / 30 / 50`을 지원하고, 행 클릭은 상세 화면으로
  이동한다.
- 테이블 컬럼은 `품번`, `품명`, `카테고리`, `Rev`, `상태`, `도면`,
  `하위 부품`을 기본으로 사용한다.
- `최종수정일`, `수정자`, `유형` 컬럼은 기획 범위에 유지하되 화면에는 아직
  없다. 🔴 (미개발)
- 상단 액션은 `속성 분석`, `부품 업로드`, `새 부품` 🔴 (미개발)으로 둔다.
- 선택 부품이 1건 이상이면 `프로젝트 연결`과 `내려받기` 드롭다운을 노출한다.
- 내려받기는 `선택`, `전체`, `검색 결과` 3가지 범위를 지원한다.
- 삭제와 일괄 상태 변경은 목록에 두지 않는다.

## 리뷰 포인트

- 상태 배지와 현재 리비전이 한 줄에서 함께 읽혀야 한다.
- 도면 아이콘과 하위 부품 수는 텍스트보다 빠르게 인지되게 보여야 한다.
- 선택 상태에서는 필터 버튼보다 선택 액션이 우선 노출되어야 한다.
- Obsolete 행은 탐색은 가능하되 시각적으로 회색 처리한다. 🔴 (미개발)
- 목록은 탐색과 진입, 선택 기반 보조 액션만 담당하고, 상태 변경과 삭제는
  상세 화면으로 넘긴다.
