# 편집 화면 · 속성

속성 영역은 `핵심 속성`과 `커스텀 속성`으로 나뉜다. 핵심 속성은 고정 필드로,
커스텀 속성은 사용자가 현장에서 행을 추가해 저장할 수 있는 가변 필드로
구성한다.

## 동작 규칙

- 핵심 속성은 `재질`, `무게`, `제조사`, `RoHS`를 기본 제공한다.
- 커스텀 속성은 사용자가 `속성명`, `값`, `단위`를 입력해 행 단위로 추가한다.
- `+ 속성 추가`는 새 빈 행을 추가하고, 저장 전까지는 Draft에만 반영된다.
- 커스텀 속성 삭제는 Draft에서만 허용한다.

## 필드별 제약

| 영역 | 필드 | 규칙 |
|------|------|------|
| 핵심 속성 | 재질 | 자유 텍스트 |
| 핵심 속성 | 무게 | 숫자 입력, 음수 불가 |
| 핵심 속성 | 제조사 | 자유 텍스트 |
| 핵심 속성 | RoHS | `Y / N / N/A` 중 선택 |
| 커스텀 속성 | 속성명 | 필수, 중복 불가 |
| 커스텀 속성 | 값 | 필수 |
| 커스텀 속성 | 단위 | 선택 |

## 유효성 검증

- 커스텀 속성명은 같은 Draft 안에서 중복될 수 없다.
- 속성명이나 값이 비어 있는 행은 저장할 수 없다.
- 무게는 숫자 형식이 아니면 저장할 수 없다.
- Production 이상 부품에서도 Draft 편집 안에서는 속성 수정이 가능하나,
  변경 사유와 함께 저장해야 한다.

## 예외 케이스

- 잠금 상태에서는 `+ 속성 추가`와 삭제 버튼을 숨기거나 비활성화한다.
- 서버가 지원하지 않는 속성 타입이 응답으로 오면 읽기 전용 텍스트로만 노출하고
  저장 대상에서는 제외한다.
- Obsolete 상태는 전체 읽기 전용이다.

## UI 상태

- `empty`: 커스텀 속성이 없으면 `아직 추가된 속성이 없습니다` 안내를 노출한다.
- `error`: 특정 속성 행 저장 실패 시 해당 행 하단에 오류 메시지를 붙인다.
- `locked`: 커스텀 속성 추가/삭제 액션을 비활성화한다.
