# 편집 화면 · 변경 사유

변경 사유 영역은 저장 책임성을 확보하기 위한 필드다. Draft를 저장하는 시점에
부품 상태에 따라 필수 여부가 달라진다.

## 동작 규칙

- 신규 등록에서는 변경 사유를 선택 입력으로 둔다.
- Design, Prototype 부품의 Draft 저장에서는 변경 사유를 권장 입력으로 둔다.
- Production, EOL 부품의 Draft 저장에서는 변경 사유를 필수로 요구한다.
- 변경 사유는 저장 시 감사로그 메시지의 원문 또는 요약값으로 재사용할 수 있다.

## 유효성 검증

- 필수 상태에서 공백만 입력하면 저장할 수 없다.
- 최대 길이는 백엔드 제약에 맞춰 제한하고, 초과 시 입력창 하단에 남은 글자 수
  또는 오류를 표시한다.
- 자동 생성된 Draft에서도 첫 저장 시 변경 사유 검증을 동일하게 적용한다.

## 예외 케이스

- 잠금 상태에서는 기존 변경 사유를 읽기 전용으로만 보여준다.
- 권한이 없는 사용자는 입력창 대신 감사 목적 안내 문구를 본다.
- Obsolete는 편집 불가이므로 변경 사유 입력 UI를 표시하지 않는다.

## UI 상태

- `default`: 플레이스홀더는 `무엇이 왜 바뀌었는지 입력하세요`로 통일한다.
- `required`: 필수 배지 또는 보조 문구로 `Production 이상 필수`를 함께 보여준다.
- `error`: 저장 실패 시 입력창 아래에 검증 오류 메시지를 노출한다.
