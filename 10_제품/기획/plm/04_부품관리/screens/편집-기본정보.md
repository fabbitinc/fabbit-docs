# 편집 화면 · 기본 정보

편집 화면은 `신규 등록`, `Draft 편집`, `Released 기준 새 Draft 생성 후 편집`
세 가지 상황을 하나의 레이아웃으로 처리한다. 상단 헤더 문구와 일부 필드
상태만 달라지고, 기본 정보 블록 구성은 동일하다.

## 동작 규칙

- 신규 등록 시 헤더는 `새 부품 등록`, 설명은 `초기 부품과 Rev.A Draft 생성`
  으로 노출한다.
- 기존 Draft 편집 시 헤더는 `Draft 편집`, 현재 리비전과 잠금 상태를 함께
  보여준다.
- Released 기준 편집 시 헤더는 `새 Draft 생성 후 편집`, 기존 Released를
  수정하지 않는다는 안내 문구를 노출한다.
- 저장 성공 시 사용자는 같은 편집 화면에 머무르며, 상단에 `Draft 저장됨`
  토스트를 본다.

## 필드 규칙

| 필드 | 신규 등록 | Draft 편집 | Released 이후 편집 |
|------|-----------|------------|--------------------|
| 품번 | 입력 가능, 필수 | Design 상태에서만 수정 가능 | 읽기 전용 |
| 품명 | 입력 가능, 필수 | 수정 가능 | 새 Draft에서 수정 가능 |
| 유형 | 입력 가능, 필수 | 수정 가능, 리비전 영향 항목 | 새 Draft에서 수정 가능 |
| 카테고리 | 선택 | 수정 가능 | 수정 가능 |

## 유효성 검증

- 품번은 조직 내 유일해야 한다.
- 품번, 품명, 유형이 비어 있으면 저장할 수 없다.
- 품번은 공백만 입력할 수 없다.
- 유형은 `부품`, `조립품`, `원자재` 중 하나여야 한다.

## 예외 케이스

- Released 기준 편집 진입 시 이미 Draft가 있으면 새 Draft를 만들지 않고 기존
  Draft로 이동한다.
- Obsolete 부품은 편집 진입이 불가하며 상세 화면에서 차단 메시지를 보여준다.
- 권한이 없는 사용자는 편집 화면에 진입하더라도 모든 필드가 읽기 전용이다.

## UI 상태

- `loading`: 기본 정보 스켈레톤 대신 읽기 전용 플레이스홀더를 보여준다.
- `error`: 화면 상단에 재시도 배너와 상세 오류 코드를 노출한다.
- `locked`: 헤더 아래 잠금 배너를 표시하고 필드를 모두 잠근다.
- `permission denied`: 입력 필드 대신 읽기 전용 값과 권한 안내 문구를 노출한다.
