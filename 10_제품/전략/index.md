# 제품 전략 인덱스

이 문서는 `10_제품/전략` 폴더의 상위 진입 문서다. 제품 전략 문서는 Fabbit이
어떤 문제를 누구에게 어떤 순서로 풀지 정의하지만, 현재 운영 규칙 자체를
고정하지는 않는다.

제품 전략을 읽을 때는 먼저 정책 SSOT를 확인한 뒤, 그 기준 위에서 제품 정의와
로드맵, 경쟁 구도, 문제-해결 구조를 해석하는 순서를 따른다.

## 1. 이 폴더의 역할

`10_제품/전략`은 아래 질문에 답하는 문서군이다.

- Fabbit은 어떤 제품인가
- 어떤 고객 문제를 우선 해결하는가
- 현재와 다음 단계의 제품 범위는 무엇인가
- 경쟁사와 비교할 때 어떤 차별점을 주장할 수 있는가

반대로 아래 항목은 이 폴더가 직접 고정하지 않는다.

- 현재 타깃 고객과 도입 경로의 운영 기준
- 플랜명, 좌석, 스토리지, AI, 업그레이드 정책
- 공개 사이트 CTA와 신청 폼의 세부 구조
- 서버와 웹의 실제 구현 truth

이 항목들은 각각 [[10_제품/정책/index]],
[[40_사업/정책/index]], [[10_제품/기획/index]],
[[20_개발/코드베이스-현황]]을 먼저 따른다.

## 2. 먼저 읽는 순서

제품 전략 문서를 읽을 때는 아래 순서를 기본으로 잡는다. 이렇게 읽어야
전략 문서 안의 오래된 가설이 현재 정책 SSOT를 덮어쓰지 않는다.

1. [[10_제품/정책/고객-세그먼트-도입-정책]]에서 현재 타깃 고객과 도입 기준을 확인한다.
2. [[40_사업/정책/플랜-과금-정책]]에서 현재 플랜과 과금 구조를 확인한다.
3. [[10_제품/전략/아이템-정의]]에서 제품 정의와 상위 가치 제안을 확인한다.
4. [[10_제품/전략/제품-로드맵]]에서 현재 구현 범위와 단계별 우선순위를 확인한다.
5. 그 다음에 문제-해결 매핑, 경쟁사 비교, 성능 측정 문서를 보조 근거로 읽는다.

## 3. 현재 문서 구조

현재 `10_제품/전략` 폴더는 아래처럼 읽는 것이 가장 명확하다.

| 문서 | 역할 | 분류 |
|------|------|------|
| [[10_제품/전략/아이템-정의]] | 제품 정의, 문제, 가치 제안, 상위 포지셔닝 | 기준 전략 |
| [[10_제품/전략/제품-로드맵]] | 구현 상태, 기능 범위, 단계별 우선순위 | 기준 전략 |
| [[10_제품/전략/문제해결-매핑]] | 고객 문제와 해결 구조 매핑 | 파생 전략 |
| [[10_제품/전략/경쟁사-분석]] | 경쟁사와 대체재 비교 | 파생 전략 |
| [[10_제품/전략/MVP-성능-측정]] | 성능 검증 템플릿과 근거 초안 | 보조 문서 |

현재 기준 문서로 먼저 봐야 하는 것은 `아이템-정의`와 `제품-로드맵`이다.
나머지 문서는 이 둘과 정책 SSOT를 참조해 해석하는 보조 문서로 보는 편이 맞다.

## 4. 인접 SSOT와의 관계

제품 전략 문서는 정책 SSOT와 가까워 보이지만 역할이 다르다. 아래 표처럼
경계를 잡으면 문서 배치가 쉬워진다.

| 주제 | 기준 문서 |
|------|-----------|
| 고객 세그먼트, 파일럿 중심 도입 기준 | [[10_제품/정책/고객-세그먼트-도입-정책]] |
| 플랜, 좌석, 스토리지, AI, 업그레이드 | [[40_사업/정책/플랜-과금-정책]] |
| 제품 정의와 가치 제안 | [[10_제품/전략/아이템-정의]] |
| 현재 구현 범위와 단계 | [[10_제품/전략/제품-로드맵]] |
| 화면과 사용자 흐름 | [[10_제품/기획/index]] |

전략 문서 안에서 가격표, 세그먼트 운영 규칙, 가입 정책 같은 내용이 보이면
그 서술이 현재 기준을 고정하는지 먼저 의심하고 정책 문서와 교차 확인해야
한다.

## 5. 어디까지가 전략 문서인가

`10_제품/전략`은 제품의 방향과 근거를 설명하는 폴더다. 따라서 아래 내용은
전략 문서에 두는 편이 맞다.

- 왜 이 문제를 풀어야 하는가
- 어떤 차별성이 있는가
- 어떤 기능이 어떤 단계에서 필요한가
- 어떤 경쟁 구도에서 어떤 포지션을 취하는가

반대로 아래 내용은 다른 폴더가 더 적합하다.

- 현재 회원가입, 플랜 선택, 업그레이드 흐름 설명
- 공개 랜딩과 요금제 페이지 카피
- 멤버 상한, 좌석 단가, 스토리지 계산식
- 서버 모듈, API, 웹 라우트의 구현 상세
