# MVP 성능 측정 템플릿

이 문서는 Fabbit의 성능 지표를 실제 측정해 넣기 위한 준비중 템플릿이다. 현재
기준 성능 수치나 확정 근거를 담는 문서가 아니며, 실측 데이터가 없는 상태에서
이 문서의 빈 칸이나 예시 문구를 현재 truth처럼 읽으면 안 된다.

현재 제품 범위와 구현 상태는 [[10_제품/전략/제품-로드맵]]과
[[20_개발/코드베이스-현황]]을 먼저 따른다. 이 문서는 그 기준 위에 실측값을
나중에 채워 넣는 보조 문서다.

## 1. 문서 역할

이 템플릿은 아래 상황에서만 사용한다.

- 지원사업 제출용 성능 근거가 필요할 때
- 데모와 파일럿 이후 실측 수치를 정리할 때
- 문제해결 매핑 문서의 목표 수치를 실제 측정값으로 보강할 때

반대로 아래 용도로 사용하면 안 된다.

- 현재 제품이 이미 달성한 성능의 SSOT로 인용
- 공개 사이트나 사업계획서에서 확정 수치처럼 재사용
- 아직 측정하지 않은 항목의 목표를 현재 성능처럼 해석

## 2. 작성 전에 먼저 확인할 기준

이 템플릿에 값을 넣기 전에는 아래 문서를 먼저 확인한다.

1. [[10_제품/전략/문제해결-매핑]]에서 어떤 문제를 어떤 지표로 검증할지 확인한다.
2. [[10_제품/전략/제품-로드맵]]에서 현재 구현 범위와 준비중 범위를 확인한다.
3. [[20_개발/코드베이스-현황]]에서 실제 구현 모듈과 기능 상태를 확인한다.

즉, 이 템플릿은 전략과 구현 기준을 검증값으로 보강하는 용도이지, 기준을
새로 정의하는 문서가 아니다.

## 3. 입력 원칙

실제 수치를 채울 때는 아래 원칙을 따른다.

- 샘플 수, 샘플 출처, 측정 환경을 같이 적는다.
- 목표값과 실측값을 섞지 않는다.
- 인터뷰 기반 추정치는 실측과 구분해 적는다.
- 공개 자료에 재사용할 때는 출처와 측정 날짜를 함께 적는다.

## 5. 처리 시간

이 섹션은 수작업 대비 시간 절감 효과를 기록하는 자리다. 인터뷰 기반 추정치는
근거 문구로만 두고, 실측 결과와 분리해 적는다.

## 6. 도면 인식 범위

이 섹션은 구현 여부와 측정 준비 상태를 같이 적는 용도다. 현재 상태가
준비중이면 그대로 적고, 완료처럼 포장하지 않는다.

| 항목 | 현재 상태 | 측정 여부 |
|------|----------|----------|
| 표제란 인식 | 준비중 / 완료 | 미측정 / 측정 완료 |
| 도면 번호 추출 | 준비중 / 완료 | 미측정 / 측정 완료 |
| 도면 버전 인식 | 준비중 / 완료 | 미측정 / 측정 완료 |
| 완성 예정일 | | |

## 7. 데모 자료 체크리스트

이 체크리스트는 발표나 제출 자료용 보조 자료를 준비할 때 사용한다.

- [ ] BOM 자동생성 과정 스크린샷
- [ ] 처리 시간 비교 화면
- [ ] 데모 영상
- [ ] 측정 환경 설명 1페이지

## 8. next steps

이 문서를 실제 근거 문서로 올리려면 빈 칸을 메운 뒤에도 별도 검토가 필요하다.

- 파일럿 또는 내부 테스트 결과로 최소 1차 실측값을 채운다.
- 수치가 공개 자료에 재사용 가능한지 검토한다.
- 실측값이 쌓이면 이 문서를 템플릿이 아니라 검증 보고서로 분리할지 판단한다.
