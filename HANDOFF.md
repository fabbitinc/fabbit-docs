## What Worked

- **NotebookLM + yt-dlp 워크플로우:** 유튜브 검색 → 소스 추가 → 질문으로 빠르게 공고 분석 가능
- **공고문 PDF를 NotebookLM 소스에 직접 추가:** 유튜브보다 정확한 평가항목·배점 추출 가능
- **마감순 작성:** G스타·경기기술은 gsp.or.kr 같은 플랫폼이라 양식 유사, 한번에 작성 효율적
- **원본팩트 + 사업계획서용 서술 분리:** 대표자 스토리에서 솔직한 내용과 포장된 서술을 분리 관리하면 일관성 유지 + 수정 용이
- **노션 → docs 이관:** 노션 MCP로 영업 리스트업 DB를 읽어서 인터뷰 기록/피드백으로 분리 정리
- **인터뷰 기록/피드백 분리:** 원본(누가 뭐라 했는지)과 분석(왜 실패, 어떻게 보강)을 분리하면 재활용 쉬움
- **서버 코드 기반 과금 설계:** PlanLimits 코드에서 실제 플랜 구조를 읽어 문서에 반영 → 코드/문서 불일치 방지
- **Cloudflare R2 원가 분석:** egress 무료 덕분에 스토리지를 넉넉하게 제공 가능 (AWS였으면 원가 초과)
- **NotebookLM 교차검증:** RESEARCH.md 수치를 제조업 리서치 노트북으로 교차검증 → 검색 시간 40%→25~30% 보정, 클라우드 비중 출처별 차이 발견
- **정량 근거에 출처 각주:** 발표자료 제작 시 재추적 가능하도록 [^footnote] 형식으로 출처 명시
- **archive → 활성 문서 통합:** business-roadmap.md의 바우처 GTM 전략을 수익모델-정의.md에 필요한 만큼만 정리하여 추가

## What Didn't Work

- 유튜브 소스만으로는 정확한 배점·평가지표 확인 불가 → 공고문 PDF 소스 필수
- `pip install`은 macOS externally-managed 에러 → `pipx` 사용 필수
- NotebookLM 로그인은 터미널에서 ENTER 입력 필요
- **1차 인터뷰 타겟 미스매치:** 1~9명 영세기업 대상 콜드콜 → "관리할 capa가 없다" 반응. 10~49명, 매출 10억+ 타겟 필요
- **"PLM" 용어 사용:** 6건 중 PLM 아는 곳 0건. "도면/BOM 관리"로 풀어 설명해야 함
- **AI 크레딧 과금 설계 시도 → 보류:** BOM 분석(도입 초기만), 채팅(일상), 도면 분석(비전 모델) 등 사용 패턴이 너무 달라 고려사항 과다. 실사용 데이터 확보 후 재설계
- **리서치 스킬 (Tavily 기반):** 폐기됨. 사용 불가
