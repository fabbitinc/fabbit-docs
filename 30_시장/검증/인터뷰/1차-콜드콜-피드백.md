# 1차 콜드콜 피드백

## 1. 타겟 실패 원인

### 컨택 방식 문제

- 인터넷으로 수집한 전화번호는 보통 **CS 담당자**에게 연결됨
- 의사결정자(대표/실무자)까지 도달하지 못함
- #14 씨엔원: "직원이라 결정할 수 없는 듯함"
- #15 세일레이저: "담당자가 아니라서 모르겠다"

### 타겟 규모 미스매치

- 인터뷰 대상 대부분이 **1~9명 영세기업/소공인**
- PLM을 사용할 정도의 매출액이 나와야 함
- "영세기업이라 관리할 capa가 나오지도 않음" (#11)
- "규모가 작아 사용하지 않는다" (#23)
- "큰 데다 문의해봐라" (#22)

### PLM 인지도 부재

- 6건 중 **PLM이라는 단어를 아는 곳이 0건**
- #14: "PLM이란 단어를 모르는 듯함"
- #15: "PLM은 모르겠고"
- ERP는 사용하나 PLM은 모르는 경우가 다수

## 2. 현재 관리 방식 분포

| 방식 | 건수 | 비고 |
|------|------|------|
| 엑셀 | 1건 | #22 디앤디아이 |
| 폴더/파일 기반 | 2건 | #11 동광기계, #25 두원이엔지 |
| ERP (SAP HANA, 기업은행) | 2건 | #14 씨엔원, #15 세일레이저 |
| 관리 안 함 | 1건 | #23 성진이엔지 (외부 위탁) |

## 3. 보강 전략

### 컨택 방식 전환

- 대표 직접 연락, 또는 **오픈톡방에서 실무자와 컨택**
- 매출액 **최소 10억 이상** 기업으로 필터
  - 공공데이터(중소기업현황정보시스템, 크레딧잡)에서 사전 확인
  - 또는 첫 질문에 "직원 몇 분이세요?"로 10명 미만 빠르게 종료
- 다음 인터뷰 대상: **10~49명 규모**, C25/C29/C28 업종

### 용어 전략

콜드콜/초기 미팅에서 **"PLM" 단어를 앞에 세우지 않는다.**
"모른다 → 끊는다/관심 없다"로 가기 쉬움.

대신 사용할 표현:
- "도면/BOM 통합관리"
- "설계변경(리비전) 통제"
- "ERP 연동용 BOM 자동 생성/검증"
- "최신 도면 배포/현장 혼선 방지"

### PLM ↔ ERP 시너지 전략

고객이 ERP에서 느끼는 가장 큰 가치는 **데이터 정확도/통제** (#14: "ERP의 좋은 점 = 데이터의 정확도").
그런데 현실은 설계/도면/BOM이 엑셀·폴더·메일에 흩어져서 ERP가 정확해지기 어렵다.

> "ERP를 이미 쓰고 있다"는 건 오히려 **Fabbit이 끼어들 지점(업스트림 데이터 정리)이 존재한다**는 뜻.

**ERP가 잘하는 것:**
- 구매, 재고, 생산, 출하, 원가 같은 트랜잭션 실행
- MRP/발주/입출고/전표/정산 같은 업무 처리

**ERP가 약한 것 (Fabbit의 기회):**
- 도면 파일/리비전의 버전 통제와 "최신본 보장"
- 설계 BOM(E-BOM) 정리, 변경 이력, 승인 절차 (ECO/ECN)
- "누가 언제 무엇을 바꿨고, 그 변경이 언제부터 적용되는지" 추적성/효력일(effectivity)

**어필 포인트:**
- 부품관리를 엑셀로 한다면 → 엑셀 작성 시간, 중복, 오타 등 번거로움
- ERP에 입력하기 위해 어떤 작업을 하는지 → 이를 PLM으로 대신할 수 있어야 함 (대부분 엑셀 정리 작업)

## 4. 유료 전환 가능성 순위

1. **ERP는 있는데** BOM/도면은 엑셀·폴더인 제조/조립/가공 업체
2. 설계변경이 잦고 외주/협력사가 많은 업체 (도면 배포 이슈가 큼)
3. 품번/리비전 체계가 약해서 구매/생산 오류가 반복되는 업체
4. **스마트공장 바우처 수혜 기업** — ERP/MES 도입 후 PLM이 다음 단계, 바우처로 Fabbit 비용 커버 가능
5. 아주 영세(관리 capa 자체가 없음) → 무료/라이트 플랜 유입만 노리거나 우선순위에서 제외
