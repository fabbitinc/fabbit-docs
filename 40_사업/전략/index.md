# 사업 전략 인덱스

이 문서는 `40_사업/전략` 폴더의 상위 진입 문서다. 사업 전략 문서는 Fabbit의
수익화, 시장 진입, 지원사업 대응, 실행 가설을 정리하지만, 현재 운영 정책을
직접 고정하지는 않는다.

현재 사업 전략을 읽을 때는 먼저 정책 SSOT를 확인하고, 그 다음에 매출 시나리오
와 GTM 가설을 읽는 순서가 맞다.

## 1. 이 폴더의 역할

`40_사업/전략`은 아래 질문에 답한다.

- Fabbit은 어떤 구조로 수익화할 것인가
- 어떤 고객군과 채널에서 시장에 진입할 것인가
- 어떤 가정으로 ARR과 단위경제성을 볼 것인가
- 팀과 실행 계획을 어떻게 설명할 것인가

반대로 아래 항목은 이 폴더가 직접 고정하지 않는다.

- 플랜명, 좌석 단가, 스토리지, AI, 업그레이드 규칙
- 타깃 고객의 운영 기준과 파일럿 우선순위
- 공개 페이지와 앱 내부 흐름의 세부 기획

이 항목들은 각각 [[40_사업/정책/index]],
[[10_제품/정책/index]], [[10_제품/기획/index]]를 먼저 따른다.

## 2. 먼저 읽는 순서

사업 전략 문서를 읽을 때는 아래 순서를 기본으로 한다.

1. [[10_제품/정책/고객-세그먼트-도입-정책]]에서 현재 타깃 고객과 기본 도입 경로를 확인한다.
2. [[40_사업/정책/플랜-과금-정책]]에서 현재 플랜과 과금 구조를 확인한다.
3. [[40_사업/전략/수익모델-정의]]에서 매출 구조와 ARR 가설을 확인한다.
4. [[40_사업/전략/시장진입-실행계획]]에서 GTM 가설과 실행 채널을 확인한다.
5. 개발계획과 팀구성 문서는 필요할 때 보조 문서로 읽는다.

## 3. 현재 문서 구조

현재 `40_사업/전략` 폴더는 아래처럼 구분해 읽는 편이 맞다.

| 문서 | 역할 | 분류 |
|------|------|------|
| [[40_사업/전략/수익모델-정의]] | BM 구조, 매출 가설, 단위경제성, 채널 가설 | 기준 전략 |
| [[40_사업/전략/시장진입-실행계획]] | 타깃 세그먼트와 GTM 실행 계획 | 기준 전략 |
| [[40_사업/전략/개발계획-마일스톤]] | 사업계획서용 개발 계획 골격 | 보조 문서 |
| [[40_사업/전략/팀구성-계획]] | 사업계획서용 팀 운영 골격 | 보조 문서 |

현재 실제로 우선 봐야 하는 문서는 `수익모델-정의`와 `시장진입-실행계획`이다.
나머지 두 문서는 아직 skeleton 성격이 강하므로, 운영 기준보다 제출용 보조
초안에 가깝게 읽는 편이 안전하다.

## 4. 인접 SSOT와의 관계

사업 전략 문서는 정책 SSOT를 다시 풀어쓰지 않고, 그 정책 위에서 가설과
수치를 계산해야 한다. 아래 표를 기준으로 경계를 잡는다.

| 주제 | 기준 문서 |
|------|-----------|
| 고객 세그먼트와 파일럿 우선순위 | [[10_제품/정책/고객-세그먼트-도입-정책]] |
| 플랜, 좌석, 스토리지, AI, 업그레이드 | [[40_사업/정책/플랜-과금-정책]] |
| 제품 정의와 범위 | [[10_제품/전략/아이템-정의]] |
| 공개 사이트와 앱 흐름 | [[10_제품/기획/index]] |

사업 전략 문서에 과거 가격표, 구식 플랜명, 오래된 세그먼트 기준이 남아
있다면, 먼저 정책 SSOT와 충돌하는지 확인해야 한다.

## 5. 어디까지가 사업 전략 문서인가

사업 전략 폴더에는 아래 내용이 들어가는 편이 맞다.

- BM 구조와 수익 가설
- 시장 진입 채널과 확장 순서
- CAC, LTV, ARR 같은 사업 수치 가정
- 지원사업과 투자 문맥에서 필요한 실행 논리

반대로 아래 내용은 다른 폴더가 더 적합하다.

- 현재 플랜의 정확한 가격 규칙과 업그레이드 동작
- 공개 요금제 페이지의 카피와 CTA
- 앱 내부 구독 화면과 상태 전이
