# 개발계획 마일스톤 템플릿

이 문서는 사업계획서나 지원사업 제출용으로 개발 계획을 정리할 때 사용하는
템플릿이다. 현재 확정 개발 일정이나 실제 마일스톤 SSOT가 아니며, 빈 항목과
예시 구조를 현재 실행 계획으로 읽으면 안 된다.

현재 제품의 실제 구현 상태와 범위는 [[20_개발/코드베이스-현황]]과
[[10_제품/전략/제품-로드맵]]을 먼저 따른다. 이 문서는 제출 문맥에 맞춰 그
기준을 다시 정리하는 보조 양식이다.

## 1. 문서 역할

이 템플릿은 아래 용도로만 사용한다.

- 지원사업 신청서의 개발 실행력 항목 작성
- 발표 자료의 분기별 개발 계획 요약
- 내부 초안 단계에서 마일스톤 설명 구조 정리

반대로 아래 용도로 사용하면 안 된다.

- 현재 팀의 공식 개발 일정 관리
- 서버, 웹, 디자인 작업의 실시간 운영 보드 대체
- 로드맵보다 우선하는 기준 문서

## 2. 작성 전에 먼저 확인할 기준

이 템플릿을 채우기 전에 아래 문서를 먼저 확인한다.

1. [[10_제품/전략/제품-로드맵]]에서 현재 구현 범위와 다음 단계 우선순위를 확인한다.
2. [[20_개발/코드베이스-현황]]에서 실제 서버와 웹의 구현 상태를 확인한다.
3. [[10_제품/전략/문제해결-매핑]]에서 어떤 문제를 어떤 기능으로 풀지 다시 확인한다.

즉, 템플릿은 기준 문서를 요약해서 제출형 서술로 바꾸는 자리다. 기준을 새로
만들거나 덮어쓰는 자리가 아니다.

## 3. 작성 원칙

실제 내용을 채울 때는 아래 원칙을 따른다.

- 이미 구현된 항목과 계획 중 항목을 분리한다.
- 날짜가 불확실하면 범위나 조건으로 적고, 확정처럼 쓰지 않는다.
- 제출용 표현이더라도 현재 코드베이스와 충돌하지 않게 적는다.
- 각 마일스톤은 사용자 가치와 완료 기준을 함께 적는다.
