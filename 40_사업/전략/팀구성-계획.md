# 팀구성 계획 템플릿

이 문서는 대표자 중심 팀의 역할과 확보 계획을 제출 문서 형식으로 정리할 때
쓰는 템플릿이다. 현재 실제 조직도, 확정 채용 계획, 운영 체계의 SSOT가 아니며,
빈 항목과 예시 구조를 현재 팀 상태로 읽으면 안 된다.

현재 실행 맥락과 사업 우선순위는 [[40_사업/전략/시장진입-실행계획]]을 먼저
따른다. 구현 관점의 실제 작업 상태는 [[20_개발/코드베이스-현황]]에서 확인하는
편이 맞다.

## 1. 문서 역할

이 템플릿은 아래 상황에서만 사용한다.

- 지원사업 신청서의 팀 구성 항목 작성
- 발표 자료의 팀 리스크 설명 구조 정리
- 내부 메모 수준에서 역할 공백을 빠르게 적어 보는 초안 작성

반대로 아래 용도로 사용하면 안 된다.

- 현재 조직 구조의 공식 원문
- 실제 채용 확정안 또는 계약 상태 기록
- 운영 회의나 역할 배분의 실시간 관리 문서

## 2. 작성 전에 먼저 확인할 기준

이 템플릿을 채우기 전에 아래 문서를 먼저 확인한다.

1. [[40_사업/전략/시장진입-실행계획]]에서 지금 필요한 실행 우선순위를 확인한다.
2. [[20_개발/코드베이스-현황]]에서 실제 구현과 운영 부담이 어디에 몰려 있는지 확인한다.

즉, 이 문서는 실제 운영 상태를 증빙용 구조로 정리하는 자리이지, 없는 인력을
있는 것처럼 보이게 만드는 문서가 아니다.

## 3. 작성 원칙

실제 내용을 채울 때는 아래 원칙을 따른다.

- 현재 있는 역할과 필요한 역할을 분리한다.
- 채용, 외주, 파트너 계획은 확정도와 시점을 함께 적는다.
- 사람 이름보다 역할과 책임을 먼저 적는다.
- 제출용 문구라도 현재 실행 역량과 맞지 않는 표현은 피한다.
