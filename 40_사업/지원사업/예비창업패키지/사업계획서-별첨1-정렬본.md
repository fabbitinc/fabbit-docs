## 1. 문제 인식 (Problem)\_창업 아이템의 필요성

중소 제조업의 핵심 문제는 설계와 생산 데이터가 서로 연결되지 않는다는
점이다. 도면, BOM, 작업지시, 생산실적이 각각 다른 파일과 담당자에게
흩어져 있어 검색과 변경 반영, 품질 역추적이 어렵고, 그 결과 납기 지연과
재작업, 커뮤니케이션 비용이 반복된다.

### 1-1. 국내 시장 현황과 문제점

- 국내 제조업 537,580개 사업체 중 10~299명 규모 중소 제조기업은 77,389개로
  전체의 14.4%다.
- 이 구간의 제조기업 다수는 PLM 없이 엑셀, 공유폴더, 메일로 도면과 BOM을
  관리한다. 생산 데이터도 수기 또는 단순 엑셀 관리에 머물러 설계 정보와
  연결되지 않는다.
- 소기업의 스마트공장 도입률은 일부 확대되었지만, 실제 현장에서는
  도면·BOM 관리와 생산실적 관리가 분리되어 있다.
- 결과적으로 최신 도면 확인, BOM 변경 반영, 불량 원인 역추적 같은 핵심
  업무가 사람 기억과 수작업에 의존한다.

### 1-2. 현장에서 발생하는 손실

| 구분           | 현장 문제               | 손실                                                                |
| -------------- | ----------------------- | ------------------------------------------------------------------- |
| 설계           | 도면 검색·버전 혼선     | 최신본 확인이 늦어지고 커뮤니케이션 비용이 증가                     |
| 설계           | BOM 수동 입력           | 엔지니어 시간이 반복 입력 업무에 소모됨                             |
| 설계/생산 연결 | 설계 변경 반영 지연     | 현장 작업지시서와 실제 설계 정보가 어긋남                           |
| 생산           | 불량 원인 역추적 어려움 | 어느 부품, 어느 버전, 어느 작업에서 문제가 발생했는지 추적이 늦어짐 |

도면 검색 시간 증가, BOM 엑셀 수작업, 설계 변경의 현장 미반영은 모두
하나의 공통 원인에서 발생한다. 설계와 생산 데이터가 구조화되지 않은 채
따로 관리되고 있기 때문이다.

### 1-3. 고객 검증 결과

현재까지 콜드콜 36건+와 산업 컨퍼런스 현장 접점을 통해 다음을 확인했다.

| 검증 항목   | 확인 결과                                | 시사점                                                         |
| ----------- | ---------------------------------------- | -------------------------------------------------------------- |
| 타깃 재설정 | 1~9인 영세기업은 관리 체계 수요가 낮음   | 초기 타깃은 10~49인, 매출 10억+ 제조기업이 적합                |
| 문제 표현   | `PLM` 용어 인지도는 낮음                 | `도면·BOM 관리`, `변경 반영`, `생산 연결`로 설명해야 반응이 옴 |
| 반복 pain   | 도면 검색, BOM 수동 입력, 변경 반영 누락 | 제품의 우선 기능은 AI 추출과 구조화 BOM, 경량 MES 연결이 적합  |

### 1-4. 왜 지금 이 아이템이 필요한가

1. 제조 현장의 숙련 인력 고령화로 암묵지가 빠르게 사라지고 있다.
2. 공급망 변동과 납기 압박이 커져 변경관리와 추적성이 중요해졌다.
3. AI 활용 수요는 늘고 있지만, 구조화된 제조 데이터가 없어 현장 적용이
   어렵다.
4. 기존 PLM과 MES는 각각 무겁고 비싸며, AI 추출 도구는 추출 이후의
   관리와 생산 연결까지 제공하지 못한다.

따라서 중소 제조기업에는 `빠르게 도입 가능한 AI 진입점`과 `가볍게 연결된
설계·생산 관리 구조`가 함께 필요하다.

## 2. 실현 가능성 (Solution)\_창업 아이템의 개발 계획

Fabbit의 개발 계획은 `AI 도면 추출 정확도 개선`, `구조화 BOM 데이터 관리
안정화`, `파일럿 3개사 확보`, `유료 전환 1~2개사`, `경량 MES MVP 구현`에
집중한다. 협약기간 내 바로 검증 가능한 목표에 집중하고, 온톨로지 기반
고도화 AI는 차년도 확장 전략으로 둔다.

### 2-1. 개발 또는 구체화 계획

| 개발 영역        | 협약기간 내 목표                                  | 고객 가치                                    |
| ---------------- | ------------------------------------------------- | -------------------------------------------- |
| AI 도면·BOM 추출 | 표제란 및 부품표 인식 정확도 개선, 등록 속도 개선 | 수동 입력 시간을 크게 줄임                   |
| 구조화 BOM 관리  | 계층형 BOM, 변경이력, Where-used 안정화           | 최신본 관리와 변경 반영 속도 향상            |
| 경량 MES         | 작업지시, 생산실적, 품질추적 MVP 구현             | 설계 변경과 생산 결과를 하나의 흐름으로 연결 |
| 파일럿/유료 전환 | 파일럿 3개사, 유료 전환 1~2개사 확보              | 실제 사용성과 지불 의사 검증                 |

현재 AI 도면·BOM 추출, 구조화 BOM 관리, 변경관리, 웹 기반 제품 구조는 이미
구현되어 있다. 협약기간 동안에는 기능의 범위를 넓히기보다 정확도, 사용성,
현장 적용성, 유료화 가능성 검증에 집중한다.

### 2-2. 차별성 및 경쟁력 확보 전략

| 비교 항목       | AI 도면 도구           | 기존 PLM | 통합형 제조 SW     | Fabbit          |
| --------------- | ---------------------- | -------- | ------------------ | --------------- |
| AI 자동 추출    | 일부 지원              | 미지원   | 미지원 또는 제한적 | 지원            |
| 구조화 BOM 관리 | 추출 후 외부 문서 의존 | 지원     | 일부 지원          | 지원            |
| 설계-생산 연결  | 미지원                 | 약함     | 지원               | 경량 MES로 연결 |
| 도입 장벽       | 낮음                   | 높음     | 높음               | 낮음            |

Fabbit의 핵심 차별성은 `AI로 빠르게 시작하고`, `구조화 데이터로 정착시키고`,
`경량 MES로 현장 연결까지 이어간다`는 점이다. 즉, 추출만 잘하는 도구가
아니라 설계와 생산을 한 흐름으로 묶는 제품으로 발전시키는 전략이다.

## 3. 성장전략 (Scale-up)\_사업화 추진 전략

Fabbit은 `무료 진입 → 파일럿 검증 → 유료 전환 → 바우처 및 파트너 확장`
순서로 성장한다. 초기에는 제품 완성도보다 빠른 도입성과 현장 적합성을
앞세워 10~49인 제조기업 시장을 공략한다.

### 3-1. 경쟁제품·경쟁사 분석

| 구분           | 강점                | 한계                     | Fabbit의 대응                        |
| -------------- | ------------------- | ------------------------ | ------------------------------------ |
| AI 도면 도구   | 빠른 추출           | 추출 후 관리 기능이 약함 | 추출 결과를 구조화 BOM으로 바로 연결 |
| 기존 PLM       | 체계적 변경관리     | 무겁고 도입 장벽이 높음  | 중소 제조기업용 경량 구조 제공       |
| 통합형 제조 SW | 설계·생산 연결 가능 | 초기 도입 복잡성 높음    | AI 추출 기반으로 도입 진입점 제공    |

### 3-2. 목표 시장과 진입 전략

## 4. 팀 구성 (Team)\_대표자 및 팀원 구성 계획

대표자는 PLM 도메인 이해와 풀스택 개발 역량을 바탕으로 창업 아이템의
구현과 운영을 주도한다. 팀원은 고객 접점과 기술지원을 담당하며, 협약기간
중 프론트엔드 개발 인력을 추가 확보해 제품 고도화 속도를 높일 계획이다.

### 4-1. 대표자 보유 역량

- 컴퓨터공학 전공
- PLM 솔루션 기업에서 BOM 관리 모듈 개발 경험 보유
- SI 기업에서 프론트엔드, 백엔드, 인프라를 아우르는 풀스택 개발 경험 보유
- 8명 규모 프로젝트의 기획, 실행, 일정 조율 경험 보유
- 현재 AI 도면·BOM 추출, 구조화 BOM 관리, 웹 서비스 구조를 직접 구현한
  실행력 보유

대표자는 중소 제조업 PLM 도입의 어려움을 공급자 관점에서 직접 경험했고,
`빠르게 도입 가능한 구조화 데이터 플랫폼`이 필요하다는 문제의식에서
Fabbit을 시작했다.
