## 1. 문제 인식 (Problem)_창업 아이템의 필요성

이 섹션은 왜 지금 이 문제를 풀어야 하는지, 그리고 왜 기존 방식으로는
해결되지 않는지를 설명한다.

### 1-1. 국내 시장 현황과 문제점

국내 제조업 537,580개 사업체 중 10~299명 규모 중소기업은 77,389개
수준이다. 이 구간의 제조기업은 ERP를 일부 보유하고 있어도 도면과 BOM은
여전히 엑셀, 공유폴더, NAS, 메신저로 관리하는 경우가 많다. 저장 위치는
점차 중앙화되지만, 최신본 식별, 변경 승인, 배포 통제, 리비전 추적은
여전히 사람 기억과 폴더 구조에 의존한다.

- 현장 일반 상황: ERP는 있으나 도면과 BOM은 별도 폴더·엑셀 관리
- 저장 방식 변화: `개인 PC / 엑셀 -> 공유폴더 -> NAS`
- 반복 문제: 최신본 혼선, 변경 반영 누락, 승인·배포 이력 부재
- 실제 검토 시점: 저장은 되지만 관리가 안 되는 단계에서 도면관리 솔루션 검토

```text
개인PC / 엑셀
      ↓
공유폴더
      ↓
NAS / 서버 저장
      ↓
최신본 혼선, 리비전 추적 한계, 변경 반영 누락
      ↓
도면·BOM 관리 솔루션 검토
```

- {그래프 설명: 저장 위치는 개인PC에서 공유폴더, NAS로 이동하지만, 관리
  문제는 해결되지 않는 흐름을 세로형 화살표로 표현}

### 1-2. 현장에서 발생하는 손실

문제는 단순 불편이 아니라 시간 손실, 커뮤니케이션 비용, 운영 리스크로
이어진다.

| 구분 | 현장 문제 | 발생 손실 |
| --- | --- | --- |
| 설계 | 도면 검색·버전 혼선 | 최신본 확인 지연, 검토 커뮤니케이션 비용 증가 |
| 설계 | BOM 수동 입력 및 정리 | 반복 입력 업무 증가, 엔지니어 시간 소모 |
| 설계/생산 연결 | 설계 변경 반영 지연 | 작업지시서와 실제 설계 정보 불일치 |
| 생산 | 불량 원인 역추적 어려움 | 어느 부품·버전·작업에서 문제가 발생했는지 확인 지연 |

핵심 문제는 개별 기능 부족보다 `기존 방식에서 시스템으로 넘어오는 과정`
자체의 마찰 비용이 높다는 점이다. 기존 솔루션은 기능은 충분하지만, 자사
양식과 속성 체계를 재정리해야 하는 부담 때문에 초기 도입 피로도가 높다.

### 1-3. 고객 검증 결과와 시급성

현재까지 콜드콜 36건 이상, 산업 컨퍼런스 현장 영업, 후속 인터뷰 2건을
통해 초기 고객 문제를 확인했다.

| 검증 항목 | 확인 결과 | 시사점 |
| --- | --- | --- |
| 타깃 고객 | 1~9인 영세기업보다 10~49인 제조기업의 문제의식이 더 명확 | 초기 타깃은 10~49인, 매출 10억 이상 제조기업 |
| 문제 표현 | `PLM`보다 `도면관리`, `변경 반영`, `최신본 관리` 반응이 더 큼 | 제품 메시지는 도메인 용어보다 현장 문제 중심 구성 필요 |
| 반복 pain | 도면 최신본 관리, 리비전 추적, 변경 승인·배포 프로세스 부재 | 우선 기능은 도면·BOM 관리와 마이그레이션 부담 완화 |
| 고객 반응 | 후속 인터뷰 2건 모두 POC 참여 의향 확인 | 문제 인식이 추상적 공감 수준을 넘어 실제 검토 단계 진입 |

왜 지금 필요한지는 다음 세 가지로 요약된다.

1. 설계 변경 빈도와 공급망 변동 증가로 변경관리 중요성 확대
2. 현장 숙련 인력 의존 구조 심화로 지식 유실 위험 확대
3. AI 활용 수요는 높아졌지만, 구조화된 제조 데이터 기반은 부족

## 2. 실현 가능성 (Solution)_창업 아이템의 개발 계획

이 섹션은 현재 준비 수준과 협약기간 내 무엇을 실제로 구현·검증할지
설명한다.

### 2-1. 현재 개발 현황과 준비 수준

Fabbit은 아이디어 단계가 아니라 시제품(MVP) 단계에 있다. 대표와 팀원이
함께 제품 기획, 서버, 웹, 운영 환경을 구축했고, 실제 고객 문제 검증도
병행하고 있다.

| 항목 | 현황 |
| --- | --- |
| 제품 단계 | MVP 개발 완료, 시제품 단계 |
| 제품 상태 | 핵심 기능 구현 완료 |
| 개발 방식 | 대표·팀원 협업 기반 풀스택 개발 |
| 검증 현황 | 콜드콜 36건+, 산업 컨퍼런스 현장 영업, 후속 인터뷰 2건 |
| 고객 반응 | POC 참여 의향 2건 확인, 현장 방문 및 파일럿 협의 진행 중 |

현재 구현 완료 기능은 다음과 같다.

| 기능 영역 | 구현 상태 | 설명 |
| --- | --- | --- |
| 도면 관리 | 완료 | 도면 등록, 조회, 파일 연계 기반 관리 |
| 부품관리(부품 마스터) | 완료 | 품번, 품명, 규격 등 부품 기준정보 관리 |
| 승인·결재 | 완료 | 검토·승인 흐름 반영 가능 구조 |
| 리비전 상태 관리 | 완료 | 버전 및 상태값 구분 관리 |
| 변경 이력 관리 | 완료 | 변경 내역 추적 및 확인 기능 |
| BOM 엑셀 업로드 | 완료 | 기존 엑셀 양식 기반 등록 지원 |
| AI 기반 필드 매핑 | 완료 | BOM 헤더·샘플 데이터 분석 후 시스템 필드 자동 연결 |
| 매핑 규칙 재사용 | 완료 | 동일 양식 업로드 시 기존 매핑 재적용 |

### 2-2. 차별성 및 경쟁력 확보 전략

Fabbit의 핵심 차별성은 `기존 엑셀 구조를 최대한 유지한 채 시스템으로
전환`시킨다는 점이다. 경쟁 구도는 단순히 기존 PLM과의 비교가 아니라,
현재 고객이 버티고 있는 방식 전체와의 비교다.

| 구분 | 엑셀 / 공유폴더 / NAS | AI 추출 도구 | 기존 PLM / 도면관리 | Fabbit |
| --- | --- | --- | --- | --- |
| 초기 사용성 | 익숙하지만 관리 한계 큼 | 일부 자동화 가능 | 초기 셋업과 교육 부담 큼 | 기존 방식과 유사한 진입 흐름 제공 |
| 데이터 입력 | 수동 정리 | 추출 후 별도 정리 필요 | 수동 등록 또는 자사 구조 적응 필요 | 기존 엑셀 속성을 AI로 해석해 시스템 필드에 매핑 |
| 변경관리 | 사실상 없음 | 없음 | 제공 | 리비전·이력 관리 기반 제공 |
| 마이그레이션 | 해당 없음 | 제한적 | 자사 체계 적응 필요 | 기존 속성 체계를 최대한 유지하는 매핑 기반 도입 |
| 목표 포지션 | 현재 버티는 방식 | 보조 도구 | 정식 운영 시스템 | 엑셀/NAS 단계에서 정식 운영 시스템으로 넘어가는 진입 제품 |

핵심 동작 흐름은 아래와 같다.

```text
1. BOM 엑셀 업로드
        ↓
2. AI가 헤더·샘플 데이터 해석
        ↓
3. 시스템 필드 매핑 규칙 자동 생성
        ↓
4. 사용자 검수 후 자동 적재
        ↓
5. 동일 양식 반복 업로드
```

- {플로우 설명: 고객이 기존 양식을 바꾸지 않고 업로드하면 AI가 필드 매핑을
  생성하고, 사용자는 한 번만 검수하면 동일 양식을 반복 사용할 수 있는 구조}

### 2-3. 사업 추진 일정(협약기간 내)

예비창업패키지 공고 기준 협약기간은 2026년 6월부터 2027년 1월까지 약
8개월이다. 이 기간에는 기능 폭을 무리하게 넓히기보다 정확도, 사용성,
파일럿 운영, 유료 전환 검증에 집중한다.

| 구분 | 추진 내용 | 추진 기간 | 세부 내용 |

## 3. 성장전략 (Scale-up)_사업화 추진 전략

이 섹션은 제품이 어떻게 팔리고, 어떤 채널로 확장되며, 이후 어떤 단계로
성장하는지 설명한다.

### 3-1. 경쟁제품·경쟁사 분석

초기 경쟁 구도는 대형 PLM과의 기능 비교보다 `지금 고객이 쓰는 방식`과의
비교가 더 중요하다.

| 구분 | 강점 | 한계 | Fabbit의 대응 |
| --- | --- | --- | --- |
| 엑셀 / 공유폴더 / NAS | 익숙하고 당장 비용이 적음 | 최신본 관리, 변경 추적, 데이터 재사용 한계 | 기존 방식과 유사한 진입 흐름 제공 |
| AI 추출 도구 | 빠른 추출 | 추출 후 관리 기능 약함 | 추출 결과를 시스템 데이터로 바로 연결 |
| 기존 PLM / 도면관리 | 체계적 변경관리 | 무겁고 도입 장벽 높음 | 중소 제조기업용 경량 구조 제공 |

### 3-2. 목표 시장과 진입 전략

초기 목표 시장은 종사자 10~49명, 매출 10억원 이상, 금속가공·기계장비·
전기장비 업종 제조기업이다. 이 구간은 도면과 BOM을 실제로 사용하지만,
전용 PLM과 MES를 각각 도입하기에는 비용과 운영 부담이 큰 집단이다.

| 구분 | 기업 수 | 금액 | 산정 방식 |
| --- | --- | --- | --- |
| TAM | 약 290,000개사 | 약 6,970억원 | 도면·BOM 사용 가능 제조업 전반 × 연 240만원 |
| SAM | 약 32,600개사 | 약 783억원 | 10~299명 규모이면서 도면·BOM 관리 문제와 SaaS 도입 여력을 갖는 초기 목표시장 |
| SOM | 약 80개사(3년 누적 유료) | 약 3.4억원 | Starter 800개사 × 전환율 10% 기준 bottom-up 추정 |

초기 진입 전략은 다음과 같다.

- 시장 진입 포지셔닝: `AI 기능이 많은 도구`보다 `기존 엑셀에서 옮기기 쉬운 마이그레이션 도구`
- 고객 확보 전략: 콜드콜, 산업 전시회, 제조업 네트워크, 자문 소개 기반 직접 공략
- 전환 전략: 무료 Starter와 파일럿 기반 실제 데이터 등록 유도
- 확장 전략: 운영 데이터 AI 고도화, 동아시아 제조업 확장 검토, 이후 ERP·MES·PLC 연동 확장

### 3-3. 비즈니스 모델(수익화 모델)

Fabbit은 `무료 진입 -> 실제 데이터 등록 -> 파일럿 운영 -> 유료 전환` 구조의
회사 단위 Freemium 모델을 채택한다.

| 플랜 | 월 요금 | 대상 |
| --- | --- | --- |
| Starter | 무료 | 도입 검토 및 소규모 팀 |
| Team | 24.9만원 | 10~20명 규모 초기 운영팀 |
| Business | 59.9만원 | 부서 간 협업이 필요한 운영팀 |
| Enterprise | 별도 협의 | 커스텀 연동 및 대용량 운영 |

```text
Starter 가입
    ↓
실제 데이터 등록
    ↓
파일럿 운영
    ↓
Team / Business 유료 전환
```

- {그래프 설명: 퍼널 또는 단계형 화살표 그래프로 표현. `무료 진입 ->
  실사용 개시 -> 현장 검증 -> 정식 운영 계약` 순서로 배치}

### 3-4. 투자유치(자금확보) 전략

초기에는 정부지원사업과 자체 자금을 결합해 제품 고도화와 파일럿 검증을
완수하고, 이후 유료 전환을 기반으로 후속 자금을 검토한다.

| 시기 | 자금원 | 용도 |

| 2026 하반기 | 후속 창업지원사업 | 파일럿 확대, 특허, 초기 영업 확장 |
