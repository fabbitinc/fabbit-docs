## 1. 문제인식

> 평가: "창업과제 배경 및 필요성, 목표시장(고객) 현황 분석"

### 해결하려는 문제

**중소 제조업의 제조 데이터가 엑셀·폴더·담당자 머릿속에 갇혀 있어, 검색·변경추적·의사결정이 불가능하다.**

국내 제조업 537,580개 사업체 중 10~299명 규모의 중소기업은 77,389개(14.4%)이며 (통계청 전국사업체조사, 2023), 이 중 **95.6%가 PLM(제품 수명주기 관리) 없이** 도면과 BOM(부품표)을 관리하고 있다 (KDI 스마트공장 보고서, 2014~2017 지원기업 PLM 도입 비중 4.4%). 도면은 폴더에, BOM은 엑셀에, 공급사 정보는 담당자 머릿속에 흩어져 있다.

| 현장 문제        | 정량 근거                                                                                                                                        |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 도면/BOM 수동 입력 | 엔지니어 1인당 연간 52~260시간 엑셀 BOM 작업에 소요 (OpenBOM, "The Real Cost of Managing BOMs in Spreadsheets")                                               |
| 도면 검색·버전 혼선  | 데이터 전문가가 주당 업무시간의 **30%를 데이터 검색·준비**에 소비, 20%를 중복 작업에 소비 (IDC 연구, Siemens 인용). 지식노동자 기준 25~30% (AIIM; McKinsey). 제조 엔지니어도 도면·부품 정보 검색에 동일 패턴 |
| BOM 오류 → 재발주 | BOM 오류가 전체 설계변경(ECO)의 최대 30% 차지, 건당 행정비 $1,500~3,000 (Aberdeen Group, CADTALK·OpenBOM 재인용)                                                   |
| 변경 영향 파악 불가  | 부품 하나를 바꿔도 어떤 제품·공급사·공정에 영향이 가는지 추적할 수 없음                                                                                                    |

정부 스마트공장 실태조사 (중기부, 2024.10~2025.01)에서 ERP(76.3%), MES(14.4%)는 도입률을 집계하지만, **PLM은 조사 항목 자체에 포함되지 않는다.** 제조 데이터 관리가 시스템이 아닌 사람에게 의존하고 있으며, 이 데이터가 구조화되지 않는 한 AI를 활용한 의사결정 지원도 불가능하다.

### 문제의 시급성

- **제조 현장 고령화**: 숙련공 퇴직 시 머릿속 지식(어떤 부품이 어디에 쓰이는지, 어떤 공급사가 신뢰할 수 있는지)이 함께 사라진다. 데이터가 시스템에 없으면 복구 불가.
- **공급망 불안정**: 미중 갈등(관세·수출규제), 소재 가격 변동으로 부품 대체·공급사 전환이 빈번해졌으나, 변경 영향을 추적할 수 없어 대응이 늦다.
- **AI 도입 전제조건 미충족**: OECD 기준 기업 AI 사용률이 급증(2023년 8.7% → 2025년 20.2%, OECD 통계)하지만, 국내 제조업 AI 도입률은 3.9% (한국생산성본부 산업디지털전환실태조사, 2024)에 불과하다. 데이터가 구조화되지 않으면 AI는 쓸 수 없다.

### 착안 배경

PLM 솔루션 기업에서 BOM 관리 모듈을 직접 개발하며, 중소 제조업
현장의 문제를 목격했다. 고객은 "빠른 도입과 쉬운 사용"을 원했지만,
고객사별 커스텀 개발이 반복되면서 제품은 복잡해지고 도입 장벽은
높아졌다. 근본 원인은 **데이터가 구조화되지 않은 상태에서 관리
기능만 얹으려 한 것**이었다.

2026년 3월 13일 진행한 PLM 업계 자문 인터뷰에서도 고객이 가장 많이
호소하는 문제는 `도면관리`, 특히 **최신 버전 관리와 검토·승인·배포
프로세스 부재**라는 점이 다시 확인됐다. Fabbit은 이 핵심 문제를 해결하는
가장 빠른 진입점으로 `AI 기반 초기 데이터화 + 구조화 BOM 관리`를
선택했다.

AI 도면 분석 스타트업(비링커)이 Seed+Pre-A 누적 투자 10억 이상을
유치하고 글로벌 팁스에 선정(3년 최대 6억)되며 **AI로 제조 데이터를
추출하는 시장의 존재를 입증**했다. 그러나 추출된 데이터를 실제 운영
레이어 안에서 구조화하고, 최신본 관리와 BOM 정합성까지 연결해 주는
영역은 여전히 비어 있다. Fabbit은 이 틈새를 공략한다.

> 평가: "창업과제 현황, 과제 실현 및 구체화 방안"

### 현재 개발 단계

| 프론트엔드  | 대시보드, 부품/BOM, 이슈/변경요청, 조직관리 등 구현                         |

{이미지 첨부 - 제품 스크린샷 2~3장: (1) 도면 AI 인식 결과 화면, (2) BOM 트리 관리 화면, (3) 대시보드 또는 이슈/변경요청 화면}

### 핵심 기술 및 경쟁 우위

#### 1단계 — AI 자동 추출로 데이터화 (진입 장벽 해소)

중소 제조업이 시스템을 도입하지 못하는 가장 큰 이유는 **기존 데이터를
옮기는 비용**이다. Fabbit은 AI가 이 과정을 대신한다.

- **BOM 자동 분석**: 도면(DWG/PDF) 업로드 시 LLM 기반 AI가 부품표를
  자동 인식하여 품명, 재질, 수량, 규격을 추출하고 시스템 안에 구조화
  형태로 등록한다.
- **도면 자동 인식 (고도화 예정)**: 표제란(Title Block) 인식, 도면
  간 참조 관계 추출을 고도화한다.
- 기존 방식: 엔지니어가 도면을 보며 Excel에 수동 입력한다. Fabbit은
  업로드 후 초안 구조화 결과를 수분 내 확인하는 흐름을 목표로 한다.

#### 2단계 — 구조화 BOM 관리 (데이터 축적)

AI가 추출한 데이터를 Excel로 내보내는 데 그치지 않고, **시스템 안에서
계층형 BOM 트리(Assembly-Sub Assembly-Part)로 관리**한다.
Where-used(역추적), 리비전 추적, 도면-BOM 연계 검색이 가능하다.

#### 3단계 — 연결형 제조 데이터 분석 기반 (중장기 방향)

> **협약 6개월 내 목표는 1~2단계(AI 추출 고도화 + 구조화 BOM 관리)에
> 집중**한다. 3단계는 협약 이후 검증할 중장기 방향이다.

축적된 부품, 공급사, 담당자, 도면 데이터를 연결하면 단순 저장을 넘어
변경 영향, 연관 도면, 유사 부품 탐색 같은 운영 지원 기능을 만들 수
있다. 여기서 `온톨로지`나 `그래프 DB`는 고객에게 직접 보여 주는 기능
이름이 아니라, 내부 데이터 구조를 정리하기 위한 기술 선택지다.

- **변경 영향 확인**: 특정 부품 변경 시 영향받는 제품과 관련 대상을 더
  빠르게 파악한다.
- **연관 정보 탐색**: 도면, BOM, 담당자, 공급사 정보를 한 흐름에서
  조회한다.
- **운영 지원 확장**: 축적된 데이터를 바탕으로 검색, 알림, 질의응답
  기능으로 확장한다.

### 현장 검증에서 확인한 적합성

- 2026년 3월 13일 PLM 업계 자문 인터뷰에서 고객이 가장 많이 호소하는
  문제는 `도면관리`, 특히 최신 버전 관리와 검토, 승인, 배포 프로세스
  부재라는 점이 확인됐다.
- 2차 후속 인터뷰 2건에서는 모두 POC 참여 의향을 밝혔고, 현장 방문
  일정을 협의 중이다.
- 현장에서는 `온톨로지` 같은 기술 용어보다 `쉽게 시작할 수 있는 도면·
  BOM 관리`, `최신본 표시`, `구조화 조회`에 먼저 반응했다.
- 따라서 협약 기간에는 큰 비전보다 **쉬운 도입, 실제 파일럿, 유료 전환
  가능성 검증**에 집중한다.

### 기술 확보 방안

- **AI 추출**: 범용 LLM API와 OCR, 전처리, 후처리 로직을 조합해 도면과
  부품표 추출 파이프라인을 구성한다. 자체 모델 학습보다 초기 비용이
  낮고 실행 속도가 빠르다.
- **도메인 파이프라인**: 대표가 제조 소프트웨어 기업에서 BOM 모듈을
  개발한 경험을 바탕으로, 도면 업로드 → AI 추출 → 구조화 BOM 연결
  흐름을 직접 설계하고 구현한다.
- **데이터 축적**: 파일럿 고객의 실제 도면과 BOM 데이터가 쌓이면,
  추출 정확도 개선과 업종별 템플릿 확보에 활용할 수 있다.

## 4. 팀 구성

> 평가: "기업 구성 및 보유 역량, 사회적 가치 도입"

### 대표자 전문성

컴퓨터공학 전공 후 **PLM 솔루션 기업**에서 BOM 관리 모듈을 개발하며 제조업 도메인 지식을 확보했다. 이후 SI 기업에서 프론트엔드/백엔드/인프라를 아우르는 풀스택 역량을 쌓았으며, 8명 규모 프로젝트의 기획/실행/일정 관리를 주도한 경험이 있다.

PLM 도메인 이해와 풀스택 개발 역량을 겸비하여, **MVP 설계부터 구현/운영까지 1인 실행이 가능**하다. 현재 도면·BOM AI 추출, PLM 코어 기능, 프론트엔드, 인프라를 단독 구축한 상태이다.

### 팀 구성

대표가 제품 설계와 개발 전체를 담당하고, 팀원이 영업과 고객 접점을
담당하는 **제품-시장 2인 체제**다.

| 역할 | 담당업무            | 핵심 경험                                                                        |
| ---- | ------------------- | -------------------------------------------------------------------------------- |
| 대표 | 제품 설계/개발 전체 | 컴퓨터공학 전공, PLM 솔루션 기업 BOM 모듈 개발, SI 프로젝트 풀스택 개발·관리 경험 |
| 팀원 | 영업/기술지원       | DB 모니터링 제품 기술지원 경력, 콜드콜 36건+, 산업 컨퍼런스 현장 영업 실행        |

### 자문 네트워크

| 자문 영역        | 커버리지                                                      |
| ---------------- | ------------------------------------------------------------- |
| PLM 도메인/업계  | PLM 업계 대표급 자문 1인, 2026-03-13 인터뷰 완료             |
| 제조 현장/수요자 | 제조업 품질 실무자 관점 피드백 채널 확보                     |
| B2B SaaS GTM     | 클라우드·B2B SaaS GTM 자문 채널 확보                         |

> 팀에 제조업 현장 경력자가 없는 약점은 외부 자문과 파일럿 고객 피드백으로
> 보완하고, 향후 도메인 전문가 영입을 검토한다.

### 사회적 가치 도입

1. **중소 제조업 디지털 격차 해소**: PLM 도입률 4.4%인 중소 제조업에
   AI 자동 추출과 무료 Starter 플랜으로 도입 장벽을 낮춰, 대기업과
   중소기업 간 제조 데이터 관리 격차를 줄인다. 협약 기간 내 Starter
   가입 기업 **10개사 이상**을 목표로 한다.
2. **제조 현장 지식 보존**: 숙련공 퇴직 시 사라지는 암묵지(부품 이력,
   공급사 노하우)를 시스템에 구조화해 조직 지식으로 보존한다. 파일럿
   기업의 도면과 BOM 데이터 **500건 이상** 등록을 목표로 한다.
3. **환경 부담 완화**: 최신본 혼선과 BOM 오류를 줄여 재작업과 불필요한
   출력, 폐기 비용을 줄이는 방향으로 운영한다.
4. **운영 투명성 강화**: 변경 이력과 권한 기반 접근 구조를 마련해 제조
   데이터 관리의 책임성과 추적 가능성을 높인다.
