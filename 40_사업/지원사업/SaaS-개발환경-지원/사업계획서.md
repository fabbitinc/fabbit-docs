## [양식 2] SaaS 개발환경 지원 사업계획서

- 서비스 필요성, 기능, 시장성, 사업화 전략, 팀 역량을 정리

### 서비스(솔루션) 명

- Fabbit: AI 기반 제조 데이터 온톨로지 SaaS

### 서비스 요약

- 중소 제조업의 도면, BOM, 변경이력 데이터를 구조화 관리하는 웹 기반 제조 데이터 SaaS
- 데이터 마이그레이션 부담을 AI로 줄여 기존 엑셀 양식과 폴더 기반 운영을 유지한 채 바로 사용할 수 있도록 설계
- 무료 Starter 플랜으로 초기 도입 장벽을 낮추고 운영 정착 후 좌석 기반 유료 전환으로 확장
- 중장기적으로는 제조 데이터를 온톨로지로 연결해 변경 영향 분석, 리스크 탐지, AI 의사결정 지원까지 확장

### 현재 수행 중인 정부지원 과제

- 없음

### 1. 서비스 개요

- 중소 제조업의 도면·BOM을 엑셀·폴더 대신 SaaS로 통합 관리
- ERP 도입률 76.3%이나 PLM은 조사 항목에조차 미포함 → 설계 데이터 관리 공백
- 최대 장벽은 데이터 마이그레이션 → Fabbit은 AI가 기존 엑셀 양식을 시스템에 자동 매핑

**클라우드 지원이 필요한 이유**

- 현재: 단일 서버 + 오픈소스(AGE, 외부 LLM API)로 MVP 개발 완료
- 한계: 2인 팀이 직접 운영·백업·스케일링 → 상용 수준 안정성 확보 어려움
- 필요: 매니지드 그래프DB(Neptune) + AI 서비스(Bedrock)로 전환해야 상용화 가능
- 예비창업자로 자체 인프라 투자 여력 없음 → 바우처 지원이 필수

### 2. 서비스의 용도, 특징 및 주요 기능

- 설계 데이터 관리와 제조 현장 협업을 위한 경량 제조 데이터 플랫폼 지향

| 구분      | 내용                                                                       |
| --------- | -------------------------------------------------------------------------- |
| 용도      | 도면, BOM, 변경이력, 부품 정보를 구조화 관리하고 협업하는 제조 데이터 SaaS |
| 주요 대상 | 종사자 10~299명 제조기업 중 도면 기반 업무를 수행하는 중소 제조업          |
| 제공 방식 | 클라우드 기반 웹 서비스                                                    |

**구현 완료 (MVP)**

| 기능                | 설명                                                            |
| ------------------- | --------------------------------------------------------------- |
| AI BOM 자동 매핑    | 고객사별 BOM 엑셀 양식을 AI가 분석해 시스템 필드로 자동 매핑    |
| 부품 및 BOM 관리    | 부품 CRUD, 계층형 BOM 트리, 카테고리 분류, 기본 담당자 관리     |
| 변경관리 워크플로우 | 변경요청 생성, 제출, 리뷰, 병합, 종료까지 추적. 이슈 연동       |
| 프로젝트 협업       | 프로젝트별 부품, 멤버, 활동 이력 관리                           |
| 파일 연동 구조      | 기존 로컬 파일 관리 방식을 유지하면서 서비스와 연동 가능한 구조 |

**사업기간 내 개발 예정**

| 기능                    | 설명                                                                                 |
| ----------------------- | ------------------------------------------------------------------------------------ |
| AI 매핑 고도화          | BOM LLM 인식 정확도 개선, 다양한 엑셀 양식 대응력 확장                                |
| AI 자연어 인터페이스    | 자연어로 부품 검색, 관계 정보 조회, 이슈 생성 등 데이터 질의와 작업 실행을 통합 처리 |
| 온톨로지 기반 관계 분석 (사업기간 이후) | 부품, 도면, 공급사, 공정 간 관계를 그래프DB로 연결. 변경 영향 추적, 리스크 탐지. 사업기간 내 Neptune 환경 구축까지 진행, 본격 활용은 이후 |

- 현재 기술 스택: Python(FastAPI), React(TypeScript), PostgreSQL, Apache AGE
- 현재 구현 범위: API 150개 규모, 웹 라우트 29개

### 3. 경쟁 제품과의 차별성

- 경쟁 구도는 기존 대형 PLM보다 엑셀, 공유폴더, 단순 파일 관리 방식의 대체 시장에 가까움

| 구분                | 엑셀/공유폴더             | 기존 PLM                                      | Fabbit                                                      |
| ------------------- | ------------------------- | --------------------------------------------- | ----------------------------------------------------------- |
| 도입과 데이터 전환  | 전환 불필요하나 체계 부재 | 셋업, 교육, 데이터 이관 필요 (높은 진입 장벽) | AI가 기존 엑셀 양식을 시스템에 자동 매핑 (전환 비용 0)      |
| 변경관리            | 사실상 없음               | 제공되나 복잡하고 고가                        | 핵심 기능 중심 경량 제공                                    |
| 데이터 간 관계 분석 | 불가                      | 제한적 (모듈 분리, 커스터마이징 필요)         | 온톨로지 기반 그래프 구조로 부품-도면-공급사-공정 자동 연결 |

- 데이터가 쌓일수록 전환비용이 형성되어 장기 고객 유지력이 커짐
- 축적된 데이터를 온톨로지로 연결해 변경 영향 분석, 리스크 탐지, AI 의사결정 지원까지 확장 — 단순 문서 관리 도구와 구분되는 핵심 차별점

### 4. 시장 파급효과

- PLM 미도입 제조기업 95.6%에 무료 진입 경로 제공 → 중소 제조업 디지털 전환 촉진
- 엑셀·폴더에 갇힌 도면·BOM 데이터 구조화 → ERP 등 기존 시스템 연동 가능한 데이터 허브
- 제조 AI 도입률 0.1% 환경에서 BOM 자동 매핑이라는 실용적 AI 사례 확산
- 제조업 10인 이상 클라우드 이용률 73.9% 기반 위에 핵심 업무 영역 클라우드 확장

### 8. 경쟁상황 및 경쟁우위

- 시장 현황
  - 도면·BOM 관리 SaaS 급성장 (비링커 2년간 매출 10배 2.4→20억, 마이링크 2024년 600% 성장)
  - PLM 직접 경쟁 시장보다 엑셀·폴더 관리 대체 전환 시장 성격
  - Fabbit 포지션: 가장 쉽게 시작하는 도면·BOM 관리 SaaS
- 경쟁우위
  1. AI 마이그레이션 — 기존 BOM 엑셀 양식 유지, AI가 시스템 필드에 자동 매핑. 데이터 전환 비용 0
  2. 무료 진입 — Starter 무료 플랜으로 도입 장벽 제거, 역할별 좌석 과금으로 합리적 비용
  3. 데이터 축적 → 전환비용 — 같은 기능으로 시작하지만 데이터가 쌓이면 다른 제품. AI 매핑은 데이터를 빠르게 모으기 위한 진입 전략, 사용할수록 전환비용 형성
  4. 온톨로지 확장 — 중장기 핵심. 축적된 데이터를 그래프DB로 연결 → 부품 간 인과관계 분석, AI 의사결정 지원

### 현재 개발환경 및 핵심기술

**현재 개발환경**

- 백엔드: Spring Boot (Kotlin), PostgreSQL 16, Apache AGE(그래프DB 확장)
- 프론트엔드: React, TypeScript
- AI: 현재 OpenAI API로 BOM 매핑·자연어 질의 처리
- 인프라: AWS (EC2, RDS, S3, CloudFront), Docker, GitHub Actions CI/CD
- 구현 범위: API 150개 규모

**클라우드 전환 계획 (사업기간 내)**

| 현재 | 전환 대상 | 목적 |
|------|----------|------|
| EC2 단일 서버 | Fargate 컨테이너 | 운영 부담 절감, 오토스케일링 |
| OpenAI API (외부 호출) | Amazon Bedrock | 매니지드 LLM, 보안·비용 관리 |
| Apache AGE (직접 운영) | Amazon Neptune | 매니지드 그래프DB, 상용 안정성 |

**핵심기술과 클라우드 활용 연결**

1. **AI BOM 자동 매핑** — LLM이 고객사별 엑셀 양식을 분석해 시스템 필드로 자동 매핑. Bedrock 전환으로 모델 선택 유연성과 비용 관리 확보
2. **온톨로지 기반 관계 분석** — 부품-도면-공급사-공정 관계를 그래프로 모델링. Neptune 전환으로 프로토타입 수준에서 상용 서비스로 안정성 확보
3. **파일 에이전트** — 로컬 도면 파일과 클라우드 자동 동기화. S3+CloudFront 기반 파일 서빙
4. **변경관리 워크플로우** — 변경요청 생성→리뷰→병합 추적. RDS 기반 트랜잭션 안정성 위에 구축
