# SaaS 개발환경 지원 사업계획서 (초안)

이 문서는 2026년도 SaaS 개발환경 지원 수요기업 신청 양식의 요구 항목
10개와 추가 작성 항목인 현재 개발환경, 핵심기술, 추진 일정을 기준으로
정리한 초안이다. 평가 포인트인 지원 필요성, 구체적 활용 계획, SaaS 전환
및 개발 기대효과가 한 문서 안에서 일관되게 드러나도록 구성했다.

## 서비스 기본 정보

서비스 개요와 현재 상태를 먼저 요약한다. 실제 신청서 기입 시 기업 기본
정보와 함께 이 표를 축약해 사용할 수 있다.

| 항목 | 내용 |
| --- | --- |
| 서비스 솔루션명 | **Fabbit** |
| 서비스 한 줄 정의 | **중소 제조업의 도면·BOM·생산 데이터를 AI로 통합하는 제조 데이터 플랫폼** |
| 서비스 요약 | 도면 업로드만으로 BOM을 자동 추출하고, 구조화된 BOM 관리와 변경 이력 관리, 협업 워크플로우를 클라우드에서 제공하는 B2B SaaS |

## 1. 서비스 개요

Fabbit은 중소 제조업이 엑셀, 공유폴더, 메일에 흩어져 관리하던 제조
데이터를 클라우드에서 통합 관리할 수 있도록 설계된 서비스다. 이번 지원을
통해 개발환경을 안정화하고 파일럿 운영 수준까지 고도화하는 것이 목적이다.

### 개발 배경

국내 제조업 537,580개 사업체 중 10~299명 규모 중소기업은 77,389개이며,
이 구간에서 스마트공장 지원 DB 기준 PLM 도입 비중은 4.4% 수준에
불과하다. 현장에서는 도면은 폴더에, BOM은 엑셀에, 공급사와 변경 이력은
담당자 개인 경험에 의존하는 경우가 많아 검색, 버전관리, 변경 영향 분석,
생산 연계가 어렵다.

특히 1차 고객 검증과 콜드콜 36건+ 과정에서 다음 문제가 반복 확인됐다.

- "PLM" 자체를 모르지만, "도면/BOM 관리" 문제는 강하게 인식한다.
- 1~9인 영세 제조사는 관리 체계가 약해 도입 가능성이 낮고, **10~49명,
  매출 10억 이상** 제조사가 초기 타겟으로 적합하다.
- ERP를 이미 쓰더라도 도면과 BOM은 엑셀과 폴더로 관리하는 경우가 많아
  업스트림 데이터 정리 수요가 존재한다.

### 개발 내용

Fabbit의 개발 방향은 세 단계로 정리된다.

1. **AI 추출**: 도면에서 BOM을 자동 추출해 엑셀 수기 입력을 대체한다.
2. **구조화 관리**: 추출된 데이터를 시스템 안에서 BOM, 도면, 변경 이력으로
   연결해 검색과 추적이 가능하도록 만든다.
3. **데이터 활용 고도화**: 축적된 데이터를 기반으로 변경 영향 분석,
   공급망 리스크 탐지, 생산 연계 기능으로 확장한다.

현재는 1단계와 2단계의 핵심 기능이 구현되어 있으며, 본 사업을 통해
클라우드 기반 운영 안정화와 상용화 직전 수준의 완성도를 확보하고자 한다.

## 2. 서비스의 용도, 특징 및 주요 기능

Fabbit은 중소 제조업의 설계 데이터와 협업 데이터를 클라우드에서 관리하는
업무용 SaaS다. 기존에 사람이 직접 입력하고 정리하던 작업을 AI와 구조화된
데이터 모델로 대체하는 것이 핵심 특징이다.

### 서비스 용도

서비스는 단순 도면 보관 도구가 아니라, 제조 데이터의 생성, 관리, 협업을
클라우드에서 연결하는 업무 시스템을 목표로 한다.

- 도면 기반 BOM 작성 자동화
- 설계 데이터의 중앙화와 최신본 통제
- 부품, 도면, 변경요청, 담당자 간 관계 추적
- 향후 생산실적, 품질, 공급사 데이터와 연결되는 기반 구축

### 주요 기능

현재 제공하거나 본 사업 기간 내 고도화할 주요 기능은 다음과 같다.

| 구분 | 주요 기능 | 고객 가치 |
| --- | --- | --- |
| AI 데이터화 | DWG/PDF 도면 업로드, BOM 자동 추출, 추출 결과 검수 | 수작업 입력 시간 절감, 초기 도입 장벽 제거 |
| 구조화 관리 | 부품 목록, BOM 트리, 카테고리, 담당자 관리 | 엑셀/폴더 중심 운영에서 시스템 기반 운영으로 전환 |
| 협업 워크플로우 | 이슈, 변경요청, 프로젝트, 팀, 알림 | 설계 변경 이력과 담당 흐름 가시화 |
| 데이터 기반 확장 | 온톨로지, 공급사, 대시보드, AI 사용량 추적 | 향후 추천, 영향 분석, 운영 지표 관리 기반 확보 |

### 현재 구현 수준

문서 기준 2026년 3월 현재 서버 OpenAPI는 `118`개 path, `150`개
operation, `212`개 schema 규모이며, 다음 기능이 구현되어 있다.

- 인증, 조직, 프로젝트, 팀, 멤버 관리
- 도면 업로드와 파일 저장
- 도면 AI 인식(`mapping`)과 BOM 합성(`synthesis`)
- 부품/BOM 트리 관리와 공급사/온톨로지 연계
- 이슈/변경요청과 알림, 활동 이력

즉, 단순 아이디어 단계가 아니라 **클라우드 개발환경만 확보되면 파일럿 고객
운영이 가능한 MVP** 상태다.

## 3. 경쟁 제품과의 차별성

Fabbit의 차별성은 "AI 추출" 자체보다, 추출된 데이터를 시스템 안에서 바로
관리하고 이후 업무 흐름으로 연결한다는 점에 있다. 기술·기능 관점의 차이를
아래와 같이 정리할 수 있다.

| 비교 항목 | AI 도면 도구 | 기존 PLM/PDM | **Fabbit** |
| --- | --- | --- | --- |
| AI 도면/BOM 추출 | 있음 | 없음 | **있음** |
| 추출 결과의 시스템 내 등록 | 제한적 | 없음 | **있음** |
| 구조화 BOM 관리 | 없음 | 있음 | **있음** |
| 변경 이력/협업 흐름 | 없음 | 부분 제공 | **있음** |
| 클라우드 기반 빠른 도입 | 부분 제공 | 공급사별 상이 | **있음** |
| 향후 생산/공급사 데이터 연결 | 없음 | 제한적 | **확장 가능** |

### 핵심 차별 포인트

1. **AI로 진입 장벽을 없앤다.**
   기존 PLM은 먼저 데이터를 정리해 넣어야 시작할 수 있지만, Fabbit은
   도면 업로드 자체가 데이터 입력의 시작점이다.
2. **추출 후 관리가 이어진다.**
   AI 추출 결과를 엑셀로 내보내는 데서 끝나지 않고, BOM 트리, 부품,
   담당자, 변경요청과 연결된 상태로 관리한다.
3. **클라우드 SaaS 구조에 최적화되어 있다.**
   별도 구축형 도입 없이 중소 제조업이 바로 체험하고 파일럿할 수 있다.

## 4. 시장 파급효과

Fabbit이 해결하려는 문제는 특정 회사 한 곳의 불편이 아니라, 중소 제조업
전반에서 반복되는 구조적 문제다. 따라서 서비스가 상용화되면 개별 고객의
업무 효율 개선을 넘어 시장 전반의 디지털 전환 비용을 낮추는 효과가 있다.

### 시장 측면의 파급효과

시장 파급효과는 "한 기업의 업무 효율 개선"을 넘어, 그동안 구축형 솔루션이
커버하지 못했던 중소 제조업 구간의 디지털 전환을 확장하는 데 있다.

- **중소 제조업의 디지털 전환 저변 확대**:
  PLM 도입률이 낮은 시장에서, 고가 구축형 대신 클라우드 기반 경량 SaaS로
  초기 진입을 가능하게 한다.
- **도면/BOM 관리 영역의 SaaS 전환 촉진**:
  제조업 10인 이상 기업의 클라우드 이용률은 73.9% 수준으로, SaaS 수용
  기반은 이미 존재한다.
- **설계 데이터 표준화 촉진**:
  도면, BOM, 변경 이력이 구조화되면 이후 ERP, MES, 공급망 시스템과의
  연계 기반이 생긴다.
- **AI 활용 문턱 완화**:
  제조업 AI 도입률이 낮은 이유 중 하나는 정리된 데이터 부재다. Fabbit은
  AI 활용을 위한 데이터 기반을 먼저 제공한다.

### 기대되는 산업적 효과

서비스가 확산되면 제조 현장의 데이터 운영 방식에도 다음과 같은 변화가
누적될 수 있다.

| 구분 | 기대 변화 |
| --- | --- |
| 업무 방식 | 엑셀/폴더/메일 중심 관리에서 시스템 기반 협업으로 전환 |
| 데이터 관리 | 담당자 개인 의존에서 조직 자산화로 전환 |
| 공급망 대응 | 변경 영향과 리스크 추적 가능성 확대 |
| 후속 시장 | ERP 연계, 경량 MES, 품질추적 등 인접 SaaS 시장 확장 |

## 5. 기대 효과

본 사업을 통해 확보하려는 효과는 고객 가치와 기업 성과 두 축으로 나뉜다.
정량성과 정성 효과를 함께 제시하면 평가 항목인 "SaaS 전환 및 개발
기대효과 및 성과"를 명확히 설명할 수 있다.

### 고객 관점 기대 효과

고객이 직접 체감하는 효과는 반복 수작업 제거와 최신본 통제에 집중된다.

| 항목 | 기존 방식 | 도입 후 기대 효과 |
| --- | --- | --- |
| BOM 작성 | 도면 보고 수기 입력 | AI 자동 추출 기반 작성 시간 단축 |
| 도면 검색 | 폴더 탐색, 최신본 확인 어려움 | 시스템 검색과 이력 기반 조회 |
| 변경 반영 | 메일/구두 전달 의존 | 변경요청과 담당 흐름 기록 |
| 데이터 활용 | 파일 단위 보관 | 부품·도면·담당자 단위 구조화 |

### 기업 관점 기대 효과

사업 수행 기업인 Fabbit 관점에서는 검증 가능한 성과지표를 확보하는 것이
핵심이다.

- 파일럿 고객 3~5개사를 확보해 실제 현장 사용성을 검증한다.
- 파일럿 고객 중 2개사 이상 유료 전환 가능성을 확인한다.
- SaaS 운영에 필요한 월간 이용 현황 관리 체계를 구축한다.
- 향후 2년 성과관리 대상이 되는 매출, 고용, 특허, 투자 지표의 기초 데이터를
  축적한다.

### 협약 종료 시점 목표

협약 종료 시점에는 아래 목표가 확인 가능해야 한다.

| 항목 | 목표 |
| --- | --- |
| 제품 완성도 | 파일럿 운영 가능한 상용 직전 수준 확보 |
| 고객 검증 | 파일럿 3~5개사 확보 |
| 사업화 | 유료 전환 2개사 이상 목표 |
| 기술 고도화 | 도면 표제란 인식, 운영 안정화, 모니터링 체계 확보 |
| 성과 기반 | 사례화 가능한 고객 운영 데이터 확보 |

## 6. 수익모델

Fabbit은 중소 제조업의 협업 구조를 고려해 **회사 단위 Freemium** 모델을
채택한다. 조회 인원이 많은 제조 현장의 특성상 인당 과금보다 팀 단위 과금이
도입 장벽을 낮추고 전사 확산에 유리하다.

### 기본 요금 체계

요금 체계는 파일럿 진입과 유료 전환을 동시에 고려해 설계했다.

| 플랜 | 월 요금 | 멤버 | 스토리지 | 대상 |
| --- | --- | --- | --- | --- |
| Starter | 무료 | 5명 | 10GB | 파일럿 및 소규모 팀 |
| Team | 24.9만원 | 20명 | 500GB | 중소 제조팀 운영 |
| Business | 59.9만원 | 50명 | 2TB | 다부서 협업 운영 |
| Enterprise | 별도 협의 | 무제한 | 5TB+ | 전사 도입 |

### 수익 구조의 특징

수익 구조는 제조 현장의 사용 패턴과 도입 심리를 반영한다.

1. **무료 진입**으로 초기 사용 데이터를 확보한다.
2. **멤버 수와 데이터량 증가**에 따라 자연스럽게 상위 플랜으로 전환한다.
3. 추후 ERP 연동, 협력사 포털, 고급 분석 기능을 별도 부가 모듈로 확장할 수
   있다.

## 7. 국내외 주요 고객 프로파일 및 예상 시장 규모

Fabbit의 초기 고객은 제조 현장에서 실제로 도면과 BOM을 다루지만, 전용
시스템 없이 엑셀과 폴더로 관리하는 기업이다. 국내에서 우선 검증한 뒤,
클라우드 SaaS 구조를 바탕으로 해외 확장 가능성을 확보하는 전략이다.

### 국내 주요 고객 프로파일

국내 초기 시장은 영업 효율보다 문제 적합도가 높은 세그먼트를 우선 선택한다.

| 항목 | 내용 |
| --- | --- |
| 업종 | C25 금속가공, C29 기계장비, C28 전기장비 우선 |
| 규모 | 종사자 10~49명 우선, 이후 50~299명 확장 |
| 매출 | 10억원 이상 |
| 현재 상태 | ERP는 쓰지만 도면/BOM은 엑셀·폴더 중심 관리 |
| 의사결정자 | 대표, 설계팀장, 생산기술 책임자 |

### 해외 주요 고객 프로파일

해외 시장은 국내 제품 적합도 검증 이후 확장 대상으로 본다.

해외는 국내 PMF 확인 이후의 확장 시장이다. 우선순위는 일본, 동남아 등
중소 제조업 비중이 높고 클라우드 SaaS 확장이 가능한 지역이다.

### 시장 규모

시장 규모는 보수적 접근을 위해 국내 제조업 중 도면/BOM 활용 비중이 높은
업종과 기업 규모를 기준으로 계산했다.

| 구분 | 기준 | 규모 |
| --- | --- | --- |
| TAM | 도면/BOM 사용 제조업 전체 | 약 290,000개사 / 6,970억원 |
| SAM | 10~299명, PLM 미사용 제조업 | 약 32,600개사 / 783억원 |
| SOM | 3년 내 현실적 확보 목표 | 약 600개사 / 14.4억원 |

추가로 국내 PLM 시장은 약 1.36조원 수준으로 추정되며, 글로벌 PLM 시장은
출처에 따라 약 279억~463억 달러 규모로 형성되어 있다. Fabbit은 이 중
도입 장벽이 높은 기존 PLM 시장의 하단, 즉 아직 시스템화되지 않은 중소
제조업 구간을 공략한다.

## 8. 경쟁상황 및 경쟁우위

실질적인 경쟁 상대는 세 부류다. 첫째는 엑셀과 공유폴더 같은 현행 방식,
둘째는 AI 기반 도면 추출 도구, 셋째는 기존 PLM/PDM 또는 제조 IT 통합
솔루션이다. Fabbit은 이들 사이의 공백을 메우는 포지션을 지닌다.

### 경쟁 구도

현재 시장에서 고객이 비교하게 될 대안은 아래 네 가지로 요약된다.

| 경쟁군 | 강점 | 한계 | Fabbit의 우위 |
| --- | --- | --- | --- |
| 엑셀/폴더 | 익숙함, 초기 비용 없음 | 검색/이력/협업 부재 | 시스템 기반 구조화와 추적성 |
| AI 도면 추출 도구 | 빠른 추출 | 추출 후 관리 부재 | 추출과 관리가 한 시스템 안에서 연결 |
| 기존 PLM/PDM | 성숙한 관리 기능 | 수동 입력, 도입 부담 | AI 기반 진입과 빠른 클라우드 도입 |
| 제조 IT 통합 솔루션 | 기능 폭 넓음 | 복잡도와 도입비 부담 | 중소 제조업에 맞는 경량 SaaS |

### 경쟁우위 정리

Fabbit의 경쟁우위는 "기능 수"보다 "도입 순서와 연결 방식"에 있다.

1. **도입 속도**: 클라우드 서비스라 별도 구축 부담이 낮다.
2. **입력 자동화**: AI 추출이 시작점이라 초기 데이터 전환 비용이 낮다.
3. **업무 연결성**: 도면, BOM, 변경요청, 담당자 흐름이 한 제품 안에서
   이어진다.
4. **확장성**: 향후 생산실적, 품질, 공급사 데이터와 연결되는 구조를 이미
   고려하고 있다.

## 9. 개발완료 후 판매전략

개발 완료 후 판매전략은 "직접 검증이 가능한 시장"에서 시작해 "반복 가능한
확산 구조"로 넘어가는 방식으로 설계한다. 초기에는 영업 효율보다 고객 문제
적합도와 사례 확보가 중요하다.

### 1단계: 파일럿 확보 중심 직접 영업

초기 판매는 제품 설명보다 고객 문제를 정확히 짚는 접근이 중요하다.

- 경기도·수도권과 제조 밀집 지역을 중심으로 직접 접촉
- 콜드콜, 산업 컨퍼런스, 제조업 커뮤니티를 통한 리드 확보
- 무료 Starter 또는 저비용 파일럿으로 도입 장벽 최소화

### 2단계: 전환율 개선

파일럿 이후에는 사용성과 전환 논리를 정교화해 반복 가능한 영업 구조를
만든다.

- 파일럿 고객의 사용 데이터를 바탕으로 시간 절감, 검색 편의, 변경 이력
  관리 효과를 사례화
- 고객군을 10~49명, 매출 10억+, ERP 사용 기업 중심으로 재정렬
- 유료 전환 기준과 온보딩 절차를 표준화

### 3단계: 파트너 및 지원사업 연계

직접 영업으로 기준 고객군을 확보한 이후에는 확산 채널을 붙인다.

- 클라우드·스마트공장·AI 바우처 등 정부 지원사업과 연계
- ERP 벤더 또는 제조 IT 파트너와 업스트림 데이터 정리 도구로 협업
- 장기적으로는 산업군별 템플릿과 레퍼런스 기반 확산

### 판매 목표

판매 목표는 지원 기간 내 검증 가능한 수치만 제시했다.

| 구분 | 목표 |
| --- | --- |
| 초기 파일럿 | 3~5개사 |
| 유료 전환 | 2개사 이상 |
| 핵심 타겟 | 10~49명 제조기업, ERP 사용 중, 도면/BOM 엑셀 관리 기업 |

## 10. 조직/팀 구성 현황 및 조직/팀 구성원의 사업화 적합도

Fabbit 팀은 제품 개발과 시장 검증을 동시에 수행할 수 있는 소규모 실행
조직으로 구성되어 있다. 대표가 제품과 기술을 책임지고, 팀원이 영업과 고객
접점을 담당하는 구조다.

### 조직 구성

현재 조직은 제품 개발과 시장 검증을 동시에 돌릴 수 있도록 최소 인력으로
구성돼 있다.

| 구분 | 역할 | 주요 역량 |
| --- | --- | --- |
| 문성하 / 대표 | 제품 기획, 서버, 웹, 인프라 총괄 | PLM 도메인 이해, 풀스택 개발, 프로젝트 관리 |
| 남형우 / 팀원 | 영업, 기술지원, 고객 접점 | 기술 제품 고객지원 경험, 초기 영업 실행 |

### 사업화 적합도

팀의 강점은 경력 연차보다 문제 적합도와 실행 속도에 있다.

- 대표는 PLM 솔루션 기업에서 BOM 관리 모듈을 개발하며 현장 문제를 직접
  확인했고, 이후 SI 경험을 통해 프론트엔드, 백엔드, 인프라를 모두 다루는
  실행 역량을 축적했다.
- 팀원은 기술 제품 지원 경험을 바탕으로 고객 커뮤니케이션과 초기 영업을
  맡고 있으며, 실제 콜드콜 36건+와 현장 접촉을 통해 타겟 고객을 좁혀 왔다.
- 제품과 시장 검증이 동시에 가능한 구조라, 지원 기간 내 빠른 제품 개선과
  고객 피드백 반영이 가능하다.

## 현재 개발환경 및 핵심기술 관련 서술

현재 Fabbit은 단순한 기획안이 아니라 실제 기능이 구현된 SaaS 제품이다.
이번 지원사업이 필요한 이유는 이 제품을 **개발자 개인 환경 수준**에서
**파일럿·상용 운영 가능한 클라우드 환경 수준**으로 끌어올려야 하기 때문이다.

### 현재 개발환경

현재 개발환경은 기능 구현은 완료됐지만, 운영 체계는 더 보강해야 하는
상태다.

| 영역 | 현황 | 고도화 필요 이유 |
| --- | --- | --- |
| 서버/API | 인증, 조직, 프로젝트, 부품, 도면, 변경요청 API 구현 | 파일럿 고객 증가 시 운영 안정성, 로그 관리 필요 |
| 파일 처리 | 도면 업로드, 변환, AI 인식 파이프라인 구현 | 대용량 파일과 배치 처리 안정화 필요 |
| 저장소 | 도면/파일 저장 구조 구현 | 버전관리, 백업, 접근통제, 비용 최적화 필요 |
| 데이터 모델 | BOM, 부품, 공급사, 온톨로지 구조 구현 | 검색 속도와 데이터 연결성 강화 필요 |
| 웹 서비스 | 대시보드, 부품/BOM, 이슈/변경요청 화면 구현 | 고객 온보딩과 운영 편의성 보강 필요 |

### 핵심기술

본 서비스의 기술 차별성은 아래 네 축으로 설명할 수 있다.

1. **도면 AI 인식 및 BOM 자동 합성**
   도면 파일에서 부품표를 추출하고 구조화된 데이터로 변환한다.
2. **구조화 BOM 관리**
   부품, BOM 트리, 카테고리, 담당자 데이터를 연결해 관리한다.
3. **변경 협업 워크플로우**
   이슈, 변경요청, 알림 기능으로 설계 협업 흐름을 기록한다.
4. **온톨로지 기반 확장 구조**
   향후 공급사, 품질, 생산 데이터까지 연결할 수 있는 데이터 모델을 갖고
   있다.

### SaaS 개발환경 지원 필요성

연간 최초 1,000만원 규모의 바우처는 제품 기능 개발보다도 실제 운영이
가능한 SaaS 개발환경 구축에 우선 사용해야 한다. 이번 지원으로 필요한
클라우드 환경은 다음과 같다.

| 클라우드 자원 | 활용 목적 |
| --- | --- |
| 컴퓨팅 자원 | API 서버, 배치 작업, 도면 변환, 테스트/스테이징 운영 |
| 오브젝트 스토리지 | 도면 원본, 변환 파일, 추출 결과 저장 |
| 데이터베이스 | 서비스 운영 데이터와 구조화된 메타데이터 저장 |
| 네트워크/보안 | 외부 접속 안정화, 접근 제어, 서비스 보호 |
| 모니터링/로그 | 사용량 추적, 장애 대응, 월간 보고 체계 구축 |

활용계획서는 위 자원을 월별로 배치해 지원 기간 전반부에 핵심 인프라 구축,
중반부에 파일럿 운영, 후반부에 상용화 안정화가 이뤄지도록 설계할 예정이다.
이를 통해 중간 시점 소진율 관리와 월 1회 운영 보고 의무에도 대응한다.

즉, 지원받는 개발환경은 단순 인프라 비용 보전이 아니라 **실제 상용 SaaS 운영
체계를 만드는 데 필요한 핵심 생산수단**이다.
