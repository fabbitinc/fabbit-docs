# Fabbit 문서 인덱스

이 문서는 Fabbit 프로젝트 문서의 기본 구조와 각 폴더의 역할을 안내한다.
문서를 추가할 때는 아래 분류 기준을 우선 적용한다.

## 문서 시작점

- `README.md`: 문서 루트 인덱스
- `개요.md`: PLM 범위, 우선순위, 실행 단계의 기준 문서
- `10_제품/plm/제품-로드맵.md`: 제품 기능 로드맵/구현 상태
- `20_개발/인터페이스/서버-코드베이스-현황.md`: 서버/웹 코드베이스 및 API·라우트 현황
- `00_문서관리/표현-규칙/웹-표현-규칙.md`: API 용어 웹 UI 표현 규칙

## 문서 작성 공통 규칙

모든 신규 문서는 문서 최상단에 Obsidian 속성(frontmatter)을 포함한다.

```yaml
---
문서상태: 초안
문서소유자: 팀명 또는 이름
최종수정일: 2026-03-04
관련문서:
  - "[[개요]]"
---
```

추가 운영 규칙은 다음과 같다.

- 폴더별 진입 문서가 필요하면 `index.md`를 생성한다.
- 사용법/운영가이드는 폴더 대표 문서(예: `index.md` 또는 해당 폴더의 기준 문서)에 작성한다.
- 실제 기준 데이터 문서는 본문 내용만 유지한다.
- 파일명은 간단명료한 한글 중심으로 작성하고, 불필요한 약어 나열은 피한다.
- 예시: 운영 가이드는 대표 문서에 두고, 각 기준 문서에는 정의/규칙만 기록한다.
