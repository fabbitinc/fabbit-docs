# Fabbit 문서 인덱스

이 문서는 Fabbit 프로젝트 문서의 기본 구조와 각 폴더의 역할을 안내한다.
문서를 추가할 때는 아래 분류 기준을 우선 적용한다.

## 문서 시작점

- `index.md`: 문서 루트 인덱스
- `개요.md`: PLM 범위, 우선순위, 실행 단계의 기준 문서
- `00_문서관리/index.md`: 문서관리 운영 가이드
- `00_문서관리/용어집/index.md`: 용어집 분류 인덱스

## 폴더 구조

### 00_문서관리

문서 운영 기준을 관리한다.

- `용어집/`: 도메인/사업 용어집
- `용어집/도메인-용어집.md`: Fabbit 도메인 용어 정의
- `용어집/사업-용어집.md`: 사업/시장/성과 용어 정의
- `표현-규칙/웹-표현-규칙.md`: API 용어의 웹 UI 표시 문구 규칙
- `문서-네이밍-규칙.md`: 파일명, 상태, 버전 규칙
- `문서-수명주기.md`: 초안/검토/확정 흐름
- `템플릿/`: 문서 템플릿 모음

### 50_사업계획

예비창업패키지 제출용 평가 대응 문서를 관리한다.

- `서류평가/`: 평가항목 체크와 정합성 점검
- `서류평가/평가항목-체크리스트.md`: PSST 평가항목 점검표

### 10_제품

제품 도메인 지식과 로드맵을 관리한다.

- `plm/도메인/`: 도면관리, BOM, 변경관리 정의
- `plm/데이터모델/`: 엔터티, 상태, 관계 정의
- `plm/로드맵/`: 단계별 구현 계획

### 20_개발

구현과 운영에 필요한 기술 문서를 관리한다.

- `아키텍처/`: 시스템 구조와 설계 원칙
- `인터페이스/`: API 명세
- `연동/`: 외부 시스템 연동 방식
- `품질검증/`: 테스트 전략과 결과
- `배포/`: 릴리스, 배포 운영 기록

### 30_영업

영업 실행 문서를 관리한다.

- `가치제안/`: 고객 가치 메시지
- `경쟁-대응표/`: 경쟁사 비교 및 대응 포인트
- `데모-스크립트/`: 데모 시나리오
- `제안서/`: 제안 문서
- `자주묻는질문.md`: 대외 FAQ

### 40_타겟

시장/고객 세분화 문서를 관리한다.

- `산업군/`: 산업별 특성
- `이상적-고객-프로파일/`: 이상적 고객 프로파일
- `페르소나/`: 의사결정자/실무자 페르소나
- `계정-전략/`: 타겟 계정별 전략
- `유스케이스/`: 고객 문제와 적용 시나리오

### 90_회의

의사결정과 협업 기록을 관리한다.

- `주간회의/`: 정기 회의록
- `고객회의/`: 고객 미팅 기록
- `의사결정/`: 결정 로그와 변경 이력

## 문서 작성 공통 규칙

모든 신규 문서는 문서 최상단에 Obsidian 속성(frontmatter)을 포함한다.

```yaml
---
문서상태: 초안
문서소유자: 팀명 또는 이름
최종수정일: 2026-02-20
관련문서:
  - "[[개요]]"
---
```

추가 운영 규칙은 다음과 같다.

- 모든 폴더에는 `index.md`를 생성하고, 해당 폴더 문서의 진입점으로 사용한다.
- 사용법/운영가이드는 폴더의 `index.md`에 작성한다.
- 실제 기준 데이터 문서는 본문 내용만 유지한다.
- 파일명은 간단명료한 한글 중심으로 작성하고, 불필요한 약어 나열은 피한다.
- 예시: `00_문서관리/index.md`에 용어집 사용법/템플릿 사용법을 작성하고,
  각 용어집 문서에는 용어 정의만 기록한다.
