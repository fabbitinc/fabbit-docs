# Fabbit 문서 인덱스

이 문서는 Fabbit 프로젝트 문서의 기본 구조와 각 폴더의 역할을 안내한다.
문서를 추가할 때는 아래 분류 기준을 우선 적용한다.

## 문서 시작점

- `README.md`: 문서 루트 인덱스
- `10_제품/리서치/PLM-개요.md`: PLM 범위, 우선순위, 실행 단계의 기준 문서
- `10_제품/기획/index.md`: 전체 제품 기획 기준 문서
- `10_제품/기획/mes/index.md`: MES 전체 기획 문서
- `10_제품/전략/제품-로드맵.md`: 제품 기능 로드맵/구현 상태
- `20_개발/코드베이스-현황.md`: 서버/웹 코드베이스 및 API·라우트 현황
- `00_문서관리/규칙/웹-표현-규칙.md`: API 용어 웹 UI 표현 규칙

## 폴더 구조

### 00_문서관리

문서 운영 기준을 관리한다.

- `규칙/`: 문서-네이밍-규칙, 문서-수명주기, 웹-표현-규칙
- `용어집/`: 도메인-용어집, 사업-용어집
- `템플릿/`: 요구사항, 원페이지, 회의록

### 10_제품

제품 도메인 지식, 전략, 기획을 관리한다.

- `리서치/`: 도메인 리서치 (PLM-개요, 제조-도메인-시스템, 온톨로지-리서치)
- `전략/`: 제품 전략과 근거 (아이템-정의, 제품-로드맵, 문제해결-매핑, 경쟁사-분석, MVP-성능-측정)
- `기획/`: 기능별 상세 기획과 와이어프레임
  - `기획/mes/{번호}_{기능명}/기획.md`: MES 기능별 기획 + excalidraw 와이어프레임
  - `기획/plm/{번호}_{기능명}/`: PLM 기능별 기획 (예정)
  - `기획/site/{번호}_{기능명}/`: 대외 사이트 기획

### 20_개발

구현과 운영에 필요한 기술 문서를 관리한다.

- `코드베이스-현황.md`: `server` + `web/apps/web` 기준 코드베이스/노출 기능 현황

### 30_시장

시장 분석과 고객 검증 문서를 관리한다.

- `검증/`: 고객문제-검증-리포트 템플릿, 고객-인터뷰-질문지
- `검증/인터뷰/`: 회차별 콜드콜 기록
- `검증/리포트/`: 회차별 고객문제 검증 리포트
- `분석/`: 시장규모-산정, 시장조사-컨퍼런스

### 40_사업

사업 전략, 발표, 지원사업 문서를 관리한다.

- `전략/`: 수익모델-정의, 시장진입-실행계획, 개발계획-마일스톤, 팀구성-계획
- `발표/`: 대표자-스토리, 발표-구성안, 질의응답-대비
- `지원사업/`: 예비창업패키지, SaaS-개발환경-지원, 경기-기술창업지원-통합모집, 경기창업공모-G스타오디션

### 90_회의

의사결정과 협업 기록을 관리한다.

- `주간회의/`: 정기 회의록
- `의사결정/`: 결정 로그와 변경 이력

## SSOT 원칙

동일 주제의 문서는 한 곳에만 원본을 두고, 다른 곳에서는 `[[]]` 링크로 참조한다.

- 경쟁사 분석 원본: `10_제품/전략/경쟁사-분석.md`
- 제품 로드맵 원본: `10_제품/전략/제품-로드맵.md` (일정은 여기, 사업계획서에서 링크 참조)
- 기능 기획 원본: `10_제품/기획/` (개발에서 링크 참조)

## 문서 작성 공통 규칙

모든 신규 문서는 문서 최상단에 Obsidian 속성(frontmatter)을 포함한다.

```yaml
---
문서상태: 초안
문서소유자: 팀명 또는 이름
최종수정일: 2026-03-09
관련문서:
  - "[[10_제품/리서치/PLM-개요]]"
---
```

추가 운영 규칙은 다음과 같다.

- 폴더별 진입 문서가 필요하면 `index.md`를 생성한다.
- 사용법/운영가이드는 폴더 대표 문서(예: `index.md` 또는 해당 폴더의 기준 문서)에 작성한다.
- 실제 기준 데이터 문서는 본문 내용만 유지한다.
- 파일명은 간단명료한 한글 중심으로 작성하고, 불필요한 약어 나열은 피한다.
- 기획 문서는 기능 = 폴더 단위로 관리하고, 기획.md + excalidraw 와이어프레임을 함께 둔다.
