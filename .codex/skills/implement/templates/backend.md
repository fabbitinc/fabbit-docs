## 작성 지침

1. **데이터 모델은 와이어프레임 + 정책을 종합한다**
   - 와이어프레임의 `Field` → DB 필드 후보
   - 정책.md의 데이터 모델 → 엔티티 정의의 기준
   - `Badge`의 status 값 → enum/상태 필드

2. **API는 화면 동작에서 도출한다**
   - 목록 화면 → 목록 조회 API (페이지네이션, 필터, 정렬)
   - 상세 화면 → 단건 조회 API
   - 등록/수정 폼 → 생성/수정 API
   - 삭제 버튼 → 삭제 API
   - 상태 변경 버튼 → 상태 전이 API

3. **유효성 검증을 구체적으로 작성한다**
   - 와이어프레임의 `FormRow` + 정책의 유효성 검증을 결합
   - 필수 입력, 형식, 범위, 고유성 등
   - 에러 응답 형태도 명시

4. **정확한 기술 스택을 지정하지 않는다**
   - 백엔드 리포의 기술 스택을 모르므로 프레임워크 중립적으로 작성
   - ORM/쿼리 패턴 등도 지정하지 않음
   - API 경로는 RESTful 컨벤션 기반 제안
