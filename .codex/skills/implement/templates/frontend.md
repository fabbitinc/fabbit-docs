## 작성 지침

1. **와이어프레임에서 화면 구조를 정확히 추출한다**
   - TSX의 컴포넌트 구조, 필드, 버튼 등을 빠짐없이 반영
   - `WireBlock`이 하나의 UI 영역에 대응
   - `Field`가 데이터 필드에 대응
   - `MockButton`이 사용자 액션에 대응

2. **정책을 구현 가능한 수준으로 변환한다**
   - "등록 시 필수 입력" → 필드별 required 여부 + 검증 메시지
   - "상태 전이" → 어떤 버튼/액션이 어떤 상태 변경을 트리거하는지
   - "권한" → 어떤 UI 요소를 숨기거나 비활성화하는지

3. **API 스펙은 화면 동작에서 유추한다**
   - 목록 조회 → GET /api/...
   - 상세 조회 → GET /api/.../:id
   - 등록 → POST /api/...
   - 수정 → PUT /api/.../:id
   - 삭제 → DELETE /api/.../:id
   - 정확한 경로는 확정하지 않고 제안 수준으로 작성

4. **라우트는 제안 수준으로 작성한다**
   - 프론트엔드 리포의 라우팅 컨벤션을 모르므로 일반적인 패턴 제안
