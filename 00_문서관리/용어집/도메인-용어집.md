# Fabbit 도메인 용어집

이 문서는 Fabbit 제품 도메인(PLM, 도면관리, BOM, 변경관리, 온톨로지) 관련 핵심 용어를
정의한다.

## 용어

### PLM (Product Lifecycle Management)
- 분류: 제품/프로세스
- 정의: 제품의 기획부터 폐기까지 전 수명주기 데이터를 통합 관리하는 체계
- 사용 맥락: 제품 데이터 표준화, 변경 추적, 부서 간 협업 기준
- 관련 용어: PDM, BOM, ECR, ECO, SSOT
- 혼동 주의: 단순 문서 보관 시스템과 동일 개념이 아님
- 예시: "PLM 기준으로 도면 승인 이력을 추적한다."

### SSOT (Single Source of Truth)
- 분류: 데이터 거버넌스
- 정의: 동일 데이터에 대한 공식 기준 원본을 하나로 유지하는 원칙
- 사용 맥락: 도면 최신본, BOM 기준본, 상태코드 일관성 관리
- 관련 용어: 베이스라인, 리비전, 감사로그
- 혼동 주의: 데이터 백업본 또는 캐시 데이터와 구분 필요
- 예시: "도면 리비전 정보는 PLM이 SSOT다."

### 도면 버전 (Version)
- 분류: 문서관리
- 정의: 작업 중 변경 이력을 관리하는 내부 수정 단위
- 사용 맥락: 검토 전 초안 반복, 작성자 작업 이력 추적
- 관련 용어: 리비전, 승인, 배포
- 혼동 주의: 공식 배포 기준인 리비전과 동일하게 사용하지 않음
- 예시: "v0.7 버전에서 치수 오탈자를 수정했다."

### 도면 리비전 (Revision)
- 분류: 문서관리
- 정의: 승인 후 공식 배포되는 문서의 기준 개정 단위
- 사용 맥락: 현장 배포본 관리, 제조/품질 기준 연동
- 관련 용어: 버전, 승인, 배포, ECO
- 혼동 주의: 작업 버전 증가와 리비전 승격은 별도 절차
- 예시: "리비전 B가 현재 생산 기준본이다."

### BOM (Bill of Materials)
- 분류: 제품구성
- 정의: 제품을 구성하는 부품, 수량, 구조 정보를 계층으로 정리한 명세
- 사용 맥락: 설계 검토, 생산 준비, 변경 영향 분석
- 관련 용어: EBOM, MBOM, SBOM, where-used
- 혼동 주의: 단일 부품 목록과 달리 계층 구조와 관계 정보를 포함
- 예시: "BOM 비교로 변경된 하위 조립품을 확인했다."

### EBOM (Engineering BOM)
- 분류: 제품구성
- 정의: 설계 관점에서 정의한 BOM
- 사용 맥락: 설계 변경 반영, 엔지니어링 기준 관리
- 관련 용어: MBOM, ECO
- 혼동 주의: 생산 공정 최적화 관점인 MBOM과 목적이 다름
- 예시: "EBOM 기준으로 설계 변경 영향을 검토한다."

### MBOM (Manufacturing BOM)
- 분류: 제품구성
- 정의: 생산 관점에서 공정/조립 순서를 반영한 BOM
- 사용 맥락: 생산 지시, 자재 준비, 작업 표준 연계
- 관련 용어: EBOM, 공정 라우팅, Effectivity
- 혼동 주의: 설계 기준 EBOM을 그대로 복제한 문서가 아님
- 예시: "MBOM에서 대체품 적용 시점을 조정했다."

### ERP (Enterprise Resource Planning)
- 분류: 제조 시스템
- 정의: 회계, 구매, 재고, 생산 등 기업 자원을 통합 관리하는 시스템
- 사용 맥락: BOM 기반 자재 소요 산출, 발주, 원가 계산
- 관련 용어: MES, PLM, BOM
- 혼동 주의: PLM이 "무엇을 만들 것인가"라면 ERP는 "어떻게 조달·생산할 것인가"
- 예시: "ECO 승인 후 BOM 변경이 ERP로 전달됐다."

### MES (Manufacturing Execution System)
- 분류: 제조 시스템
- 정의: 생산 현장의 작업 지시, 실적 수집, 품질 관리를 실시간으로 수행하는 시스템
- 사용 맥락: 작업 지시 배분, 실적 추적, 불량 관리, 설비 모니터링
- 관련 용어: ERP, PLM, MBOM
- 혼동 주의: ERP가 계획 단위라면 MES는 현장 실행 단위
- 예시: "MES에서 작업 실적을 수집하여 ERP 생산 실적과 연동한다."

### ECR (Engineering Change Request)
- 분류: 변경관리
- 정의: 변경 필요성을 공식 제기하는 요청 단계
- 사용 맥락: 품질 이슈, 원가 개선, 규격 대응 요청 접수
- 관련 용어: ECO, 변경 영향 분석
- 혼동 주의: 승인 후 실행 지시인 ECO와 구분
- 예시: "고객 불량 대응으로 ECR을 등록했다."

### ECO (Engineering Change Order)
- 분류: 변경관리
- 정의: 승인된 변경을 실행/배포하는 지시 단계
- 사용 맥락: 도면 리비전 승격, BOM 반영, 시스템 연동 전파
- 관련 용어: ECR, 리비전, 배포
- 혼동 주의: 요청(ECR) 없이 임의로 변경을 배포하지 않음
- 예시: "ECO 승인 후 BOM 변경이 ERP로 전달됐다."

### 온톨로지 (Ontology)
- 분류: 데이터 모델링
- 정의: 특정 도메인의 개념(엔티티)과 관계를 형식적으로 정의한 의미론적 모델. 단순 스키마와 달리 "부품은 공급사로부터 조달된다", "도면은 BOM을 포함한다" 같은 관계의 의미까지 명시한다
- 사용 맥락: Fabbit 3단계 비전의 핵심. 부품·공급사·도면·담당자를 의미론적으로 연결하여 AI가 데이터 간 관계를 이해하고 추론할 수 있게 한다
- 관련 용어: 지식그래프, 그래프 DB, 디지털 스레드
- 혼동 주의: 철학의 "존재론"과 같은 단어이지만, IT에서는 데이터 모델링 방법론을 의미
- 예시: "부품-공급사-도면-담당자 온톨로지를 정의하여 변경 영향 추적을 자동화한다."

### 지식그래프 (Knowledge Graph)
- 분류: 데이터 모델링
- 정의: 온톨로지를 기반으로 실제 데이터(인스턴스)를 노드와 엣지로 연결한 그래프 구조의 데이터베이스
- 사용 맥락: 온톨로지가 "설계도"라면 지식그래프는 "실제 건물". 부품 A → 공급사 B → 담당자 C 같은 실제 관계 데이터가 들어간다
- 관련 용어: 온톨로지, 그래프 DB, SSOT
- 혼동 주의: 온톨로지(스키마)와 지식그래프(인스턴스)는 구분해서 사용
- 예시: "지식그래프에서 '이 부품이 단종되면 영향받는 제품'을 3홉 이내로 탐색한다."

### 그래프 DB (Graph Database)
- 분류: 인프라
- 정의: 데이터를 노드(정점)와 엣지(관계)로 저장·쿼리하는 데이터베이스. 관계형 DB의 JOIN 없이 관계 탐색이 가능
- 사용 맥락: Fabbit은 Apache AGE(PostgreSQL 확장)를 사용. BOM 계층 탐색, where-used, 변경 영향 분석에 관계형 DB보다 효율적
- 관련 용어: 온톨로지, 지식그래프, Neo4j, Apache AGE
- 혼동 주의: 그래프 DB는 저장 엔진이고, 지식그래프는 그 위에 올리는 데이터 모델
- 예시: "BOM 계층 10단계 재귀 탐색을 그래프 DB로 수행한다."

### 디지털 스레드 (Digital Thread)
- 분류: 제조 전략
- 정의: 제품의 설계→제조→운영→폐기까지 전 수명주기 데이터를 끊김 없이 연결하는 데이터 흐름
- 사용 맥락: 온톨로지 기반으로 부서·시스템 간 데이터 사일로를 해소하는 것이 디지털 스레드의 실현 방법. 시장 규모 CAGR 21.5% (2024~2030)
- 관련 용어: 디지털 트윈, 온톨로지, PLM
- 혼동 주의: 디지털 트윈(현실 객체의 가상 복제)과는 다른 개념. 스레드는 "연결", 트윈은 "복제"
- 예시: "설계 변경이 생산·구매·서비스까지 자동 전파되는 디지털 스레드를 구현한다."

### 디지털 트윈 (Digital Twin)
- 분류: 제조 전략
- 정의: 물리적 자산(제품, 공장, 설비)을 디지털로 복제하여 시뮬레이션·모니터링하는 가상 모델
- 사용 맥락: 팔란티어 Foundry가 온톨로지로 현실 객체를 모델링하는 것이 디지털 트윈의 일종
- 관련 용어: 디지털 스레드, 온톨로지, IoT
- 혼동 주의: 3D 모델과 동일하지 않음. 데이터 연결과 실시간 동기화가 핵심
- 예시: "공장 설비의 디지털 트윈으로 유지보수 시점을 예측한다."

### 팔란티어 Foundry (Palantir Foundry)
- 분류: 해외 사례
- 정의: 팔란티어의 데이터 통합·분석 플랫폼. 온톨로지 기반으로 다양한 데이터 소스를 연결하고, AI가 의사결정을 지원
- 사용 맥락: Fabbit 3단계 비전의 레퍼런스. "팔란티어 Foundry의 중소 제조업 버전"이 Fabbit의 포지셔닝
- 관련 용어: 온톨로지, Systems of Action, Agentic AI
- 혼동 주의: 팔란티어는 대기업·정부 대상. Fabbit은 중소 제조업 대상으로 AI 추출을 진입 전략으로 삼는 점이 다름
- 주요 수치: Forrester TEI 기준 ROI 315%, 투자 회수 6개월 미만, Airbus A350 인도 33% 가속화
- 예시: "Foundry처럼 온톨로지 기반으로 데이터를 연결하되, AI 추출로 진입장벽을 없앤다."
