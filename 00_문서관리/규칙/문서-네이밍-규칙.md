# 문서 네이밍 규칙

이 문서는 Fabbit 문서 파일명과 버전 표기 규칙을 정의한다.

## 기본 원칙

- 파일명과 폴더명은 한글을 사용한다.
- 단어 구분은 공백 대신 `-`를 사용한다.
- 확장자는 `md`를 사용한다.
- 각 폴더의 진입 문서는 반드시 `index.md`로 고정한다.

## 파일명 규칙

- 형식: `주제-세부주제.md`
- 예시: `도면관리-요구사항.md`, `시장규모-산정.md`
- 권장: 읽는 즉시 의미가 전달되는 짧은 한글 파일명을 사용한다.
- 금지: 공백 포함 파일명, 약어 나열형 파일명, 의미 없는 축약어, 임시 파일명(`new.md`, `test.md`)

## 상태/버전 표기 규칙

- 파일명에 상태값(`초안`, `검토`, `확정`)을 붙이지 않는다.
- 상태는 문서 최상단 Obsidian frontmatter에서만 관리한다.
- 파일명에 날짜를 붙이지 않는다. 날짜는 문서 본문에 기록한다.
- 같은 주제 문서가 연속 개정될 때도 파일명은 유지한다.

## 문서 상단 frontmatter 규칙

모든 신규 문서는 문서 최상단에 아래 Obsidian 속성을 포함한다.

```yaml
---
문서상태: 초안
문서소유자: 팀명 또는 이름
최종수정일: 2026-02-20
관련문서:
  - "[[10_제품/리서치/PLM-개요]]"
---
```

- `문서상태`: `초안`, `검토중`, `확정` 중 하나를 사용한다.
- `문서소유자`: 담당자 이름 또는 팀명을 기록한다.
- `최종수정일`: `YYYY-MM-DD` 형식으로 기록한다.
- `관련문서`: 내부 문서 링크를 리스트로 기록한다.

## 예외 규칙

- 외부 시스템 제약으로 영문 파일명이 필요한 경우, 사유를 문서 상단에 남긴다.
- 외부 공유용 산출물은 별도 폴더를 만들고 해당 폴더 `index.md`에 규칙을 명시한다.
