# 웹 표현 규칙

Fabbit 웹 UI에서 사용하는 표현 규칙을 정리합니다.
이 문서는 용어 사전이 아니라 **API 용어**(백엔드 식별자)를
**UI 표시**(사용자에게 보이는 텍스트)로 일관되게 변환하기 위한 기준입니다.

---

## 핵심 엔티티

| API 용어 | UI 표시 | 설명 |
|----------|---------|------|
| Part | 부품 | 제품을 구성하는 개별 부품. "품목"과 혼용하지 않음 |
| Drawing | 도면 | 부품 정보의 원본 출처인 설계 도면 |
| Supplier | 공급사 | 부품을 공급하는 외부 업체 |
| BOM (Bill of Materials) | BOM | 부품 간 구성 관계 (부모-자식). 풀어쓸 때는 "부품 구성" |
| Project | 프로젝트 | 부품을 묶어 관리하는 단위 |
| Issue | 이슈 | 작업/문제/요청을 추적하는 협업 단위 |
| Change Request | 변경 요청 | 변경 검토/반영을 위한 이슈 유형 |
| Label | 라벨 | 이슈/변경 요청 분류 태그 |
| Team | 팀 | 멤버를 묶는 협업 단위 |
| Member | 멤버 | 조직에 속한 사용자 |
| Notification | 알림 | 멘션/활동 기반 사용자 알림 |

## 관계 유형

| API 용어 | UI 표시 | 설명 |
|----------|---------|------|
| CONSISTS_OF | 구성 | 부품이 다른 부품으로 구성됨 (BOM 관계) |
| SUPPLIED_BY | 공급 | 공급사가 부품을 공급 |
| DEFINED_BY | 정의 | 도면이 부품을 정의 |
| HAS_ITEM | 포함 | 프로젝트가 부품을 포함 |

## 프로세스 / 액션

| API 용어 | UI 표시 | 설명 |
|----------|---------|------|
| Synthesis | 부품 업로드 | 업로드된 엑셀 파일을 파싱하여 Part/BOM/Supplier 등을 시스템에 적재하는 백그라운드 작업 |
| Mapping | 매핑 | 원본 데이터 컬럼을 표준 온톨로지 속성에 매핑하는 과정 |
| Onboarding | 온보딩 | 신규 사용자의 초기 설정 플로우 (워크스페이스 → 업로드 → 매핑 → 탐색) |
| Drawing Analysis | 도면 분석 | AI가 도면에서 표제란/부품표를 인식하는 과정 |
| Drawing Conversion | 도면 변환 | DWG 등 원본 파일을 PDF/썸네일로 변환하는 과정 |
| Lookup | 검색 선택 | picker/autocomplete를 위한 경량 조회 |
| Sync | 동기화 | 전달된 ID 목록을 최종 상태로 맞추는 일괄 갱신 |
| Review | 검토 | 변경 요청(CR)에 대한 승인/반려 처리 |
| Archive | 보관 | 프로젝트를 읽기 전용 상태로 전환 |
| Unarchive | 보관 해제 | 보관된 프로젝트를 활성 상태로 복원 |

## 상태값

### 합성/작업 상태

| API 용어 | UI 표시 | 사용 맥락 |
|----------|---------|-----------|
| PENDING | 대기 | Synthesis, Drawing Conversion 등 |
| PROCESSING | 처리 중 | Synthesis batch |
| COMPLETED | 완료 | 정상 완료 |
| COMPLETED_WITH_ERRORS | 부분 완료 | 일부 오류 포함 완료 |
| FAILED | 실패 | 작업 실패 |

| API 용어 | UI 표시 | 사용 맥락 |
|----------|---------|-----------|

| CLOSED | 닫힘 | 이슈/변경 요청 종료 상태 |
| DRAFT | 초안 | 변경 요청 작성 중 상태 |
| SUBMITTED | 검토 중 | 변경 요청 제출 완료 상태 |
| MERGED | 반영 완료 | 변경 요청 반영 완료 상태 |

### 리뷰 상태

| API 용어 | UI 표시 | 사용 맥락 |
|----------|---------|-----------|
| PENDING | 대기 | CR 검토 대기 상태 |
| APPROVED | 승인 | CR 검토 결과 |

## 혼용 금지

| 잘못된 사용 | 올바른 사용 | 이유 |
|------------|------------|------|
| 품목 (Part 의미로) | 부품 | "품목"은 Item 도메인에서 사용. Part는 "부품"으로 통일 |
| 합성 | 부품 업로드 | 사용자에게 "합성"은 의미 불명확. "부품 업로드"가 동작을 설명 |
| BOM 링크 | BOM 연결 | "링크"보다 "연결"이 한국어로 자연스러움 |
| CR | 변경 요청 | 사용자 대상 UI에서는 약어보다 전체 용어가 명확 |
| 이슈 상태 + CR 상태 혼용 | 상태 / 변경 요청 상태 분리 | `state`와 `cr_state`는 의미가 다름 |
| 담당자=팀 | 담당자 / 담당 팀 분리 | `assignees`와 `assigned_teams`는 별도 필드 |
| 품목 메뉴 | 부품 메뉴 | 현재 웹 기본 경로는 `/parts` 기준 |
