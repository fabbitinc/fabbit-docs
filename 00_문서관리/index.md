# 문서관리 인덱스

이 문서는 문서 운영 기준과 작성 방식의 시작점이다. 문서관리 폴더에서는
"규칙/사용법"과 "실제 기준 문서"를 분리해 유지한다.

## 구성 문서

- `용어집.md`: 프로젝트 공통 용어 사전
- `문서-네이밍-규칙.md`: 파일명, 상태, 버전 규칙
- `문서-수명주기.md`: 초안/검토/확정 흐름
- `템플릿/`: 문서 유형별 템플릿

## 용어집 사용법

용어 관련 규칙과 사용 가이드는 이 인덱스에서 관리한다. `용어집.md`에는
용어 정의만 기록한다.

1. 신규 용어가 필요하면 먼저 기존 용어 중복 여부를 확인한다.
2. 용어는 `한글명 (영문/약어)` 형식으로 추가한다.
3. 각 용어에는 분류, 정의, 사용 맥락, 관련 용어, 혼동 주의, 예시를 작성한다.
4. 기존 용어와 충돌하는 표현이 있으면 해당 문서를 먼저 정리한 뒤 반영한다.

## 템플릿 사용법

템플릿 사용 규칙도 이 인덱스에서 관리한다.

1. 새 문서를 작성할 때는 `템플릿/` 하위 문서를 먼저 복사해 시작한다.
2. 문서 최상단 Obsidian frontmatter(문서상태, 문서소유자, 최종수정일, 관련문서)를 반드시 채운다.
3. 문서 목적에 맞지 않는 섹션은 삭제하고, 필요한 섹션만 유지한다.
4. 확정 문서로 전환할 때는 템플릿 안내 문구를 제거한다.
